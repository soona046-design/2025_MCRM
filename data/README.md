# M-CRM 실데이터 폴더

> ⚠️ **`original/`의 파일은 삭제 금지** — CRM에 적재된 실제 회사 데이터의 원본입니다.
> 백업: 구글 드라이브 `내 드라이브/인사이트/MCRM/` (2026-07-19 통합 — 매출 PDF·성과 CSV·문의 원본이 한 폴더).
> `original/`은 개인정보(연락처)가 있어 git에 올라가지 않습니다(.gitignore). 이 README와 `templates/`만 커밋됩니다.

## 폴더 구조

```
data/
├── original/     ← 실데이터 원본 (git 제외 · 삭제 금지)
│   ├── 신규문의-2025.csv      2025년 문의 65건 (Notion 내보내기)
│   ├── 신규문의-2026.csv      2025.11~2026.7 문의 56건 (구글시트 내보내기)
│   ├── 광고성과-2025.csv      2025년 8~12월 주간 광고 지표
│   ├── 광고성과-2026.csv      2026년 1~7월 주간 광고 지표
│   └── 매출-2026-정리.csv     파서 산출물 (import:revenue 입력)
└── templates/    ← 재업로드용 서식 (예시 데이터 — git 포함)
```

## 다시 올릴 때 — 파일별 서식과 명령

### ① 광고성과 (`광고성과-YYYY.csv`)

- **얻는 법**: 온라인성과 구글시트 → 파일 → 다운로드 → CSV. 기존 시트 구조 그대로면 됩니다(파서는 좌측 주간 블록만 읽음 — 우측 월 요약 블록·합계행은 무시).
- **필수 컬럼**(templates/광고성과-서식.csv): `월-주차`(예: `7월 1주차`) · `채널` · `노출` · `클릭` · `클릭당비용(원)` · `비용(원)` · `클릭율`
- **채널 허용값**: 파워링크 / 플레이스 / 파워컨텐츠 / GDN / 유튜브 / 메타 (이 6개 표기 그대로 — CRM이 네이버/구글/메타로 자동 분류)
- **적재**: 파일을 `original/`에 덮어쓰기 →
  ```bash
  cd mcrm-v2-backend && php artisan import:real --ads-only
  ```

### ② 클라이언트 매출 (PDF `26년N월.pdf`)

- **얻는 법**: 매출 시트를 Chrome 인쇄 → PDF (기존 1~6월과 같은 방식). 구글 드라이브 `인사이트/MCRM/`에 저장 (2026-07-19 변경).
- **적재**:
  ```bash
  cd mcrm-v2-backend
  python3 scripts/parse_revenue_pdfs.py '<PDF 폴더>'   # 시트 소계와 자동 검산
  php artisan import:revenue
  ```
- PDF 없이 시트에서 직접 CSV를 만들 수도 있습니다 → templates/매출-정리-서식.csv 서식으로 `original/매출-2026-정리.csv` 작성 후 `import:revenue`. (`customer_id`·`matched_name`은 비워두면 **전부 기존 고객** 처리되므로, 신규유입 매칭이 필요하면 파서 경로를 쓰세요.)

### ③ 신규 문의 (`신규문의-YYYY.csv`)

- **원칙**: 신규 문의는 파일이 아니라 **CRM 수기 입력 화면**으로 넣습니다 (UTM 증거 확보 — [문의-입력-가이드](../docs/v2/문의-입력-가이드.md)).
- 부득이 파일로 일괄 이관할 때의 서식(templates/신규문의-서식.csv):
  - `날짜(금)`: `M/D/YYYY` · `유입경로`: 검색/블로그/홈페이지/메타/소개(○○)/BDEX/이벤트 · `1차 컨택 경로`: 전화/카카오톡/홈페이지/문자/네이버톡톡 · `결과`: 동의/비동의/검토 중/제안 중/보류/거절/해당없음
  - ⚠️ 파일 이관은 `import:real --fresh`(전체 초기화) 경로라 **CRM에서 입력한 데이터가 지워집니다** — 초기 이관 외에는 사용하지 마세요.
