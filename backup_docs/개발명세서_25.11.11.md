## 1. 프로젝트 개요

### 1.1 목적
치과/의료 클리닉을 위한 종합 고객관계관리(CRM) 시스템으로, 환자 여정 전체를 추적하고 마케팅 ROI를 최적화합니다.

### 1.2 환자 여정 (Patient Funnel)
```
방문자 (Visitor)
    ↓ UTM 추적
리드/문의 (Lead)
    ↓ 스코어링 & 배정
상담 (Consultation/Ticket)
    ↓ SLA 관리
예약 (Appointment)
    ↓ 리마인더
내원/시술 (Treatment)
    ↓ 결제 추적
매출/재방문 (Revenue/Rebooking)
```

### 1.3 주요 가치
- **마케팅 ROI 추적**: 채널별 광고비용 대비 매출 분석
- **리드 자동화**: 자동 배정, 우선순위 스코어링, SLA 모니터링
- **환자 경험**: 리마인더, 재예약 제안, 커뮤니케이션 히스토리
- **컴플라이언스**: GDPR 준수, 개인정보 마스킹, 감사 로그

#### users (사용자)

```sql
CREATE TABLE users (

    role ENUM('슈퍼관리자', '지점관리자', '상담매니저', '마케터', '의사'),
    clinic_id UUID,                   -- 멀티테넌시
    active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

#### visits (방문자 추적)
```sql
CREATE TABLE visits (
    visit_id UUID PRIMARY KEY,
    session_id VARCHAR(255),          -- 세션 ID
    client_id VARCHAR(255),           -- First-party client ID
    utm_source VARCHAR(100),          -- 마케팅 채널
    utm_medium VARCHAR(100),
    utm_campaign VARCHAR(100),
    utm_term VARCHAR(100),
    utm_content VARCHAR(100),
    referrer TEXT,
    landing_page TEXT,
    ip_address VARCHAR(45),
    user_agent TEXT,
    created_at TIMESTAMP
);
```

#### leads (리드/환자)
```sql
CREATE TABLE leads (
    lead_id UUID PRIMARY KEY,
    visit_id UUID,                    -- visits 테이블 참조
    name VARCHAR(255),
    primary_phone VARCHAR(20),        -- 마스킹 처리
    email VARCHAR(255),               -- 해싱 처리
    utm_source VARCHAR(100),          -- 채널 (배열 가능)
    utm_campaign VARCHAR(100),
    status ENUM('신규', '문의중', '상담완료', '미팅완료', '계약완료', '보류', '거절'),
    score INT DEFAULT 50,             -- 우선순위 점수 (0-100)
    assignee_id UUID,                 -- 담당 상담사
    treatment TEXT,                   -- 문의 서비스 (배열)

    last_contact_at TIMESTAMP,
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    deleted_at TIMESTAMP,

    FOREIGN KEY (assignee_id) REFERENCES users(user_id) ON DELETE SET NULL
```

### 4.2 리드 관리 (Lead Management)

**목적**: 문의 리드를 자동으로 스코어링하고 상담사에게 배정

#### 리드 스코어링 로직
```php
// app/Services/LeadScoringService.php
public function calculateScore(Lead $lead): int
{
    $score = 50; // 기본 점수

    // 채널 점수 (0-20)
    $channelScores = [
        'google' => 20,
        'naver' => 18,
        'facebook' => 15,
        'instagram' => 12,
        'direct' => 10,
    ];
    $score += $channelScores[$lead->utm_source] ?? 5;

    // 문의 서비스 점수 (0-20)
    $highValueServices = ['임플란트', '라미네이트', '교정'];
    if (array_intersect($lead->treatment, $highValueServices)) {
        $score += 20;
    }

    // 시간 기반 점수 (0-10)
    // 영업시간 내 문의는 높은 점수
    $hour = date('H');
    if ($hour >= 9 && $hour <= 18) {
        $score += 10;
    }

    return min(100, max(0, $score));
}
```

#### 리드 자동 배정
```php
// 라운드로빈 또는 워크로드 기반 배정
public function assignLead(Lead $lead): void
{
    $availableAgents = User::where('role', '상담매니저')
        ->where('active', true)
        ->withCount('activeLeads')
        ->orderBy('active_leads_count', 'asc')
        ->get();

    if ($availableAgents->isNotEmpty()) {
        $lead->assignee_id = $availableAgents->first()->user_id;
        $lead->save();
    }
}
```

#### 리드 중복 제거

### 4.3 티켓 시스템 (Ticket System)

**목적**: 상담 프로세스를 SLA 기준으로 관리

**SLA 기준**:
- **첫 응답**: 2시간 이내
- **해결 시간**: 24시간 이내
- **상태**: within_sla, approaching_sla (80% 경과), violated_sla

**SLA 모니터링 Job**:
```php
// app/Jobs/CheckSLAViolations.php
public function handle(): void
{
    $tickets = Ticket::where('status', '!=', 'closed')
        ->whereNull('first_response_at')
        ->get();

    foreach ($tickets as $ticket) {
        $elapsed = now()->diffInMinutes($ticket->created_at);

        if ($elapsed > 120) { // 2시간 초과
            $ticket->sla_status = 'violated_sla';
            // 관리자에게 알림 발송
            Notification::send($ticket->assignee, new SLAViolationNotification($ticket));
        } elseif ($elapsed > 96) { // 80% 경과 (1.6시간)
            $ticket->sla_status = 'approaching_sla';
        }

        $ticket->save();
    }
}
```

### 4.4 예약 관리 (Appointment Management)

**목적**: 예약 일정 관리 및 리마인더 자동화

**예약 리마인더**:
```php
// app/Jobs/SendAppointmentReminder.php
public function handle(): void
{
    $tomorrow = now()->addDay()->toDateString();

    $appointments = Appointment::where('scheduled_at', 'LIKE', "$tomorrow%")
        ->where('status', 'scheduled')
        ->with('lead')
        ->get();

    foreach ($appointments as $appointment) {
        // SMS 또는 카카오톡 발송
        SMS::send($appointment->lead->primary_phone,
            "내일 {$appointment->scheduled_at} 예약이 있습니다. 확인 부탁드립니다."
        );
    }
}
```

**재예약 제안 로직**:
```php
// 시술 후 3-6개월 후 재예약 제안
public function suggestRebooking(Appointment $appointment): void
{
    if ($appointment->status === 'completed') {
        $suggestAt = $appointment->scheduled_at->addMonths(3);
        $appointment->rebooking_suggested_at = $suggestAt;
        $appointment->save();

        // 스케줄링된 알림 생성
        RebookingReminderJob::dispatch($appointment)
            ->delay($suggestAt);
    }
}
```

## 7. 마케팅 애널리틱스

### 7.1 주요 지표 (KPI)

#### 기본 지표
- **노출수 (Impressions)**: 광고가 표시된 횟수
- **클릭수 (Clicks)**: 광고를 클릭한 횟수
- **비용 (Cost)**: 광고비 지출 금액
- **문의수 (Leads)**: 문의 양식 제출 수
- **상담수 (Tickets)**: 상담 완료된 리드 수
- **예약수 (Appointments)**: 예약 완료 수
- **계약수 (Contracts)**: 계약 체결 수
- **매출 (Revenue)**: 실제 매출 금액

#### 계산 지표
- **CTR (Click-Through Rate)**: `(클릭수 / 노출수) × 100`
- **CVR (Conversion Rate)**: `(문의수 / 클릭수) × 100`
- **CPC (Cost Per Click)**: `비용 / 클릭수`
- **CPL (Cost Per Lead)**: `비용 / 문의수`
- **CPA (Cost Per Acquisition)**: `비용 / 예약수`
- **ROAS (Return On Ad Spend)**: `(매출 / 비용) × 100`
- **LTV (Lifetime Value)**: 고객 생애 가치 (향후 구현)

### 7.2 퍼널 전환율

**전환 단계**:
```
100% - 노출 (Impressions)
  ↓ CTR (클릭률)
 5% - 클릭 (Clicks)
  ↓ CVR (전환률)
 6% - 문의 (Leads)
  ↓ 상담 전환률
80% - 상담 (Tickets)
  ↓ 예약 전환률
67% - 예약 (Appointments)
  ↓ 계약 전환률
75% - 계약 (Contracts)
```

**전체 전환율**:
- `전체 전환율 = (계약수 / 노출수) × 100`
- 목표: 0.12% 이상

### 7.3 채널별 성과 비교

**주요 채널**:
1. **Google Ads** (구글 광고)
   - 평균 CTR: 3-5%
   - 평균 CVR: 5-8%
   - 평균 CPC: ₩1,500-2,500

2. **Naver Ads** (네이버 광고)
   - 평균 CTR: 4-6%
   - 평균 CVR: 6-10%
   - 평균 CPC: ₩1,000-2,000

3. **Facebook Ads** (페이스북 광고)
   - 평균 CTR: 2-4%
   - 평균 CVR: 3-6%
   - 평균 CPC: ₩800-1,500

4. **Instagram Ads** (인스타그램 광고)
   - 평균 CTR: 1.5-3%
   - 평균 CVR: 2-5%
   - 평균 CPC: ₩900-1,600

**ROI 계산**:
```
ROI = ((매출 - 비용) / 비용) × 100

예시:
비용: ₩5,000,000
매출: ₩20,000,000
ROI = ((20,000,000 - 5,000,000) / 5,000,000) × 100 = 300%
```

### 8.1 퍼널 대시보드 개선

#### 문제점
- 채널 피벗 페이지와 데이터 불일치
- localStorage 수동 캠페인 데이터 미반영
- API 엔드포인트 불일치로 404 에러 발생

#### 해결 방안
**파일**: `/m-crm-project/src/app/dashboards/funnel/page.tsx`

1. **API 통합**:
```typescript
// 채널피벗 API 호출
const response = await api.get('/api/dashboards/channel-pivot', {
  params: {
    startDate: calculatedStartDate,
    endDate: calculatedEndDate,
  }
});

const { channelPerformance, pivotTable } = response.data;
```

2. **localStorage 수동 캠페인 통합**:
```typescript
// localStorage에서 수동 캠페인 데이터 가져오기
const storedCampaigns = localStorage.getItem('mcrm_manual_campaigns');
let manualCampaigns: any[] = [];
if (storedCampaigns) {
  manualCampaigns = JSON.parse(storedCampaigns).filter((c: any) => !c.deleted);
}

// pivotTable과 수동 캠페인 합치기
const allCampaigns = [...pivotTable, ...manualCampaigns];
```

3. **기간 필터링**:
```typescript
// 기간 필터링
const filteredCampaigns = allCampaigns.filter((campaign: any) => {
  if (!campaign.startDate) return true;
  const campaignStart = new Date(campaign.startDate);
  const campaignEnd = campaign.endDate ? new Date(campaign.endDate) : campaignStart;
  const filterStart = new Date(calculatedStartDate);
  const filterEnd = new Date(calculatedEndDate);
  return campaignStart <= filterEnd && campaignEnd >= filterStart;
});
```

4. **채널별 집계**:
```typescript
// 채널별 집계
const channelMap = new Map();
filteredCampaigns.forEach((campaign: any) => {
  const channel = campaign.channel;
  if (!channelMap.has(channel)) {
    channelMap.set(channel, {
      channel,
      impressions: 0,
      clicks: 0,
      leads: 0,
      tickets: 0,
      appointments: 0,
      cost: 0,
      revenue: 0,
    });
  }
  const ch = channelMap.get(channel);
  ch.impressions += campaign.impressions || 0;
  ch.clicks += campaign.clicks || 0;
  ch.leads += campaign.leads || 0;
  ch.tickets += campaign.tickets || 0;
  ch.appointments += campaign.appointments || 0;
  ch.cost += campaign.cost || 0;
  ch.revenue += campaign.revenue || 0;
});

const aggregatedChannels = Array.from(channelMap.values());
```

**효과**:
- 채널 피벗과 퍼널 페이지 간 데이터 일관성 확보
- 수동 입력 캠페인 데이터 반영
- 정확한 기간별 필터링

**카운팅 로직**:
```php
// 누적 상태 기반 카운팅
// 문의중: leads (전환수) - 모든 리드
$totalLeads = $leadsByChannel->count();

// 상담완료 이상: tickets_count (상담수)
$totalTickets = $leadsByChannel->filter(function($lead) {
    return in_array($lead->status, [
        '상담완료', 'contacted',
        '미팅완료', '예약완료', 'converted',
        '계약완료', 'closed'
    ]);
})->count();

// 미팅완료 이상: appointments_count (예약수)
$totalAppointments = $leadsByChannel->filter(function($lead) {
    return in_array($lead->status, [
        '미팅완료', '예약완료', 'converted',
        '계약완료', 'closed'
    ]);
})->count();

// 계약완료: contracts_count (계약수)
$totalContracts = $leadsByChannel->filter(function($lead) {
    return in_array($lead->status, ['계약완료', 'closed']);
})->count();
```

**누적 카운팅 예시**:
```
리드 생성 (문의중)
  → 전환수 +1

상담 완료 (상담완료로 변경)
  → 전환수 +1, 상담수 +1

예약 완료 (미팅완료로 변경)
  → 전환수 +1, 상담수 +1, 예약수 +1

계약 체결 (계약완료로 변경)
  → 전환수 +1, 상담수 +1, 예약수 +1, 계약수 +1
```

**보류 상태 처리**:
- "보류" 상태는 추가 카운팅에서 제외
- 기존 카운팅은 유지 (예: 상담완료 → 보류 시 상담수는 유지)

**3. 마케팅 퍼널 정렬**

**기존 퍼널** (3단계):
```
문의 → 예약 → 계약
```

**개선된 퍼널** (6단계):
```
노출 (Impressions)
  ↓ CTR
클릭 (Clicks)
  ↓ CVR
전환/문의 (Leads - 문의중 이상)
  ↓ 상담 전환률
상담 (Tickets - 상담완료 이상)
  ↓ 예약 전환률
예약 (Appointments - 미팅완료 이상)
  ↓ 계약 전환률
계약 (Contracts - 계약완료)
```

**전환율 계산**:

```typescript
const formatDate = (dateString: string) => {
  if (!dateString) return 'N/A';
  const date = new Date(dateString);
  const year = String(date.getFullYear()).slice(-2); // 마지막 2자리
  const month = String(date.getMonth() + 1).padStart(2, '0');
  const day = String(date.getDate()).padStart(2, '0');
  return `${year}/${month}/${day}`;
};
```

### 9.2 감사 로그 (Audit Trail)

**Auditable Trait**:
```php
// app/Traits/Auditable.php
trait Auditable
{
    protected static function bootAuditable()
    {
        static::created(function ($model) {
            AuditLog::create([
                'user_id' => auth()->id(),
                'entity_type' => get_class($model),
                'entity_id' => $model->getKey(),
                'action' => 'create',
                'new_values' => $model->toArray(),
                'ip_address' => request()->ip(),
                'user_agent' => request()->userAgent(),
            ]);
        });

        static::updated(function ($model) {
            AuditLog::create([
                'user_id' => auth()->id(),
                'entity_type' => get_class($model),
                'entity_id' => $model->getKey(),
                'action' => 'update',
                'old_values' => $model->getOriginal(),
                'new_values' => $model->getChanges(),
                'ip_address' => request()->ip(),
                'user_agent' => request()->userAgent(),
            ]);
        });

        static::deleted(function ($model) {
            AuditLog::create([
                'user_id' => auth()->id(),
                'entity_type' => get_class($model),
                'entity_id' => $model->getKey(),
                'action' => 'delete',
                'old_values' => $model->toArray(),
                'ip_address' => request()->ip(),
                'user_agent' => request()->userAgent(),
            ]);
        });
    }
}
```

### 10.6 스케줄 작업

**Laravel Scheduler** (`app/Console/Kernel.php`):
```php
protected function schedule(Schedule $schedule)
{
    // SLA 위반 체크 (5분마다)
    $schedule->job(new CheckSLAViolations)->everyFiveMinutes();

    // 예약 리마인더 (매일 오전 9시)
    $schedule->job(new SendAppointmentReminder)->dailyAt('09:00');

    // 재예약 제안 (매주 월요일 오전 10시)
    $schedule->job(new SendRebookingSuggestion)->weeklyOn(1, '10:00');

    // 일일 리포트 (매일 오후 6시)
    $schedule->job(new GenerateDailyReport)->dailyAt('18:00');
}
```

**Cron 설정**:
```cron
* * * * * cd /var/www/mcrm-backend && php artisan schedule:run >> /dev/null 2>&1
```
