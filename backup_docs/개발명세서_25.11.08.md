7-1. 마케팅 퍼널 구조(업데이트)

핵심 목표:
광고 캠페인별 유입 → 전환 → 예약/내원/매출 흐름을 실시간으로 추적하고, ROI 기반 마케팅 퍼널을 시각화한다.

단계	설명	주요 데이터 소스
1. 광고 노출(Impressions)	플랫폼(네이버, 구글, 메타 등)에서 노출된 횟수	cost_imports.impressions
2. 클릭(Clicks)	광고 클릭 및 랜딩페이지 진입 수	cost_imports.clicks, visits
3. 전환(Leads)	페이지 유입 후 문의글 작성	leads, communications
4. 상담(Consulting)	전화상담	tickets
5. 예약(Booking)	방문일정 / 미팅일정 확정	appointments
6. 내원·결제(Revenue)	미팅/계약 : 수납 연동 수익	visits_clinic, cost_imports

💡 병의원 CRM의 “진료/시술” 대신, 마케팅 ROI 측정이 가능한 ‘비용→성과→수익’ 흐름으로 재구성함.

핵심 계산 지표:
CTR = 클릭 / 노출 × 100
CVR = 전환 / 클릭 × 100
CPL = 광고비 / 전환
CPA = 광고비 / 예약(또는 내원)
ROAS = 매출 / 광고비 × 100
ROI = (매출 – 광고비) / 광고비 × 100

7-2. 채널 피벗 대시보드(업데이트)

핵심 목표:
각 광고 플랫폼의 비용·성과·수익 데이터를 통합하고, 주차·월별로 비교 가능한 마케팅 퍼널 기반 KPI 테이블을 제공한다.

구성 요소:
	1.	채널 구조
	•	네이버: 키워드, 플레이스, 파워컨텐츠
	•	구글: GDN, YouTube
	•	메타: SNS(인스타그램, 페이스북)
	•	오프라인: 오프라인 광고, 간판 등
	2.	피벗 테이블 기본 지표
	•	노출수 (Impressions)
	•	클릭수 (Clicks)
	•	클릭률 (CTR %)
	•	전환수 (Leads)
	•	광고비 (Cost)
	•	CPL / CPA
	•	매출 (Revenue)
	•	ROI / ROAS
	3.	시간단위 선택
	•	주차(week) / 월(month)
	•	증감률(%) 자동 계산 및 전기 대비 변화율 표시
	4.	입력 방식
	•	API 연동 또는 CSV 업로드 외에도 직접 리스트 수정으로 입력 가능하도록 UI 제공
	•	사용자 직접 입력 시 DB에 즉시 반영되어 자동 계산 로직과 연동됨
	5.	집계 구조
	•	cost_imports (API 기반) ← 네이버, 구글, 메타
	•	leads / appointments / visits_clinic 조인
	•	period_label 기준(예: 2025-W43)으로 Pivot Grouping
	6.	시각화
	•	상단 KPI 카드: 총 광고비 / 전환수 / CPL / ROI
	•	중앙 피벗 테이블: 플랫폼별·캠페인별 퍼널 성과
	•	하단 트렌드 그래프: 주차별 전환율·ROAS 추이
	7.	기술적 특징
	•	각 API 연동 실패 시 수동 CSV 업로드 또는 수동 입력으로 대체
	•	모든 지표는 “기간 단위(period_label)”별 upsert 저장
	•	프론트는 WeeklyMarketingTable.vue를 기반으로 렌더링
	•	모든 사용자 입력값은 DB에 저장되어 버전 업데이트 시에도 유지되며, 기능 수정 시 데이터는 변경되지 않음
	•	사용자 관리 데이터 또한 DB에 저장되어 시스템 업데이트로 인해 삭제되지 않음
	•	입력 파일은 주기적으로 백업되며, 휴지통을 통한 복원 기능을 제공함 (삭제 시 휴지통 이동 → 복원 시 연동 기능 재활성화)

7-3. API 연동 규칙 정의

1. API 플랫폼별 기본 규칙

플랫폼	주요 엔드포인트	집계 단위	주요 필드	비고
Naver	/stats	일/주	impCnt, clkCnt, cost	일부 리포트는 7일 제한
Meta	/insights	일/주	impressions, clicks, spend, actions	breakdowns 파라미터로 디바이스, 연령 등 지원
Google	Ads Reports API	일/주	impressions, clicks, cost_micros	currency 변환 필요

2. 차원(Dimension) 구조 및 매핑
	•	period_label (YYYY-Wxx, YYYY-MM)
	•	platform, campaign_id, ad_id
	•	device, placement, region, gender, age (옵션)

3. 데이터 매핑 및 upsert 원칙
	•	광고 데이터는 raw_metrics 테이블에 저장 (platform, campaign_code, period_label)
	•	upsert 정책: 동일 period_label, platform, campaign_code 존재 시 update, 없으면 insert
	•	원본 데이터(raw)는 별도 백업 테이블에 저장 (versioned_raw_metrics)

4. 오류 및 누락 처리
	•	API 실패 시 로그 자동 기록 후 CSV/수동 입력 대체 루틴 수행
	•	누락 데이터는 period_label 기준으로 추후 재수집 및 overwrite

5. 보안 및 백업 정책
	•	일일 백업 자동 생성 (S3 or Local Storage)
	•	백업 주기: 1일/1주
	•	삭제된 파일은 휴지통으로 이동 후 30일 보관, 이후 자동 영구 삭제

7-4. KPI 자동 계산 및 리포트 생성 로직

1. KPI 산출 공식

CTR = (clicks / impressions) * 100
CVR = (leads / clicks) * 100
CPL = (cost / leads)
CPA = (cost / appointments)
ROAS = (revenue / cost) * 100
ROI = ((revenue - cost) / cost) * 100

2. 리포트 생성 프로세스
	1.	API 수집 → raw_metrics upsert
	2.	KPI 계산 스크립트 실행 (Node or Laravel Scheduler)
	3.	결과를 kpi_reports 테이블에 저장
	4.	프론트엔드에서 KPI 카드 및 그래프로 시각화
	5.	PDF 및 CSV 내보내기 기능 포함

3. 백업 및 버전 관리
	•	모든 리포트는 version_id 부여
	•	리포트 변경 시 diff 로그 기록
	•	백업 스냅샷은 S3/local dual 저장
