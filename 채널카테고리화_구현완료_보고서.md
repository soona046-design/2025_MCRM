## 🔑 주요 기능

### 1. 자동 카테고리 분류
- **데이터베이스 기반 매핑**: 관리자가 추가한 채널-카테고리 매핑 우선 적용
- **폴백 규칙 기반**: 매핑이 없을 경우 내장 규칙으로 자동 분류
- **Real-time 할당**: 새로운 방문 기록이 생성될 때마다 자동 카테고리 할당

### 2. 동적 채널 관리
- **추가**: 새로운 UTM source와 카테고리 매핑 추가
- **수정**: 기존 채널의 카테고리 변경
- **삭제**: 더 이상 사용하지 않는 채널 제거
- **활성화/비활성화**: 채널의 상태 제어

### 3. 카테고리별 성과 분석
- **실시간 집계**: 카테고리별 문의, 상담, 예약, 계약 통계
- **ROI 분석**: 온라인/오프라인/DB별 투자 수익률 비교
- **시각화**: 막대 차트, 파이 차트로 직관적 표현
- **상세 지표**: 비용, 매출, 전환율, CPC, CPA 등

### 4. 확장 가능한 아키텍처
- **모듈화된 Helper**: ChannelCategoryHelper로 분류 로직 중앙화
- **데이터베이스 기반 설정**: 코드 수정 없이 채널 관리 가능
- **마이그레이션 지원**: 기존 데이터 자동 변환

- **Database**: SQLite (개발), MySQL (프로덕션)
- **Pattern**: Service → Helper → Model 아키텍처

### 프론트엔드
- **Language**: TypeScript
- **Framework**: Next.js 14
- **UI**: Material-UI (MUI)
- **State**: React Hooks + API 통신

### 데이터베이스
```sql
-- 채널 카테고리 테이블
CREATE TABLE channel_categories (
  id INT PRIMARY KEY,
  code VARCHAR(20) UNIQUE,           -- 'online', 'offline', 'db'
  name VARCHAR(50),                  -- '온라인', '오프라인', 'DB'
  color VARCHAR(7),                  -- 시각화용 색상
  sort_order INT,                    -- 정렬 순서
  active BOOLEAN DEFAULT true,
  timestamps
);

-- 채널-카테고리 매핑 테이블
CREATE TABLE channel_category_mappings (
  id INT PRIMARY KEY,
  utm_source VARCHAR(100),           -- 'naver', 'google', 등
  display_name VARCHAR(100),         -- '네이버 키워드', 등
  category_id INT (FK),              -- channel_categories.id
  priority INT,                      -- 매칭 우선순위
  active BOOLEAN DEFAULT true,
  timestamps
);

-- visits 테이블 (수정됨)
ALTER TABLE visits ADD COLUMN channel_category ENUM('online', 'offline', 'db');
CREATE INDEX idx_channel_category ON visits(channel_category);
```

## 📊 API 엔드포인트

### 채널 관리 API
```
GET    /api/channel-management/categories          # 모든 카테고리 조회
GET    /api/channel-management/mappings            # 채널 매핑 조회 (필터 가능)
POST   /api/channel-management/mappings            # 채널 추가
PUT    /api/channel-management/mappings/{id}       # 채널 수정
DELETE /api/channel-management/mappings/{id}       # 채널 삭제
PATCH  /api/channel-management/mappings/{id}/toggle # 활성화/비활성화
```

### 방문 API
```
GET    /api/visits                                 # 방문 목록 (필터 가능)
GET    /api/visits/{id}                            # 방문 상세 조회
POST   /api/visits                                 # 방문 생성 (자동 카테고리 할당)
PUT    /api/visits/{id}                            # 방문 수정
DELETE /api/visits/{id}                            # 방문 삭제
GET    /api/visits/statistics/category             # 카테고리별 통계
```

### 대시보드 API
```
GET    /api/dashboards/channel-pivot               # 채널 피벗 분석
  응답:
  {
    "channelPerformance": [...],          # 채널별 성과
    "categoryPerformance": [...],          # ✨ NEW: 카테고리별 성과
    "pivotTable": [...]                   # 캠페인별 세부 데이터
  }
```

## 🎯 사용 시나리오

### 시나리오 1: 새로운 채널 추가
1. 관리자가 `/settings/channels` 페이지 방문
2. "채널 추가" 버튼 클릭
3. UTM Source: "baidu", Display Name: "바이두", Category: "온라인" 입력
4. 저장 버튼 클릭
5. 이후 모든 utm_source="baidu" 방문은 자동으로 온라인 카테고리로 분류됨

### 시나리오 2: 채널 카테고리 변경
1. 채널 목록에서 "Google Ads" 채널 선택
2. 편집 아이콘 클릭
3. 카테고리를 "온라인"에서 "DB"로 변경
4. 저장
5. 새로운 Google Ads 방문은 DB 카테고리로 분류됨 (기존 데이터는 변경 안 됨)

### 시나리오 3: 성과 분석
1. 채널별 ROI 분석 대시보드 접속 (`/dashboards/channel-pivot`)
2. 카테고리별 성과 카드 확인:
   - 온라인: 문의 1200건, ROI 25%
   - 오프라인: 문의 300건, ROI 15%
   - DB: 문의 50건, ROI 8%
3. 온라인 채널에 예산 집중 결정
