# **Ⅱ. 개발 명세서 (실행 중심)**

## **0\. 목표/범위**

* **목표**: 채널별 **신환 유입 실시간 집계**, 상담-예약-내원-시술 **엔드투엔드 추적**, **ROI/전환 퍼널** 시각화.

* **MVP 범위(8\~10주)**

  1. 유입 수집(UTM+서버측 이벤트)

  2. 리드/고객/예약/상담 히스토리

  3. 대시보드(채널/캠페인/소재별 전환)

  4. 알림/업무할당/권한

  5. 외부광고 API 1\~2종 연동(예: 네이버, 메타)

  6. 개인정보·접근통제·감사 로그

## **1\. 사용자 역할/권한**

* **슈퍼관리자(본부)**: 전 지점/모든 데이터·설정

* **지점관리자**: 소속 지점 데이터·사용자 관리

* **상담매니저**: 리드/상담/예약, 메모/태그 편집

* **마케터**: 캠페인/소재 코드, 리포트 접근

* **의사/시술스태프**: 일정·진료결과 요약 열람(민감정보 최소화)

* **외주/분석가(옵션)**: 익명화 보고서 접근

## **2\. 핵심 기능 요구사항**

### **A. 유입·리드 관리**

* **save\_visitor.php**: 서버측에서 UTM, gclid/nclid, referrer, client\_id(퍼스트파티 쿠키), ip, ua, landing\_path, session\_id 저장.

* **중복 리드 병합**: 전화/카톡/폼 등 **다채널 식별자 매칭**(전화번호, 해시 이메일, 카카오톡 ID 토큰 등).

* **리드 스코어링(규칙 기반)**: 채널·페이지 체류·버튼 클릭 수·업무시간 외 문의 등 가중치.

### **B. 상담·작업흐름**

* **상담 티켓**: 상태(신규/진행/보류/완료), 우선순위, 담당자, 태그(증상/진료과), 템플릿 스크립트(예: 임플란트/교정/프롤로).

* **커뮤니케이션 연동**: 클릭투콜·알림톡(친구톡/알림톡 템플릿)·SMS·카카오채널·웹챗 기록 자동 첨부.

* **SLA/알림**: X분 내 미응답 시 관리자/대체담당자 푸시·슬랙/팀스 웹훅.

### **C. 예약·내원 연결**

* **예약 엔진**: 일정 슬롯, 리마인더(알림톡/SMS), 노쇼 표기, 재예약 제안.

* **EMR 연동(옵션)**: 방문 완료/처치 코드/수납 금액 수신 → **전환·수익 연결**.

### **D. 분석 대시보드**

* **퍼널**: 방문→리드→상담→예약→내원→시술→수납

* **채널/캠페인/소재별** 전환·CPL/CPA·ROAS(연동 시)

* **시간대/요일/지점** 분포, 상담원별 응답속도/전환율

* **코호트**: 유입 주차별 재내원/문의 재발생률

### **E. 마케팅 채널 연동**

* **네이버/구글/메타 API(순차)**: 캠페인 구조 동기화, 비용·클릭·노출 수집

* **웹훅 수신**: 외부 랜딩툴(타사 폼/챗봇) → 표준 Lead Webhook

### **F. 자동화/룰**

* **예시 룰**:

  * 고가치 채널+긴 체류 → **우선 배정 \+ 즉시콜**

  * 예약 D-1 리마인드·D+2 후기요청·D+30 리콜

  * 노쇼 2회 → **상담 스크립트 전환(장벽 낮추는 제안)**

### **G. 보안·감사·컴플라이언스**

* **PIPA(개인정보보호법), 의료법 광고규정** 준수

* **RBAC**, **필드 레벨 마스킹(전화번호 뒷자리 별표)**

* **접속·조회·다운로드 감사 로그**, **IP 제어/2FA**, **전송구간 TLS**, **DB 암호화(휴면/민감 필드)**

* **백업**: 일일/주간 스냅샷, 포인트인타임 리커버리(옵션)

## **3\. 비기능 요구사항**

* **성능**: API p95 \< 300ms(조회), 배치 동기화 15분 주기, 대시보드 2초 이내 초기 렌더

* **가용성**: 99.5%+, 장애시 읽기 전용 모드

* **확장성**: 멀티테넌시(지점 분리), 메시지 큐 기반 비동기 처리

* **로그/모니터링**: APM, 구조화 로그(JSON), 알람(에러율, 지연, 큐 적체)

## **4\. 기술 스택 제안(현재 환경 고려)**

* **백엔드**: PHP 8.2 (Laravel 권장) / MariaDB 10.x

* **프론트**: Vue 3 또는 React(관리자 콘솔), Tailwind UI

* **수집/큐**: Redis \+ Laravel Queue,(옵션) Kafka(향후)

* **배포**: Docker, Nginx, Zero-downtime 배포

* **인증**: JWT \+ Refresh, 지점별 SSO(옵션)

## **5\. 데이터 모델(요약 ERD)**

* **visits**(visit\_id, client\_id, session\_id, utm\_source/medium/campaign/content/term, referrer, landing\_path, first\_seen\_at, ip, ua)

* **lead\_channels**(lead\_id, channel\_code, campaign\_code, adgroup\_code, creative\_code, cost\_attrib\*)

* **tickets**(ticket\_id, lead\_id, assignee\_id, state, priority, tags\[\], notes, last\_contact\_at)

* **communications**(comm\_id, ticket\_id, type\[sms/kakao/call/chat\], direction, content, meta, at)

* **appointments**(apt\_id, lead\_id, clinic\_id, doctor\_id, slot\_at, status\[noshow/done\], reminder\_sent)

* **visits\_clinic**(clinic\_visit\_id, apt\_id, emr\_visit\_no, procedure\_code, charge\_amount, paid\_at)

* **users**(user\_id, role, clinic\_id, name, phone, 2fa\_secret, active)

* **cost\_imports**(platform, campaign\_code, date, impressions, clicks, cost)

(\* 민감 필드는 암호화/마스킹 저장)

## **6\. API 설계(샘플)**

* POST /api/collect/visit

  * in: {clientId, sessionId, utm, referrer, path, ts}

  * out: {visitId}

* POST /api/leads

  * in: {name?, phone\*, email?, consent, sourceVisitId, memo}

  * out: {leadId, score}

* POST /api/tickets/{leadId} (생성) / PATCH /api/tickets/{ticketId} (상태·담당자)

* POST /api/appointments

  * in: {leadId, clinicId, doctorId?, slotAt} → 리마인드 예약

* POST /api/webhooks/ads/{platform} 비용/성과 수신

* GET /api/dashboards/funnel?range=2025-09\&clinic=busan

* GET /api/reports/roi?groupBy=campaign\&range=Q3-2025

# **Ⅳ. 확장 로드맵 & 천재 아이디어 10+ (3,000자+)**

여기서는 **실전 가치**와 **차별화**에 집중한 아이디어를 제안합니다. 실행 난이도/가치/리스크도 간단히 표기합니다.

1. **CDP형 고객 360° 프로필**

   * **내용**: 리드→상담→예약→내원→결제 이력뿐 아니라 **콘텐츠 소비(블로그/영상/랜딩)**, 설문, 만족도, 사후관리(홈케어 가이드 열람)까지 묶어 **단일 고객 카드**로 시각화.

   * **가치**: 상담 시 “무엇을 보고 왔는지” 맥락 기반 대화 → 전환율↑.

   * **난이도/리스크**: 중; 개인정보·쿠키 정책 준수 필요.

2. **예측 스코어링(ML Light)**

   * **내용**: 채널·시간대·페이지·문의 유형·과거 응대속도·담당자 성과를 특징으로 **예약 확률/내원 확률**과 **예상 매출 범위**를 예측. 상위 20% 리드를 즉시 콜백 큐로 승격.

   * **가치**: 상담 리소스 집중, CPA 절감.

   * **난이도**: 중상(데이터 축적 필요).

3. **광고 예산 자동재배분 시뮬레이터**

   * **내용**: “현재 주간 데이터로 다음 주 예산을 어떻게 바꾸면 예약이 X% 늘까?”를 예측하는 **샌드박스**. 지점별 포트폴리오 최적화(네이버:구글:메타 비율).

   * **가치**: 의사결정 속도 및 효율 향상.

   * **난이도**: 중.

4. **노쇼 방지 심리·언어 템플릿 A/B**

   * **내용**: 알림톡 문구를 **심리적 장벽** 낮추는 표현(예: “5분 전 도착 시 대기 없이 진행”) vs 정보형(“위치/주차/준비사항”)으로 A/B 테스트. 예약간격, 진료유형별 최적 문구 학습.

   * **가치**: 노쇼율 10\~30% 개선 가능.

   * **난이도**: 하.

5. **오프라인 유입 코드화(간판/배너/브로슈어 QR)**

   * **내용**: 오프라인 매체에 **단축 URL/QR \+ UTM**을 표준 발급. “로비 배너 A vs 엘리베이터 포스터 B” 효과 비교.

   * **가치**: 오프라인도 디지털처럼 측정 → 예산근거 확보.

   * **난이도**: 하.

6. **상담 스크립트 자동 추천(컨텍스트 인식)**

   * **내용**: 리드 카드 열 때 **증상 키워드/유입 콘텐츠**를 분석해 관련 **FAQ/리스크 설명/보험/소요시간**을 자동 제시.

   * **가치**: 신규 상담원 온보딩 가속, 상담 품질 표준화.

   * **난이도**: 중.

7. **의료광고법 준수 가드레일**

   * **내용**: 상담·메시지·랜딩 문구 작성 시 **금지 문구/비교표현/전후사진 규정** 자동 감지 경고.

   * **가치**: 리스크 감소, 내부 컴플라이언스 신뢰 상승.

   * **난이도**: 중.

8. **캠페인-콘텐츠 매핑 그래프(지식 그래프)**

   * **내용**: 캠페인↔키워드/소재↔랜딩 섹션↔상담 태그↔시술 코드가 **그래프**로 연결된 인터랙티브 맵. 병목(유입은 많은데 예약 전환 낮은 노드) 시각적 발견.

   * **가치**: 최적화 포인트 빠른 탐지.

   * **난이도**: 중.

9. **콜센터 하이브리드(콜봇+사람) 라우팅**

   * **내용**: 업무시간 외/대량 캠페인 시 **TTS/ASR 콜봇**이 1차 분류·예약 링크 전송, 고가치 리드는 아침 9시에 사람에게 우선 라우팅.

   * **가치**: 응답속도 개선, 인건비 효율화.

   * **난이도**: 중상.

10. **리콜·재방문 엔진(치료 주기 캘린더)**

    * **내용**: 시술/처치 주기에 맞춘 **개인화 리콜 스케줄** 자동 생성(예: 스케일링 6개월, 교정 내원 주기).

    * **가치**: LTV 증대, 유실 환자 회수.

    * **난이도**: 중.

11. **멀티브랜치 본부 관제**

    * **내용**: 지점별 KPI 신호등, 이슈 발생시 원클릭 딥다이브. “부산점 예약 급감(금주 \-28%)” → 채널·소재 수준까지 굴착.

    * **가치**: 본부 운영 통제력 강화.

    * **난이도**: 중.

12. **현장 DID/LED 연동 콘텐츠 오케스트레이션**

    * **내용**: CRM에서 프로모션 설정 시 **대기실 DID/LED** 콘텐츠 자동 교체(시술 프로모션/후기/주의사항). 캠페인과 현장 경험 연결.

    * **가치**: 일관된 브랜딩, 체감 가치 상승.

    * **난이도**: 중.

13. **프라이버시 퍼스트 리포팅**

    * **난이도**: 하.

    * **가치**: 정확한 성과평가.

    * **난이도**: 중.

    * **내용**: 요일/시간대별 수요예측으로 **슬롯 자동 개방/축소**, 과밀·공백 최소화.

    * **가치**: 자원 효율, 대기시간 감소.
