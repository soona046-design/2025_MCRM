## 1. 프로젝트 개요

### 1.1 목적
치과/의료 클리닉을 위한 종합 고객관계관리(CRM) 시스템.  
환자 여정 전체를 추적하고 마케팅 ROI를 최적화합니다.

### 1.2 환자 여정 (Patient Funnel)
```
방문자 (Visitor)     ← UTM 추적
    ↓
리드/문의 (Lead)     ← 스코어링 & 배정
    ↓
상담 (Ticket)        ← SLA 관리
    ↓
예약 (Appointment)   ← 리마인더
    ↓
내원/시술 (Treatment)
    ↓
매출/재방문 (Revenue/Rebooking)
```

### 1.3 배포 현황
| 항목 | URL |
|------|-----|
| 프론트엔드 (Vercel) | https://insight-mcrm.vercel.app |
| 백엔드 (Cafe24) | https://insightmcrm.mycafe24.com |
| 로컬 백엔드 | http://localhost:8000 |
| 로컬 프론트엔드 | http://localhost:3000 |

### 1.4 테스트 계정

## 5. 핵심 기능

### 5.1 리드 관리

#### 상태 흐름
```
new → contacted → converted → (closed)
new → pending
new → rejected
```
- `LeadController` → `Lead.php`의 `getStatusAttribute()`에서 영문→한글 표시 변환
- 한글 상태값 입력 시 `page.tsx`의 `statusMap`으로 영문 변환 후 API 전송

#### 리드 스코어링 (규칙 기반)
```php
$score = 50; // 기본
$score += channelScore($lead->utm_source);  // 최대 +20
$score += serviceScore($lead->memo);         // 임플란트/교정 등 +20
$score += timeScore(now());                  // 영업시간 내 +10
return min(100, max(0, $score));
```

#### utm_source 수정 시 Visit 자동 업데이트
```php
// LeadController::update()
if (array_key_exists('utm_source', $validatedData)) {
    if ($lead->sourceVisit) {
        $lead->sourceVisit->update(['utm_source' => $utmSource]);
    } else {
        $visit = Visit::create(['utm_source' => $utmSource, 'first_seen_at' => now()]);
        $validatedData['source_visit_id'] = $visit->visit_id;
    }
}
```

### 5.4 티켓 SLA 관리

**SLA 기준**: 첫 응답 2시간 이내 (120분)
- `within_sla`: 정상
- `approaching_sla`: 80% 경과 (96분)
- `violated_sla`: 120분 초과

```php
// CheckSLAViolations Job — 5분마다 실행
$elapsed = now()->diffInMinutes($ticket->created_at);
if ($elapsed > 120) $ticket->sla_status = 'violated_sla';
elseif ($elapsed > 96) $ticket->sla_status = 'approaching_sla';
```

### 5.5 예약 리마인더 자동화

```php
// SendAppointmentReminder Job — 매일 오전 9시
$tomorrow = now()->addDay()->toDateString();
Appointment::where('scheduled_at', 'LIKE', "$tomorrow%")
    ->where('status', 'scheduled')
    ->get()
```

## 8. 마케팅 애널리틱스

### 8.1 주요 KPI 지표

| 지표 | 공식 |
|------|------|
| CTR | (clicks / impressions) × 100 |
| CVR | (leads / clicks) × 100 |
| CPC | cost / clicks |
| CPL | cost / leads |
| CPA | cost / appointments |
| ROAS | (revenue / cost) × 100 |
| ROI | ((revenue - cost) / cost) × 100 |

### 8.2 퍼널 전환 단계 (6단계)
```
노출 (Impressions)  ←— CTR
클릭 (Clicks)       ←— CVR
전환/문의 (Leads)   ←— 상담 전환율
상담 (Tickets)      ←— 예약 전환율
예약 (Appointments) ←— 계약 전환율
계약 (Contracts)
```

### 8.3 누적 카운팅 시스템 (리드 상태 기반)
```php
totalLeads:        모든 리드
totalTickets:      contacted/converted/closed 상태인 리드
totalAppointments: converted/closed 상태인 리드
totalContracts:    closed 상태인 리드
```

### 8.4 채널별 성과 벤치마크

| 채널 | 평균 CTR | 평균 CVR | 평균 CPC |
|------|---------|---------|---------|
| Naver Ads | 4-6% | 6-10% | ₩1,000-2,000 |
| Google Ads | 3-5% | 5-8% | ₩1,500-2,500 |
| Facebook Ads | 2-4% | 3-6% | ₩800-1,500 |
| Instagram Ads | 1.5-3% | 2-5% | ₩900-1,600 |

### 11.2 Cafe24 배포 (SSH 불가 환경)

**⚠️ 중요**: Cafe24는 SSH 접속 불가 → 마이그레이션은 PHP 웹 스크립트 방식으로 수행

**절차**:
1. PHP 스크립트 작성 (Laravel bootstrap + 직접 SQL)
2. FTP로 `/insightmcrm/www/` 업로드
3. 브라우저로 `http://insightmcrm.mycafe24.com/{script}.php` 실행
4. **즉시 파일 삭제** (보안)
5. migrations 테이블에 수동으로 레코드 추가

**주의사항**: `php artisan migrate` 실행 시 외래 키 중복 오류 (errno: 121) 발생 가능  
→ 신규 테이블만 직접 생성하는 방식 채택 (Bug #8 참조)
