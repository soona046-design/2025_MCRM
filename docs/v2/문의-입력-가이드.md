# M-CRM v2 — 신규 문의 입력 가이드 (채널별 ROAS를 살리는 법)

> 대상: 상담·문의를 받는 실무자. 문의를 **CRM 수기 입력 화면**으로 등록할 때
> 광고 귀속 증거(UTM·클릭ID)를 남기는 방법을 정리합니다.

---

## 1. 왜 채널별 ROAS가 지금 0인가

ROAS는 두 층위가 있습니다:

| 지표 | 계산 | 현재 상태 |
|---|---|---|
| **전사 ROAS** (대시보드 카드·추이) | 신규 유입 고객 매출 전체 ÷ 광고비 | ✅ 실수치 (예: 6월 21.8배) |
| **채널·캠페인별 ROAS** (채널별/캠페인별 상세) | **확정 귀속** 매출 ÷ 해당 채널 광고비 | ❌ 0 |

채널별이 0인 이유는 사슬이 세 곳에서 끊겨 있기 때문입니다:

1. **과거 문의에 증거가 없음** — 이관한 121건의 유입경로는 "블로그/검색/홈페이지" 같은 텍스트뿐입니다. 명세 규칙상 UTM·클릭ID 같은 **강한 증거 없이는 광고 유입으로 확정하지 않습니다**(자연검색과 구분 불가 — 추정 귀속은 ROAS를 왜곡). 그래서 매출 ₩78.2M이 신규 유입으로 잡혀도 "어느 광고 덕인지"는 모르는 상태입니다.
2. **광고 랜딩 URL에 UTM이 안 붙어 있음(추정)** — 고객이 파워링크를 눌러 들어와도 URL에 표식이 없으면 증거가 생길 수 없습니다. **이게 근본 원인이고, 광고 소재 URL 태깅이 선행 작업입니다** (아래 §4).
3. **귀속 엔진 미구현** — 증거가 쌓인 계약 금액을 캠페인별 매출로 굴리는 백엔드 모듈이 마지막 남은 작업입니다. 증거가 쌓이기 시작하면 붙입니다.

## 2. 입력 화면 사용법 — `/customers/new` (고객 유입 관리 → 수기 입력)

원칙: **문의 발생 후 24시간 이내 입력** (입력률 KPI 90%).

**기본 정보**: 병원명* · 고객명(담당자) · 연락처* · **유입일** · 상담 채널*(전화/카카오톡/홈페이지 폼/기타) · 단계 · 담당자 · 메모

> 📅 **유입일**은 기본값이 오늘이지만 **실제 문의가 들어온 날짜로 바꿔 넣으세요** (미래 날짜 불가). 며칠 지나 몰아서 입력하는 경우 특히 중요합니다 — 입력률·퍼널·기간별 집계가 이 유입일 기준으로 잡힙니다. 잘못 넣었으면 나중에 **고객 상세 › 기본 정보 편집**에서 고칠 수 있고(변경 이력 남음), **파일 업로드**로 넣을 땐 CSV의 **발생일** 컬럼이 그대로 유입일이 됩니다.

**유입 경로* 선택** — 여기가 갈림길입니다:

| 선택 | 언제 | 결과 |
|---|---|---|
| **온라인 광고** | 광고를 보고 온 것이 확인될 때 | ↓ §3 증거 입력으로 진행 |
| 온라인 비광고 | 블로그·자연검색·홈페이지 직접 방문 | 광고비와 매칭 안 됨 (정상) |
| 오프라인 광고 / 지인 소개 / 기타 | 박람회(BDEX)·이벤트 / 소개 / 그 외 | 〃 |

## 3. "온라인 광고" 선택 시 — 증거 남기는 3가지 방법

셋 중 **하나만** 있으면 "확정 귀속" 배지가 붙고 채널 ROAS 분자에 들어갑니다.

**방법 A — 채널·캠페인 드롭다운** (전화 문의에 적합):
광고 채널(네이버/메타/구글) 선택 → 캠페인 드롭다운이 **광고 계정의 실제 캠페인 목록**으로 자동 연동됩니다(네이버는 실 API — 인사이트/파워컨텐츠#1/플레이스). 목록에서 선택하면 그 자체가 강한 증거(캠페인 ID)입니다. 캠페인을 특정 못 하면 "목록에 없음 — 직접 입력"에 `캠페인 미상` → 온라인 광고-미상으로 분리 집계됩니다.

**방법 B — URL 붙여넣기 (권장, 가장 쉬움)**:
고객이 들어온 랜딩페이지 주소를 통째로 붙여넣으면 `utm_source/medium/campaign`, `gclid`(구글)/`nclk`(네이버) 클릭ID를 **자동 파싱**해 채널·캠페인이 채워지고 확정 귀속 배지가 켜집니다.

```
예) https://insight.co.kr/landing?utm_source=naver&utm_medium=cpc&utm_campaign=파워링크_임플란트
    → 채널: 네이버 · 캠페인: 파워링크_임플란트 · 확정 귀속 ✓
```

**방법 C — UTM/클릭ID 직접 입력**: source/medium/campaign 또는 클릭ID를 아는 만큼 수동 입력.

**증거가 하나도 없으면** → "소스 미상(온라인 추정)"으로 저장됩니다. 온라인 비중 파악에는 쓰이지만 광고비와 매칭되지 않습니다(ROAS·CPA 제외). 이건 시스템이 데이터를 왜곡하지 않겠다는 규칙이니, 억지로 채널을 찍는 것보다 미상이 맞습니다.

## 4. 선행 작업 — 광고 랜딩 URL에 UTM 태깅 (마케터 1회 작업)

문의 접수자가 아무리 잘 입력해도 **광고 URL 자체에 표식이 없으면 증거를 얻을 수 없습니다.** 각 광고 소재의 랜딩 URL 뒤에 다음 규칙으로 파라미터를 붙여 주세요:

| 채널·상품 | 붙일 파라미터 (예시) |
|---|---|
| 네이버 파워링크 | `?utm_source=naver&utm_medium=cpc&utm_campaign=파워링크` |
| 네이버 플레이스 | `?utm_source=naver&utm_medium=place&utm_campaign=플레이스` |
| 네이버 파워컨텐츠 | `?utm_source=naver&utm_medium=content&utm_campaign=파워컨텐츠` |
| 구글 GDN | `?utm_source=google&utm_medium=display&utm_campaign=GDN` (+`gclid` 자동) |
| 유튜브 | `?utm_source=google&utm_medium=video&utm_campaign=유튜브` |
| 메타 | `?utm_source=meta&utm_medium=social&utm_campaign=메타` (리드폼 DB는 그 자체가 증거) |

- `utm_campaign` 값은 CRM 캠페인 목록과 같은 이름을 쓰면 드롭다운과 바로 매칭됩니다.
- 홈페이지 문의 폼에는 **현재 페이지 URL(UTM 포함)을 숨은 필드로 자동 수집**하게 해두면, 접수자가 그 값을 URL 붙여넣기 칸에 복사하는 것으로 끝납니다.

## 5. 실무 시나리오 요약

| 문의 유형 | 하는 일 |
|---|---|
| 홈페이지 폼/DB | 폼에 찍힌 유입 URL 복사 → **방법 B** 붙여넣기 |
| 전화 | "어떻게 보고 연락 주셨어요?" → 광고 확인되면 **방법 A** 드롭다운, 애매하면 온라인 비광고/미상 |
| 카카오톡/네이버톡톡 | 채널 유입 경로가 광고면 방법 A, 아니면 비광고 |
| 메타 리드폼(DB) | 온라인 광고+메타 — DB 자체가 전환 이벤트 증거 |
| 소개·박람회 | 지인 소개 / 오프라인 광고 (증거 불필요) |

이렇게 쌓이면: 확정 귀속 문의 → 계약·매출 입력 → (귀속 엔진 가동 후) 채널별 상세의 "매출·확정 귀속"과 ROAS가 캠페인 단위로 살아납니다.
