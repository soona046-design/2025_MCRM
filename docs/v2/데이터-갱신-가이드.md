# M-CRM v2 — 월별 데이터 갱신 가이드

> 대상: 매월 새로 나오는 **클라이언트 매출 PDF**와 **온라인 광고성과 시트**를 CRM에 반영하는 절차.
> 모든 명령은 재실행해도 안전합니다(중복 생성 없음 — 2026-07-13 멱등성 리허설 검증 완료).

---

## A. 클라이언트 매출 자료 갱신 (예: 7월 자료가 나왔을 때)

**1. PDF 준비** — 매출 시트를 기존과 같은 방식(Chrome 인쇄 → PDF)으로 저장하고, 기존 폴더에 넣습니다.

- 파일명 규칙: **`26년7월.pdf`** (기존 1~6월과 동일한 형식이면 자동 인식). 연간 통합본(`2025_all_매출관리.pdf`처럼 페이지당 1개월)도 파일 경로를 직접 넘기면 처리됩니다.
- 폴더: `~/Library/CloudStorage/GoogleDrive-.../내 드라이브/최안과의원/images/`

**2. 파싱 + 매칭** — 폴더의 월 파일을 전부 자동 감지합니다:

```bash
cd mcrm-v2-backend
python3 scripts/parse_revenue_pdfs.py '/Users/soona/Library/CloudStorage/GoogleDrive-soona046@gmail.com/내 드라이브/최안과의원/images'
```

- 월마다 `✓ ₩금액`이 찍히면 **시트의 소계와 원 단위 일치** 검증 통과. 불일치하면 에러로 멈춤(시트 서식이 바뀐 경우 — 그때는 Claude에게 파일을 보여주세요).
- 결과: `data/original/매출-2026-정리.csv` 갱신. **새 클라이언트가 생겼다면** 이 CSV의 `matched_name` 열을 열어 신규문의와 제대로 매칭됐는지 한 번 훑어보세요(비어 있으면 기존 고객으로 들어갑니다).

**3. 적재**:

```bash
php artisan import:revenue
```

- 파일 출처 매출 계약을 전량 교체하므로 7월 추가분·과거 월 수정분이 함께 반영됩니다.
- **CRM 화면에서 직접 입력한 계약 금액은 건드리지 않습니다** (파일 출처 계약만 교체).
- 월 스냅숏(ROAS·ROI 추이)도 자동 재계산됩니다.

## B. 온라인 광고성과 갱신 (7월 주차 데이터가 채워졌을 때)

**1.** 광고성과 구글시트를 CSV로 내보내(파일 → 다운로드 → CSV) 교체:

```
data/original/광고성과-2026.csv   ← 새 파일로 덮어쓰기
```

**2.** 광고 지표만 갱신 (문의는 재적재하지 않음 — 중복 방지):

```bash
cd mcrm-v2-backend
php artisan import:real --ads-only
```

## C. 신규 문의는 파일이 아니라 CRM에서

신규 문의는 앞으로 **CRM 수기 입력 화면**(`/customers/new`) 또는 **파일 업로드**(`/customers/upload`)로 넣는 것을 권장합니다 — URL 붙여넣기로 UTM 귀속 증거가 쌓여야 채널별 ROAS가 살아납니다. **입력 요령·UTM 태깅 규칙은 [문의-입력-가이드.md](문의-입력-가이드.md) 참고.**

> ⚠️ `import:real --fresh`(전체 초기화)는 **초기 이관 1회용**입니다. CRM에서 입력한 고객·금액·목표가 전부 지워지므로 월별 갱신에는 절대 사용하지 마세요. (`migrate:fresh --seed`도 동일 — 데모 시드로 되돌아감)

## D. 갱신 후 확인 (1분)

1. 대시보드(`/`) — 해당 월 광고비·매출 반영 확인
2. 매출 관리 → 추이·기여도 분석 — ROAS·ROI 추이에 새 달 포인트 생성 확인
3. 계약 현황 — "미입력" 건이 새로 생겼으면 금액 입력

## E. VPS 배포 후에는

같은 명령을 컨테이너 안에서 실행합니다:

```bash
docker compose -f docker-compose.prod.yml exec app php artisan import:revenue
```

(파서는 로컬에서 돌려 CSV만 서버로 올리거나, CSV를 git 외 경로로 scp)

## 명령 요약

| 상황 | 명령 |
|---|---|
| 월 매출 PDF 추가/수정 | `parse_revenue_pdfs.py <폴더>` → `php artisan import:revenue` |
| 광고성과 시트 갱신 | CSV 교체 → `php artisan import:real --ads-only` |
| 신규 문의 | CRM 화면에서 입력 (파일 아님) |
| 전체 초기화 (비상시만) | `import:real --fresh` → `parse_revenue_pdfs.py` → `import:revenue` |
