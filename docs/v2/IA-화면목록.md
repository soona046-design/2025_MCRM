## 1. 내비게이션 구조 (사이드바)

```
📊 대시보드              ← 기본 진입 화면
👥 고객 유입 관리
   ├ 유입 고객 목록
   ├ 수기 입력
   ├ 파일 업로드
   └ 미입력/지연 현황
💰 매출 관리
   ├ 계약 매출 목록
   ├ 기존 고객 목록
   ├ 매출 목표
   └ 추이·기여도 분석
📄 보고서
   ├ 보고서 목록
   └ 보고서 생성
⚙️ 설정
   ├ 광고 채널 연동
   ├ 사용자 권한
   ├ 개인정보 정책
   ├ 시스템 설정 (임계치)
   └ 온보딩 재시작
```

상단 바: 데이터 신뢰도 상태 요약 (채널별 ✔️/⚠️/🔴 + 최종 업데이트 시각, 수동 재수집 버튼) · 알림 배지 · 계정

## 4. 공통 UI 규칙 (전 화면 적용)

1. **0 표시 금지** — 데이터 없음은 플레이스홀더("데이터 대기 중") 또는 회색 공백. 누락 구간은 0으로 채우지 않는다.
2. **월간 기본** — 기간 필터 기본값은 월간. 주간 수치 옆에는 "경과 확인용 — 전략 판단은 월간 기준" 문구.
3. **신뢰도 병기** — 합산 수치엔 "수집 성공 채널 N/전체 N 기준" 주석, ROAS엔 계약 금액 입력률 병기(90%+ 🟢 / 70~89% 🟡 / 70%↓ 🔴).
4. **시각 표기** — 상대 시각 기본("2시간 전") + 툴팁 절대 시각(KST).
5. **이력 불변** — 수정/내보내기/권한 변경 이력은 읽기 전용, 관리자도 수정 불가.
6. **Insight 톤** — 오렌지 #FF5B2C는 화면당 포인트(주 CTA, 활성 상태, 핵심 KPI)에만. 나머지는 뉴트럴+화이트, 컴팩트 밀도.
7. **ROAS vs ROI 라벨 분리** — ROAS(신규 유입 고객 매출 기준)와 ROI(전체 매출 기준)가 한 화면에 함께 표시될 때는 반드시 "신규 유입 기준"/"전체 포함" 라벨을 병기한다. ROAS는 광고 성과 대시보드(B)에서만, ROI는 매출 관리(D)에서만 사용한다.
8. **매출 입력 단일 경로** — 계약 금액 입력 UI는 매출 관리(D) 화면에서만 노출한다. 광고 성과 대시보드·고객 유입 관리 화면에는 노출하지 않는다.
9. **수집 표현 통일** — "매일 자동 갱신", "실시간" 표현을 쓰지 않는다. 정기 수집(매일 06:00) / 수동 재수집(하루 최대 3회) 두 모드로만 안내한다.
