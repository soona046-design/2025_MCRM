## 1. 프로젝트 개요
---
### 한 줄 정의
통합 마케팅 성과 및 고객 관리 CRM 툴

### 제품 목표
마케팅 ROI를 개선하고, 분산된 광고 및 고객 데이터를 통합하여 관리함으로써, 부서 팀장들이 월간 단위를 기본 판단 기준으로, 주간 단위를 경과 확인용으로 활용해 광고 성과를 신속하고 정확하게 파악하고 효과적인 의사결정을 내릴 수 있도록 지원합니다. 이를 통해 효율적인 사내 보고 자료를 제공하고 전반적인 마케팅 효율성 증대를 목표로 합니다.

### 배경
다양한 디지털 광고 채널의 확산으로 인해 광고 성과 추적이 복잡해지고 있으며, 광고 데이터와 고객 유입 단계 데이터가 여러 시스템에 분산되어 관리되는 비효율성이 발생하고 있습니다. 이로 인해 마케팅 팀장들은 통합된 시각에서 성과를 분석하고 인사이트를 도출하는 데 어려움을 겪고 있으며, 이는 보고 및 전략 수립 과정의 지연과 마케팅 ROI 저하로 이어지고 있습니다.

[출시 단계 구분]
- 1차 출시 — 광고 성과 자동 연동 및 통합 대시보드 / 고객 유입 단계 데이터 관리(수기 입력 및 파일 업로드) / 맞춤형 보고서 및 분석: 팀장의 주간·월간·연간 보고 및 의사결정 지원에 필수적인 핵심 기능
- 2차 출시 검토 — 자동 잠재 고객 점수화 및 우선순위 지정 / 개인화된 고객 커뮤니케이션 도구: 1차 출시 후 실사용 데이터를 바탕으로 범위 및 우선순위를 재검토

### 사용자 문제
마케팅 부서 팀장들은 여러 광고 채널(네이버, 메타, 구글 등)에 걸친 광고 성과를 통합적으로 추적하고 분석하는 데 어려움을 겪고 있으며, 고객 문의부터 계약까지의 유입 후 데이터가 광고 성과 데이터와 분리되어 있어 전반적인 마케팅 퍼널의 효율성을 판단하기 어렵습니다. 이는 분산된 데이터와 수동 보고 작업으로 인한 비효율성으로 이어집니다.

### 해결 방안
주요 광고 플랫폼(네이버, 메타, 구글) API 연동을 통해 광고 노출-클릭 데이터를 자동으로 수집하고, 문의-상담-예약-계약 데이터를 수기 입력 및 스프레드시트 파일 업로드를 통해 유연하게 관리할 수 있도록 합니다. 이 모든 데이터를 하나의 대시보드에서 주간/월간/연간 단위로 통합 분석하여 제공함으로써, 마케팅 퍼널 전체에 대한 명확한 성과 시각화를 제공하고 맞춤형 보고서 작성을 지원합니다.

### 차별점
경쟁사와 달리, 본 CRM 툴은 여러 광고 채널의 데이터를 API로 자동 연동하여 통합하고, 고객 유입 후 단계 데이터를 수기 및 스프레드시트 파일 업로드라는 유연한 방식으로 결합합니다. 특히, 사내 보고 자료를 주로 사용하는 팀장급 사용자의 니즈에 맞춰 채널별 성과 비교, 캠페인 ROI/ROAS, 키워드 효율 분석 등 맞춤형 보고 대시보드를 제공하며, 자동 잠재 고객 점수화와 개인화된 커뮤니케이션 도구를 통해 전환율 향상에 직접적으로 기여합니다.

### 타겟 사용자
주요 타겟 사용자는 마케팅 또는 영업 부서의 팀장입니다. 이들은 여러 광고 채널의 성과를 통합적으로 분석하고, 고객 유입 단계별 전환율을 파악하여 사내 보고 자료를 작성하며, 마케팅 전략 수립 및 개선을 담당합니다. 데이터 기반의 빠른 의사결정과 효율적인 보고서 작성을 필요로 합니다.

### 사용자 시나리오
**[첫 접속 온보딩 시나리오]**
마케팅 팀장인 김 팀장이 처음 CRM 툴에 접속했을 때, 시스템은 3단계 초기 설정 마법사를 안내합니다.

1단계 — 광고 계정 연결: 네이버·메타·구글 광고 계정을 단계별 가이드에 따라 순서대로 연결합니다. 각 채널 연결 완료 시 즉시 연결 성공 여부와 마지막 데이터 수집 시각이 표시됩니다. 계정 연결을 건너뛸 수 있으나, 미연결 채널은 대시보드에서 '미설정' 상태로 표시됩니다.

2단계 — CRM 데이터 업로드 설정: 업로드 템플릿(스프레드시트)을 다운로드하고, 유입 경로·상담일·계약 여부 등 필수 컬럼과 매핑 규칙을 안내받습니다. 첫 업로드 시 샘플 데이터를 활용한 테스트 업로드를 유도하여 필드 매핑이 올바른지 사전 확인합니다.

3단계 — 온보딩 완료 확인: 연결된 채널 수, 업로드 완료 여부, 필수 필드 설정 상태를 체크리스트 형태로 보여줍니다. 모든 항목 완료 시 대시보드가 활성화되며, 미완료 항목이 있을 경우 대시보드 상단에 '초기 설정 미완료 — 일부 수치가 표시되지 않을 수 있습니다' 배너가 노출됩니다.

**[일상 사용 시나리오]**
마케팅 팀장인 김 팀장은 매주 월요일 오전, 지난 한 주간의 광고 성과를 검토하고 보고서를 작성해야 합니다. 김 팀장은 이 CRM 툴에 접속하여 대시보드에서 네이버, 메타, 구글 광고의 노출, 클릭 수치를 주간 단위로 확인하고, 각 채널별 성과를 비교합니다. 또한, 영업팀에서 수기로 입력하거나 스프레드시트로 업로드한 문의, 상담, 예약, 계약 데이터를 함께 확인하여 전체 마케팅 퍼널의 전환율을 분석합니다. 이를 통해 어떤 광고 채널이나 캠페인이 가장 효율적이었는지 파악하고, 다음 주 전략을 수립하며, 이를 기반으로 사내 주간 보고 자료를 신속하게 작성합니다.

### 핵심 지표
- **ROAS(배) 개선**
  - 기준선: 도입 전 최근 4주(또는 직전 1개월) 채널·캠페인별 ROAS 평균
  - 목표: 도입 후 8주 내, 기준선 대비 **ROAS 10% 이상 개선**(동일 채널/캠페인 조건에서 비교)
  - 측정: 주간 집계(대시보드) + 월간 요약(보고서)
  - 유의: 계약 금액 입력률 70% 미만 구간은 '신뢰도 낮음'으로 분리 집계

- **주간 보고서 작성 시간 단축**
  - 기준선: 도구 도입 전 팀장 기준 주간 보고서 작성 소요 시간(사전 설문 1회)
  - 목표: 도입 후 4주 내 **작성 시간 50% 단축**
  - 측정: 팀장 대상 주 1회 1문항 체크(이번 주 보고서 작성에 걸린 시간)

- **대시보드 데이터 최신성(SLA)**
  - 목표: 광고 성과/비용 데이터가 **최근 24시간 이내 갱신**
  - 측정: 화면에 '최종 업데이트 시각' 표시, 채널/캠페인 단위로 최신성 확인
  - 운영 지표: 24시간 초과 지연 건수 및 비중(데이터 신뢰도 지표)

- **고객 유입 단계 입력률(운영 KPI)**
  - 목표: **90%**
  - 정의: '상담/문의(유입 이벤트) 발생 후 24시간 이내에 CRM 전환 퍼널 단계(리드/상담/견적/계약) 중 해당되는 단계가 1회 이상 기록된 건'의 비율
  - 분모: 유입 이벤트 전체(전화/카톡/폼/DM/예약 등)
  - 분자: 24시간 내 단계 기록 완료 유입 이벤트 수
  - 책임: 담당자(실무자)(입력), 마케팅 팀장(모니터링/개선), 관리자(필드·정책 설정 및 권한)
  - 측정: '미입력/지연' 목록과 지연 시간(예: 24h 초과)을 함께 대시보드/리포트로 확인

- **온보딩 완료율(초기 설정 완료)**
  - 기준선: 첫 접속 사용자 기준
  - 목표: 가입/초대 후 7일 내 **온보딩 완료율 80% 이상**
  - 정의: 광고 계정 1개 이상 연결 + 업로드 템플릿 매핑/첫 업로드 완료 + 온보딩 체크리스트 완료
  - 측정: 온보딩 체크리스트 완료 상태 집계(주간)

## 2. 주요 기능 목록
---

## 1. 광고 성과 자동 연동 및 통합 대시보드
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

다중 광고 채널의 성과 데이터를 자동으로 연동하여 통합 대시보드에서 한눈에 확인할 수 있도록 지원한다.

## 대시보드 메인 화면 기본 레이아웃 (정상 상태 기준)

> 메인 대시보드는 **그래프·차트·카드 위주**로 구성한다. 수치 표(테이블)는 세부 페이지(채널별 분석, 보고서)에서 제공한다.

| 행 | 구성 요소 | 시각화 유형 |
|---|---|---|
| 1행 (요약) | ROAS / 광고비 / 계약 금액 / 전환수 | 수치 카드 4개 (전월 대비 증감 표시) |
| 2행 (비교) | 채널별 성과 비교 | 가로 막대그래프 (CTR·CPC·광고비 기준) |
| 2행 (비교) | 전환 퍼널 | 깔때기(Funnel) 차트 (유입→상담→계약) |
| 3행 (추이) | ROAS 기간별 추이 | 꺾은선 그래프 (월간 기본, 주간 토글) |
| 3행 (추이) | 채널 비중 변화 | 누적 막대 또는 파이 차트 |
| 하단 | 데이터 신뢰도 요약 | 상태 배지 + 수집 완결도 수치 |

### 시각화 원칙
- 메인 대시보드: 그래프·차트·카드만 배치. 표는 노출하지 않는다.
- 세부 페이지(채널별 분석, 캠페인 분석, 보고서): 표(테이블)와 그래프를 나란히 배치한다.
- 기간 기본값: 월간. 주간 토글은 보조 선택지로 배치 (S-SVKWCD 기준).
- 빈 상태: 수치 카드에 0 표시 금지. 데이터 없음 플레이스홀더 또는 '데이터 대기 중' 안내로 대체.
- 데이터 신뢰도 경고(누락 채널, 낮은 입력률)는 해당 카드/차트 위에 배너로 노출.

**수용 기준:**
```
1. ○ 네이버, 메타, 구글 등 주요 채널 API가 연동되어 매일 자동으로 데이터가 갱신된다.
2. ○ 채널별, 캠페인별 성과 지표(CTR, CPC 등)를 대시보드에서 세분화하여 확인할 수 있다.
3. ○ ROAS가 자동 계산되어 대시보드에 시각화된다.
4. ○ API 연동 오류 발생 시 관리자에게 알림이 제공된다.
5. ○ 1개 이상 채널 수집 실패 시, 대시보드에 실패 채널명과 누락 구간을 명시한 경고 배너가 자동 노출된다.
6. ○ 채널 실패 비율이 20% 이상일 때 합계 수치는 숨김 또는 '데이터 불완전' 상태로 표시되며, 0으로 채우지 않는다.
7. ○ 부분 집계 허용 시 합산 수치 툴팁에 '수집 성공 채널 N/전체 N 기준' 주석이 표시된다.
8. ○ 메인 대시보드에는 표(테이블)를 노출하지 않고, 카드·그래프·차트만 배치한다.
```


### 1.1 다중 광고 채널 API 연동 자동화
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

네이버, 메타, 구글 등 다양한 광고 플랫폼의 API와 안정적으로 연동하여 광고 성과 데이터를 매일 자동으로 수집하고 갱신하는 기능입니다. 연동 상태를 모니터링하고 API 변경에 따른 오류를 자동 감지하여 관리자가 즉시 조치할 수 있도록 알림을 제공합니다.

## API 연동 실패 시 처리 원칙

**1) 실패 유형 분류**
- 전체 실패: 모든 채널 수집 불가 → 대시보드 전체에 '수집 중단' 배너 표시, 이전 정상 수집 데이터 유지
- 부분 실패: 일부 채널만 실패 → 성공한 채널 데이터만 집계, 실패 채널은 '데이터 없음' 별도 표기
- 지연: 수집은 됐으나 기준 시각(매일 오전 6시) 초과 → '업데이트 지연' 표기 및 마지막 수집 시각 노출

**2) 데이터 표시 규칙**
- 누락 구간의 수치는 0으로 채우지 않고 회색 공백으로 표시하여 팀장이 누락 여부를 즉시 인지할 수 있도록 한다.

통합된 광고비와 전환 데이터를 기반으로 캠페인별 **ROAS(매출÷광고비)** 를 자동 계산하고, 대시보드에 직관적인 그래프와 수치로 표시하는 기능입니다. (본 제품 내 실제 측정 지표는 **ROAS(=계약 금액÷광고비)**로 산출하며, 상위 목표의 마케팅 ROI 개선을 풀기 위한 해심 차렷 지표입니다.)

- **ROAS(배)** = 계약 금액 합산 ÷ 광고비 합산

## 결측/부분 데이터 처리

3) 안정성이 중요한 팀은 '외부 매출 데이터 연동 옵션'을 통해 수기 의존도를 낮춘다.

- 입력률 차이가 큰 캠페인 비교 시 '입력률 차이로 ROAS 비교가 왜곡될 수 있음' 안내 표시

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 1.3.4 ROAS/전환율 계산 시 온라인 광고 매칭 규칙 및 미분류 처리

ROAS 및 퍼널 전환율 계산에서 '광고 성과(비용/클릭)'와 'CRM 전환(문의~계약)'을 연결하는 기준을 명확히 정의합니다. 핵심 원칙은 **증거 기반으로만 광고 성과(ROAS/CPA 등)에 귀속**하고, 근거가 부족한 전환은 '미상'으로 분리해 신뢰도를 지키는 것입니다.

## 접근 권한(역할)
- 본 항목은 운영/정책 기준 정의이므로, 마케팅 팀장 / 관리자만 편집·설정 가능
- 담당자(실무자)는 조회만 가능

0) 용어 정리(핵심)
- 홈페이지 폼 유입: '유입 채널'이 아니라 **전환 이벤트(Conversion)**
- 유입 채널(소스): 전환이 어디서 왔는지(예: 구글 검색광고/네이버 검색광고/자연검색/다이렉트/리퍼럴 등)
- 귀속(Attribution): 전환을 특정 채널/캠페인에 연결하는 규칙

1) 유입 채널(소스) 분류 원칙
- 검색광고(확정): 광고 클릭/캠페인 근거가 있어 **특정 매체/캠페인까지 귀속 가능한 경우**
  - 예: gclid/gbraid/wbraid, 네이버 클릭 ID(nclk 등), UTM(utm_source, utm_medium, utm_campaign), 광고 플랫폼 전환 이벤트ID
- 자연검색(SEO): 검색엔진 자연유입으로 확인되는 경우
- 다이렉트: 직접 입력/즐겨찾기 등 참조 정보가 없는 경우
- 리퍼럴/소셜/기타: 외부 도메인/소셜/기타로 확인되는 경우
- 소스 미상: 폼 전환은 확인되나, 소스/캠페인 근거가 부족해 확정 분류가 불가한 경우

2) 온라인 광고 '증거(매칭 근거)' 정의
- 강한 증거(권장): 캠페인ID/광고플랫폼 전환 이벤트ID/UTM/클릭 ID(gclid 등)
- 약한 증거(보조): 상담 시 '검색광고 보고 왔다' 진술 + 상담 채널/기간/지역 등 추가 맥락

3) 귀속 및 집계 우선순위(ROAS/CPA 계산 기준)
- (1) 강한 증거로 매체/캠페인 특정 가능 → '검색광고(확정)'로 귀속
  - 광고비(비용/클릭)와 연결해 ROAS/CPA 산출 가능
- (2) 온라인 광고로는 추정되나 캠페인 특정 불가(강한 증거 없음) → '소스 미상(온라인 추정)'으로 분리
  - **광고비에 매칭하지 않음**(ROAS/CPA 분모·분자 왜곡 방지)
  - 전환 수(문의/상담/계약)와 매칭은 별도 지표로만 노출
- (3) 오프라인/지인소개/기타 → 광고 ROAS 영역에서 제외, '유입 채널별 CRM 성과' 리포트에서 전환율·계약률·객단가 등으로 비교

4) 홈페이지/폼/전화·카카오톡 등 상담 채널 처리
- 상담 채널(전화/카카오톡/홈페이지 폼 등)은 '유입 채널(소스)'와 별도로 저장
- 홈페이지 폼 유입은 전환 이벤트로 기록하고, 다음 규칙으로 소스/캠페인을 귀속
  - UTM/클릭 ID/전환 이벤트ID 등 근거가 있으면: 해당 매체/캠페인으로 귀속(검색광고 확정)
  - 근거가 없으면: '소스 미상'으로 남김(다이렉트로 자동 간주 금지)

5) 리포트(대시보드/보고서) 노출 원칙
- 광고 ROAS/CPA 리포트에는 '검색광고(확정)' 전환만 포함(증거 기반)
- 퍼널/전체 성과 리포트에는 전체 전환을 포함하되, '확정'과 '미상'을 반드시 분리 표기
- 데이터 품질 지표 함께 표기:
  - 캠페인 귀속률(%): 전체 전환 중 '검색광고(확정)'로 귀속된 비율
  - 소스 미상 비중(%): 전체 전환 중 '소스 미상' 비율

6) 확정 전환 0건 기간의 ROAS 카드 처리 (빈 상태 규칙)
- **ROAS 카드에 0배 또는 0을 표시하지 않는다.**
- 표시 기준:
  - 광고비 있음 + 확정 전환 0건 → '검색광고(확정) 전환 없음 — ROAS 계산 불가' 플레이스홀더
  - 소스 미상 전환이 있는 경우 → '소스 미상 전환 N건 (ROAS 미포함)' 별도 행으로 표시
  - 광고비도 없고 전환도 없음 → ROAS 카드 숨김 + 온보딩 안내 CTA
- 툴팁: 'UTM/클릭 ID가 포함된 전환 데이터를 입력하면 ROAS가 산출됩니다.'

7) 예시
- 구글 검색광고 클릭 후 랜딩→폼 제출(gclid/UTM 존재) → '검색광고(확정)'로 귀속, 캠페인별 ROAS/CPA 계산에 포함
- 네이버 검색광고 클릭 후 폼 제출(nclk 존재) → '검색광고(확정)'로 귀속
- 폼 제출은 있었지만 UTM/클릭 ID 없음 → '소스 미상'(전환/매칭은 집계, 광고비 미연결)
- 전화로 들어왔고 '검색광고 보고 왔다'라고 말했지만 캠페인 근거 없음 → '소스 미상(온라인 추정)'(전환만 집계, 광고비 미연결)
- 간판 보고 전화 옴 → 오프라인(광고 ROAS 제외, 유입 채널 성과로만 비교)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 1.3.5 ROI 수익 확정 기준 및 집계 정책

본 문서에서 혼선이 있던 ROI/ROAS 용어를 정리합니다. **제품 내 지표명은 ROAS로 통일**하며, 산출 기준/집계 정책도 ROAS 기준으로 기술합니다.

## 1) 제품 내 지표 정의(표준)
- **ROAS(배) = 계약 금액 합산 ÷ 광고비 합산**
- 표기 단위: 배(예: 3.2배)
- (참고) ROI(%) 용어는 제품 내 화면/보고서에서 사용하지 않습니다.

## 2) 집계 기간 단위 정의 및 우선순위

> ⚠️ **매출(계약 금액) 집계 및 팀 보고·전략 판단의 기준 단위는 월간입니다.** 주간은 경과 확인용 참고 지표로만 활용하며, 연간은 누적 성과 확인용입니다. 동일 수치를 서로 다른 기준으로 해석하는 혼선을 막기 위해 각 화면/보고서에서 기간 단위의 용도를 아래 표 기준으로 명확히 안내합니다.

| 기간 단위 | 판단 기준 | 주요 용도 | 설명 |
|---|---|---|---|
| **월간 (기본)** | ✅ 팀 보고·전략 판단의 기준 단위 | 광고비·ROAS·계약 금액 집계, 월간 보고서 | 해당 월 1일 00:00 KST ~ 말일 23:59 KST |
| 주간 (참고) | 🔍 경과 확인용 참고 지표 | 중간 점검·이슈 탐지 | 일요일 00:00 KST ~ 토요일 23:59 KST |
| 연간 (요약) | 📊 연간 누적 성과 확인 | 연말 보고·예산 기준 | 1월 1일 00:00 KST ~ 12월 31일 23:59 KST |

> 💡 **화면/보고서 표기 원칙**: 주간 수치 옆에는 반드시 '경과 확인용 — 전략 판단은 월간 기준을 사용하세요' 안내 문구를 노출합니다. 월간 집계가 기본값으로 선택되어야 하며, 주간 필터는 보조 선택지로 배치합니다.

### 월간 집계 정확성 보장 원칙
- **월간 집계 대상 기간**: 반드시 해당 월 **1일 00:00 KST** 부터 **말일 23:59 KST** 까지로 고정
- 말일 23:59 KST까지 '계약 완료' 상태로 변경된 건만 해당 월 매출에 포함
- 말일 23:59 KST 이후(다음 날 00:00 이후) 처리된 건은 다음 달 집계로 귀속
- 광고비(비용): 해당 일자 KST 기준으로 귀속 (오전 6시 수집이 늦어져도 귀속 날짜는 전날 유지)

### 월간 마감 시 주의사항
- 월말 마감 타이밍(예: 31일 밤~1일 새벽)에 처리된 계약은 반드시 '계약 완료' 상태 변경 시각 기준으로 자동 귀속
- 팀장은 매월 1일 오전에 전월 집계 확정 여부를 대시보드에서 확인 가능
- 전월 마감 후 소급 입력된 계약 금액은 해당 월 ROAS 자동 재계산 (소급 반영 이력 기록)

## 3) 매출(계약 금액) 확정 시점
- CRM 상태가 '계약 완료'로 변경된 **날짜(KST 기준)** 를 매출 인식 시점으로 사용한다.
- 광고비 귀속일과 동일한 KST 기준을 사용하여 같은 기간 비교가 일관되게 유지된다.

## 4) 매출 값 출처 및 우선순위
- **1순위**: 외부 연동 옵션(ERP/결제 시스템/타 CRM)이 설정된 경우 연동 데이터 자동 수집
- **2순위(fallback)**: 담당자가 CRM에 수기 입력한 계약 금액 필드
- 연동 데이터와 수기 입력 데이터가 동시에 존재하는 경우: 연동 데이터를 우선 사용하고, 수기 입력 값은 참고값으로 병기

## 5) 결측 데이터 처리
- 계약 완료 상태이지만 계약 금액이 비어 있는 건 → ROAS 계산에서 제외, 대시보드에 '매출 미입력 건수: N건' 별도 표기
- 광고비가 누락된 기간/캠페인 → '광고비 데이터 대기 중' 표시, ROAS 미표시

## 6) 수기 입력 지연/누락 경고 및 알림
- 계약 완료 후 24시간 내 미입력: 담당자 알림
- 48시간 초과 미입력: 담당자 + 팀장 이메일
- 월간 입력률 70% 미만: 팀장 대시보드 배너 + 이메일 (월간 마감 전 마지막 영업일에 추가 알림)

## 7) 소급 변경 규칙
- 계약 금액 수정 시 해당 기간 ROAS 자동 재계산
- 재계산 시점을 이력에 기록(수정 전/후 값 동시 보존)
- 전월 마감 후 소급 수정 시 '전월 ROAS 재계산됨 (○월 ○일 기준)' 알림 노출

> 관련 스펙(통합 지원/대시보드)은 본 정책을 참조하여 계산식 불일치가 발생하지 않도록 한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 관리자


#### 1.3.6 채널별 데이터 신뢰도 지표 표시

ROAS와 합계 수치 옆에 각 채널의 데이터 완결도를 숫자로 표시하여, 팀장이 현재 수치를 얼마나 신뢰할 수 있는지 직관적으로 판단할 수 있도록 지원합니다.

## 접근 권한(역할)
- 마케팅 팀장 / 대표 / 이사 / 관리자만 조회 가능
- 담당자(실무자)는 기본적으로 접근 불가(보고/의사결정 화면 성격)

## 1) 채널별 데이터 완결도 표시 규칙
- 대시보드 각 채널 컨테이너에 **데이터 완결도(%)** 표시
  - 산산 방식: 조회 기간 중 정상 수집된 일수 ÷ 전체 일수 × 100
  - 예시: 7일 조회 기간 중 5일 수집 성공 → 완결도 71%
- 완결도 시각화 기준
  - 100~90%: 정상 (초록 아이콘)
  - 89~70%: 주의 (노란 ⚠️ 아이콘)
  - 69% 이하: 경고 (빨간 🔴 아이콘)
- 전체 ROAS/합계 수치 툴팁에 '이 수치는 X개 채널 중 Y개 기준입니다 (데이터 완결도 Z%)' 주석 자동 표시

## 2) 수치 그룹별 표기 기준
- 완결도 90% 이상: 정상 계산된 수치 표시
- 완결도 70~89%: 수치 + ⚠️ 아이콘 노출, 툴팁으로 누락 날짜·채널 표시
- 완결도 70% 미만: 합계 수치에 '포함된 데이터만 기준, 신뢰도 제한' 표시

## 3) 예외 처리
- 특정 채널이 전체 기간 동안 수집 불가 상태일 경우: '○○ 채널 수집 구간 없음 — ROAS 제외됨' 표시
- 완결도 임계치(70%)(~기본값)는 관리자가 시스템 설정에서 조정 가능
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 대표, 이사, 관리자


#### 1.3.7 외부 매서 데이터 연동 옵션 — ERP/결제 시스템/타 CRM

수기 입력에만 의존하는 ROAS 분자(계약 금액) 데이터를 외부 시스템에서 자동 수집할 수 있는 연동 옵션입니다. 연동을 설정하면 수기 입력 동얰어 관계없이 안정적으로 ROAS 분자를 확보하여 ROAS 신뢰도를 높일 수 있습니다.

> 이 기능은 **옵션**입니다. 미설정 시 수기 입력(fallback)으로 동작합니다.

## 1) 지원 연동 유형

| 연동 방식 | 설명 | 대상 시스템 예시 |
|---|---|---|
| 웹훅(Webhook) | 외부 시스템에서 계약 확정 시 실시간 신호 수신 | 코주카운슬, 독자 ERP |
| REST API 주기 폴링 | 설정한 주기(1일 1회 등)로 자동 집계 | 영업 CRM, 결제 PG |
| 파일 자동 업로드 | 제정 시각에 특정 위치에 놓인 파일을 자동 처리 | Excel/CSV 배치 수신 |

## 2) 연동 설정 항목
- 연동 시스템 유형 선택 (Webhook / API 폴링 / 파일 자동 업로드)
- 수집할 필드 매핑: 외부 시스템의 '계약 금액' 필드 ↔ 본 CRM '계약 금액' 필드
- 확정 상태 필드 매핑: '계약 완료' 상태를 나타내는 필드의 값 정의
- 연동 실패 시 fallback: '수기 입력' / '마지막 성공 수집 값 유지' 중 선택 (기본: 마지막 성공 수집 값 유지)

## 3) 연동 데이터 처리 규칙

### 연동 데이터 vs 수기 입력 충돌 시
- 연동 데이터를 **우선** 사용하고, 수기 입력 값은 '참고값'으로 병기

### 연동 실패/지연 시 화면 처리
- 실패 감지 즉시: '외부 데이터 수집 실패 — 마지막 성공 데이터 유지 중' 표시
- 24시간 이상 실패 지속 시: '외부 연동 실패 — 수기 입력 데이터로 대체 중' 상태로 전환
- 관리자에게 실패 시작 시각 + 원인 + 발송 전환 여부 알림

## 4) 연동 상태 표시

| 연동 상태 | 대시보드 표시 | 카드 레이블 |
|---|---|---|
| 정상 수집 중 | 마지막 수집 시각 | 철 녹선 아이콘 |
| 지연(예상 시각 초과) | '연동 수집 지연 중' | 철 노란 아이콘 |
| 실패 | '연동 실패 — 수기 입력 대체 중' | 철 빨간 아이콘 |
| 미설정 | 수기 입력(fallback) 중 | 회색 상태 |

## 5) 데이터 품질 비교 지표
- 연동 수집 건수 vs 수기 입력 건수 비교 표를 대시보드에 제공
- '연동 데이터 N건 / 수기 입력 N건 / ROAS 산출 포함 N건' 3가지 수치를 요약 표시
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 관리자


#### 1.3.8 대시보드 메인 화면 핵심 카드 및 차트 기본 레이아웃

팀장이 접속했을 때 '무엇을 먼저 봐야 하는지' 헤매지 않도록, 메인 대시보드에 기본으로 배치되는 카드·차트·그래프의 구성과 순서를 정의합니다.

> ✨ **메인 대시보드 원칙**: 표(테이블)는 노출하지 않는다. 모든 정보는 카드·그래프·차트로 시각화한다. 세부 수치 표는 채널별 분석·보고서 등 세부 페이지에서만 제공한다.

## 1) 요약 수치 카드 행 (1행)

| 카드 | 표시 지표 | 부가 정보 |
|---|---|---|
| ROAS | X.X배 | 주요 지표. 전월 대비 증감률 배지 + 계약금액 입력률 표시 |
| 광고비 | N원 | 수집 성공 채널 기준 주석 자동 표기 |
| 계약 금액 | N원 | 미입력 건 제외 주석 표시 |
| 전환수 | N건 | 확정 귀속(검색광고 확정) / 소스 미상 분리 표시 |

- 데이터가 없는 카드: 0 표시 금지. 데이터 대기 중'플레이스홀더 노출
- 기간 기본값: 월간. 우측 상단에 주간 토글 버튼 제공

## 2) 채널별 성과 비교 (2행 좌)
- **차트 유형**: 가로 막대그래프
- **X축**: 채널명 (네이버 / 메타 / 구글)
- **Y축**: 기본값 광고비. 우측 차트 상단에 CTR / CPC / ROAS 탭 전환
- 레이블: 광고비 N원 + ROAS N배 동시 표시
- 누락 채널: 해당 채널 막대는 회색 포인트로 표시 + '데이터 누락' 툴팁

## 3) 전환 퍼널 (2행 우)
- **차트 유형**: 깔때기(Funnel) 차트
- **단계**: 광고 클릭 → 유입 이벤트 → 상담 → 계약
- 각 단계별 전환율 백분율을 광로파이 데이터 레이블로 표시
- 데이터 없는 단계: '입력 없음' 플레이스홀더, 0% 표시 금지

## 4) ROAS 추이 그래프 (3행 좌)
- **차트 유형**: 꺾은선 그래프
- **X축**: 날짜 (월간 기본: 월별, 주간 토글 시: 주별)
- **Y축**: ROAS(배)
- 샄 채널을 서로 다른 색상 선으로 동시 표시 (유제, 파란, 노란)
- 데이터 누락 구간: 회색 점선 구역으로 표시

## 5) 채널 비중 변화 (3행 우)
- **차트 유형**: 누적 막대그래프 (오염차트)
- **X축**: 기간, **Y축**: 채널별 광고비 비율(%)
- 팀장이 특정 채널의 비중 변화를 엔최릴 가능
- 월간 기본: 월별 누적 분포표시

## 6) 데이터 신뢰도 요약 (하단)
- 채널별 수집 완결도 상태 배지 (하단 1행 가로 나열)
  - 네이버 ✓ / 메타 ⚠️ 지연 / 구글 ✓ 식으로 스캔할 수 있도록 표시
- 실패/지연 채널이 있으면 클릭 시 세부 상태 패널 토글
- 계약금액 입력률 바(패널 진도막대스타일): 입력률 70% 미만 시 붉은색 전환

## 7) 카드·차트 코나 공통 규칙
- 모열 시각 표시(툴팁): 해당 수치 + 수집 기준 주석
- 영감어(0, 빈 데이터) 표시 금지. 플레이스홀더 또는 '데이터 대기 중' 안내로 대체
- 실패/누락 데이터: 0 또는 빈엘로 채우지 않고 회색 공백으로 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

## 2. 고객 유입 단계 데이터 관리 (수기 입력 및 파일 업로드)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

[1차 출시] 문의, 상담, 예약, 계약(결제) 등 고객 유입 후 단계 데이터를 수기 입력하거나 스프레드시트(CSV, Excel) 파일 업로드를 통해 관리할 수 있는 기능을 제공합니다. 각 단계별 전환율을 자동으로 계산하여 보여줍니다.

**수용 기준:**
```
1. ○ 사용자는 각 고객 유입 단계(문의, 상담, 예약, 계약)에 대한 정보를 수기로 입력할 수 있어야 한다.
2. ○ 스프레드시트 파일을 업로드하여 여러 건의 고객 유입 단계 데이터를 일괄적으로 등록할 수 있어야 한다.
3. ○ 업로드된 파일의 필수 항목 누락 시 사용자에게 알림 메시지를 제공해야 한다.
4. ○ 각 단계별 전환율이 실시간으로 계산되어 대시보드에 표시되어야 한다.
```


### 2.1 수기 입력 UI 및 유효성 검사
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

수기 입력을 통해 고객 유입 단계 데이터를 관리한다.

- 입력 책임(운영): 담당자(실무자)가 상담/문의 발생 후 24시간 이내 입력을 완료한다.
- 모니터링 책임(운영): 마케팅 팀장이 '미입력/지연' 현황을 주기적으로 확인하고 입력률 90%를 관리한다.
- 정책/권한 책임(운영): 관리자가 필수 필드/허용값/권한을 설정하고 운영 변경을 승인한다.

입력률(90%)은 '유입 이벤트 발생 후 24시간 이내에 전환 퍼널 단계가 1회 이상 기록된 비율'로 측정한다.

## ROAS 분자 확보 규칙(추가)
- 계약 단계 필수 필드에 '계약 금액(원, 숫자)' 필드를 포함한다.
- 계약 금액은 0원보다 크거나 같아야 하며, 미입력 시 '미입력'으로 저장(0으로 반영하지 않음)한다.
- '미입력' 계약 금액은 ROAS 계산에서 제외하고 '금액 미입력' 비율을 별도 표기한다.
- 계약 금액이 입력된 건에 한해서만 ROAS를 산산하고, 미입력 건은 'ROAS 계산 제외(매수정보 없음)'으로 표시한다.


#### 2.1.1 수기 입력 데이터 자동 저장 및 수정 이력 관리

사용자가 고객 유입 단계 데이터를 수기로 입력하는 도중 자동으로 저장하며, 데이터 변경 시 수정 이력을 기록하여 추적할 수 있도록 지원합니다.

| 항목 | 설명 |
|---|---|

| 수정 전 값 | 변경 이전 필드의 원본 값 |

| 수정 이유(선택) | 정보 보완 / 오류 정정 / 계약 내용 변경 / 기타 |

## 이력 조회 권한
- **담당자(실무자)**: 본인이 입력/수정한 건의 이력만 조회
- **마케팅 팀장**: 전체 리드 이력 조회 가능
- **대표 / 이사**: 전체 이력 조회 가능 (연락처 마스킹)
- **관리자**: 전체 이력 조회, 마스킹 없이 원본 확인 가능

## 불변성 보장
- 이력 데이터는 수정하거나 삭제할 수 없다 (읽기 전용).
- 관리자도 이력 자체를 수정할 수 없다.

#### 2.1.2 수기 입력 데이터 입력 가이드 및 도움말 제공

사용자가 수기 입력 시 각 필드별 입력 기준과 예시를 실시간으로 안내하는 가이드와 도움말 기능을 제공합니다. 이를 통해 입력 오류를 줄이고, 사용자의 학습 곡선을 완화합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


#### 2.1.3 상담 유입 경로(온라인/오프라인/기타) 분류 및 입력 필수화

상담/문의 단계 데이터를 수기 입력할 때 '유입 경로'를 필수로 받습니다.
- 1차 분류: 온라인 광고 / 온라인 비광고(홈페이지 자연유입 등) / 오프라인 광고 / 지인소개 / 기타
- 온라인 광고를 선택한 경우에만 2차로 광고 채널(네이버/메타/구글 등) 및 코인 정보(가능한 범위)를 입력 받습니다.
- 온라인 광고 코인을 특정할 수 없는 경우 '코인 미상'으로 저장하여 대시보드에서 별도 집계합니다.
- 전화/카카오톡/홈페이지 폼 등 상담 채널도 함께 저장하여 유입 경로와 교차 분석이 가능하도록 합니다.

### 2.2 스프레드시트 파일 업로드 및 필드 매핑 기능
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

스프레드시트 업로드로 고객 유입 단계 데이터를 반영한다.

- 업로드 책임(운영): 마케팅 팀장이 업로드 운영을 총괄하고, 필요한 경우 담당자(실무자)가 파일을 준비한다.
- 정책/권한 책임(운영): 관리자가 필수 컬럼/허용값/권한을 설정한다.
- 입력률(90%) KPI 측정은 '유입 이벤트 발생 후 24시간 이내 단계 기록' 기준을 동일하게 적용한다(업로드로 반영된 건도 포함).

업로드 누락으로 퍼널 데이터가 비는 것을 막기 위해, '미입력/지연' 현황에서 업로드 미반영 구간을 함께 식별할 수 있어야 한다.


#### 2.2.1 스프레드시트 업로드 파일 유효성 검증 및 오류 안내

사용자가 업로드한 CSV, Excel 파일 내 필수 필드 및 데이터 형식 검증 기능을 제공하며, 누락 또는 형식 오류가 발견될 경우 구체적인 오류 내용을 사용자에게 친절하게 안내하여 즉시 수정할 수 있도록 지원합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 2.2.2 스프레드시트 필드 매핑 자동 제안 및 사용자 편집 지원

업로드된 스프레드시트 파일의 컬럼명을 분석하여 시스템 데이터 필드와 자동으로 매핑을 제안하고, 사용자가 매핑 결과를 수정, 저장할 수 있도록 직관적인 UI를 제공합니다. 이를 통해 파일 포맷 차이에 따른 사용자 편의성을 극대화합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 2.2.3 대용량 스프레드시트 데이터 처리 최적화 및 진행 상태 표시

수천 건 이상의 대용량 고객 유입 단계 데이터를 업로드할 때 서버 부하를 최소화하며, 업로드 진행 상태와 완료 알림을 실시간으로 표시하여 사용자 경험을 개선합니다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 2.2.4 스프레드시트 업로드 시 유입 경로 필수 컬럼 및 매핑 규칙

문의/상담/예약/계약 데이터를 스프레드시트로 업로드할 때 ‘유입 경로’ 컬럼을 필수로 지정하고, 필드 매핑에서 반드시 연결되도록 합니다.

## 접근 권한(역할)
- 마케팅 팀장 / 담당자(실무자) / 관리자만 사용 가능
- 대표 / 이사는 기본적으로 접근 불가(운영 화면 성격)

- 필수 컬럼: 유입 경로(1차), 상담 채널(전화/카카오톡/홈페이지 등), 발생일
- 조건부 필수: 유입 경로=온라인 광고일 때 광고 채널, 캠페인명(또는 캠페인ID/UTM 중 가능한 것)
- 누락/오류 처리: 필수 컬럼 누락 또는 값이 허용 목록에 없으면 업로드를 실패로 처리하고, 행 단위 오류 메시지로 어떤 값이 문제인지 안내합니다.
- ‘캠페인 미상’ 값 허용: 온라인 광고지만 캠페인을 특정 못하는 경우에도 업로드 가능하되, 보고서에서 ‘온라인 광고-미상’으로 분리 집계합니다.

## 확정 귀속(검색광고 포함)을 만들기 위한 업로드 규칙(추가)
- ‘검색광고(확정)’으로 귀속하려면 아래 강한 증거 컬럼 중 1개 이상이 매핑되어야 함(가능하면 2개 이상 권장)
  - utm_source, utm_medium, utm_campaign(UTM)
  - click_id(예: gclid/gbraid/wbraid/nclk 등)
  - conversion_event_id(광고 플랫폼 전환 이벤트ID)
  - campaign_id(캠페인ID)
- 강한 증거 컬럼이 전부 비어 있는데 유입 경로가 ‘온라인 광고/검색광고(확정)’로 들어오면:
  - 업로드 실패(엄격 모드) 또는
  - ‘소스 미상(온라인 추정)’으로 자동 강등(관대 모드)
  - 기본값은 ‘자동 강등(관대 모드)’로 하되, 대시보드에 데이터 품질 경고를 표시
- 강한 증거가 없으면 다이렉트/자연검색/리퍼럴·소셜로 자동 분류하지 않음(근거 없으면 미상)

## 에러 메시지(행 단위) 예시
- "유입 경로가 검색광고(확정)인데 UTM/클릭ID/전환 이벤트ID/캠페인ID가 없습니다. 소스 미상(온라인 추정)으로 저장됩니다."

### 3.1 주간·월간·연간 보고서 기본 템플릿 정의
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)

팀장이 사내 보고에 바로 활용할 수 있도록 주간·월간·연간 보고서의 기본 섹세 구성과 포함 지표를 정의합니다.

## 보고서 종류별 기본 템플릿 구성

| 보고서 | 섹세 순서 | 핵심 지표 |
|---|---|---|
| 주간 | 채널별 성과 비교 → 전환 퍼널 요약 → ROAS 카드 → 데이터 신뢰도 주석 | CTR, CPC, ROAS, 계약 금액 입력률 |
| 월간 | 광고비·ROAS·계약 금액 집계 → 채널별 성과 → 캠페인 비교 → 전환 퍼널 | 광고비, ROAS, 계약 금액, 채널별 전환율 |
| 연간 | 월별 트렌드 그래프 → 연간 ROAS 요약 → 채널 비중 변화 → 주요 성과 하이라이트 | 연간 ROAS, 채널별 비중 변화, 월별 광고비 추이 |

## 시각화 원칙
- 수치 표(테이블)를 단독으로 쓰지 않고, 각 섹션에 대응하는 그래프를 반드시 병행 배치합니다.
  - 채널별 성과 비교 → 막대 그래프(채널별 CTR/CPC/광고비)
  - 전환 퍼널 → 깔때기(Funnel) 차트
  - ROAS 추이 → 꺾은선 그래프(기간별)
  - 채널 비중 변화 → 누적 막대 또는 파이 차트
- 데이터 신뢰도 경고(누락 채널, 낮은 입력률)는 해당 섹세 상단에 배너로 노출

## 기간 단위 기본값
- 월간 집계를 기본값으로 선택, 주간은 보조 선택지로 배치 (S-SVKWCD 기준)

## 템플릿 커스터마이징
- 팀장은 기본 템플릿에서 섹세 표시/숨김을 선택할 수 있으나, 기본 구성은 변경되지 않습니다.


#### 3.1.1 주간 보고서 템플릿 — 섹션 구성 및 차트 배치

주간 보고서 선택 시 자동으로 생성되는 기본 섹션 구성과 시각화 요소를 정의합니다.

## 섹션 배치 순서

**ⓢ 상단: 데이터 신뢰도 상태바**
- 포시 내용: 채널별 수집 상태(완료 ✔️ / 지연 ⚠️ / 실패 포함 시 주황 배경)
- 누락 채널이 있으면 '⚠️ ○○ 채널 ○월 ○일 데이터 누락 — 포함된 수치는 수집 성공 채널 기준입니다' 경고 문구 자동 삽입

**ⓣ 1영역: 채널별 성과 비교**
- 차트 유형: 뭕대 그래프(철형 거뉁 형태, 채널별 막대 병렬)
- X축: 채널명(네이버/메타/구글)
- Y축: 해당 주간 광고비(기본) — 지표 토글 시 CTR·CPC로 전환 가능
- 데이터 레이블: 카드별 수치 직접 표시

**ⓤ 2영역: 전환 퍼널 요약**
- 차트 유형: 깔때기형(퍼널) 그래프
- 단계: 유입 이벤트 → 리드 → 상담 → 견적/제안 → 계약
- 각 단계 카드에 건수와 단계 간 전환율(%) 표시
- '이 주 유입 이벤트 N건 → 계약 N건(전환율 N%)' 형태로 요약 표시

**ⓥ 3영역: ROAS 주간 현황**
- 차트 유형: 채널별 ROAS 가로 카드 나열
- 각 카드에 채널명 + ROAS(배) + 전주 대비 증감 화살표 직접 표시(↑ 증가 / ↓ 감소)
- 계약 금액 입력률 N%를 카드 하단에 표시(신뢰도 관리)
- '경과 확인용 — 전략 판단은 월간 기준 사용하세요' 안내 문구 자동 노출

**ⓦ 하단: 미주 참고 안내**
- '전주 보고서 보기' 링크 + '월간 보고서로 이동' 버튼 노출
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


#### 3.1.2 월간 보고서 기본 구조 정의

월간 보고서 선택 시 자동으로 생성되는 기본 섹션 구성과 시각화 요소를 정의합니다. 월간 보고서는 팀 보고 및 전략 판단의 기본 단위입니다.

## 섹션 배치 순서

**ⓢ 상단: 데이터 신뢰도 상태바**
- 주간 템플릿과 동일 규칙 적용

**ⓣ 1영역: KPI 요약 카드**
- KPI 카드 3종 가로 나열: 광고비 합계 / ROAS / 계약 금액 합계
- 각 카드에 당월 수치 + 전월 대비 증감률(%) + 방향 화살표 직접 표시
- ROAS 카드 하단에 '계약 금액 입력률 N%' 표시

**ⓤ 2영역: 채널별 성과 상세**
- 차트 ①: 겹친 선 그래프(채널별 주차 단위 광고비·ROAS 추이 중첩)
- 차트 ②: 도넛 차트(지출 비중별 채널 비율)
- 누락 구간은 차트 내 회색 점선 구역으로 표시

**ⓥ 3영역: 캠페인별 ROAS 비교**
- 차트: 수평 막대(캠페인별 ROAS 내림차순 정렬), 상위 3개 강조 색상

**ⓦ 4영역: 전환 퍼널 월간 요약**
- 차트: 단계별 전환율 선(주차 변화), 이 달 평균 기준선 점선

**ⓧ 하단: 데이터 신뢰도 주석**
- 수집 성공 채널 N/전체 N 기준 주석 자동 삽입
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


#### 3.1.3 Year-End 보고서 구조 정의

연간(연말) 보고서 선택 시 자동으로 생성되는 기본 섹션 구성과 시각화 요소를 정의합니다.

## 셀션 배치 순서

**ⓢ 상단: 데이터 신뢰도 상태바**
- 당년 월별 수집 성공률 요약

**ⓣ 1영역: 월별 트렌드**
- 차트: 복합(막대 + 겹친 선) — 막대=월별 광고비, 선=ROAS·계약금액 중첩
- X축: 1월~12월, 누락월 회색 점선 구역

**ⓤ 2영역: 연간 ROAS 요약**
- 차트: 반원형 게이지 — 누적 ROAS 바늘 + 목표어 유특표시
- 계약금액 연간 입력률 N%를 게이지 아래 표시

**ⓥ 3영역: 채널 비중 변화**
- 차트: 스택 막대(100% 스택, 채널별 지출 비율)
- X축: 1월~12월, 각 막대에 채널명 레이블

**ⓦ 하단: 데이터 품질 요약**
- 연평균 ROAS + 연간 계약금액 입력률 + '수집 성공 채널 기준' 주석 자동 삽입
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


### 3.2 맞춤형 보고서 필터 및 조건 설정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)

팀장이 광고 채널, 캠페인, 기간, KPI 지표를 자유롭게 선택해 원하는 조합의 보고서를 만들 수 있는 필터·조건 설정 기능입니다.

## 설정 가능한 필터 항목
- **기간**: 직접 입력(날짜 범위) 또는 주간·월간·연간 빠른 선택
- **채널**: 네이버 / 메타 / 구글 / 전체
- **캠페인**: 채널 선택 후 해당 캠페인 목록에서 다중 선택
- **지표**: ROAS, 광고비, CTR, CPC, 전환율, 계약 금액 중 포함할 항목 선택

## 시각화 원칙
- 필터 결과를 표(테이블)와 그래프를 나란히 배치합니다.
  - 다중 채널 선택 시 → 채널별 꺾은선 그래프 자동 생성
  - 캠페인 비교 시 → 가로 막대 그래프(ROAS/광고비 기준 정렬)
- 필터 조건이 변경되면 그래프와 수치가 실시간으로 갱신됩니다.

## 필터 저장 및 재사용
- 자주 쓰는 필터 조합을 이름 붙여 저장하고, 다음 접속 시 바로 불러올 수 있습니다.

## 6. 초기 설정 온보딩 흐름
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

처음 접속한 팀장 및 관리자가 광고 계정 연결 → CRM 데이터 업로드 설정 → 온보딩 완료 확인의 3단계를 거쳐 대시보드가 의미 있는 수치로 채워지도록 안내합니다. 초기 설정이 완료되지 않은 상태에서는 대시보드 주요 수치가 비활성(미설정) 상태로 표시되어, 불완전한 데이터로 의사결정을 내리는 상황을 방지합니다.

**수용 기준:**
```
1. ○ 첫 로그인 시 3단계 초기 설정 마법사가 자동으로 실행된다.
2. ○ 광고 계정 미연결 채널은 대시보드에서 '미설정' 상태로 표시되며, 수치가 0으로 채워지지 않는다.
3. ○ 온보딩 완료 체크리스트에서 미완료 항목이 있을 경우 대시보드 상단에 경고 배너가 노출된다.
4. ○ 온보딩 완료(광고 계정 1개 이상 연결 + 첫 업로드 완료) 시 대시보드가 전체 활성화된다.
```


### 6.1 광고 계정 연결 단계별 가이드 및 연결 완료 확인 UI
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 관리자

첫 로그인 시 네이버, 메타, 구글 광고 계정을 순서대로 연결하도록 단계별 안내 UI를 제공합니다.

## 주요 흐름
- 채널별 연결 버튼 노출: 네이버 광고, 메타 광고, 구글 Ads 개별 연결 버튼
- 연결 클릭 시: 인증 승낙 베이스로 임시 토큰 발급 방식 안내 팝업 노출
- 연결 성공 시: 채널명 + '연결됨 ✔️' + '마지막 데이터 수집: ○월 ○일' 표시
- 미연결 시: '미연결' 마크 + '지금 연결하기' / '나중에 연결하기' 선택 가능

## 안내 요소
- 각 채널별 '연결 방법 보기' 링크: 광고 플랫폼 관리자 페이지에서 토큰 얻는 방법을 단계별 해제(스크린샷 포함)로 안내
- '전체 연결 건너뛰기' 버튼: 클릭 시 온보딩 2단계(업로드 설정)로 직접 이동, 단 '미설정 채널이 N개 있습니다' 확인 다이얼로그 노출

## 연결 후 대시보드 상태
- 미연결 채널: '미설정' 태그 + 레이아웃에서 리스트의 맨 아래 위치
- 연결된 채널 수가 0개이면 대시보드 전체에 '광고 계정 미연결 — 광고 성과를 확인하려면 재설정이 필요합니다' 스티키 배너 노출


### 6.2 업로드 템플릿 다운로드 및 필드 매핑 초기 안내
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 관리자

첫 접속 시 가이드를 통해 스프레드시트 템플릿을 다운로드하고, 필수 컬럼과 매핑 규칙을 확인한 뒤 테스트 업로드를 완료하도록 안내합니다.

## 주요 흐름
1. 템플릿 다운로드
   - '업로드 템플릿 다운로드' 버튼 클릭 시 Excel 파일 제공
   - 템플릿 내 각 컬럼 역할 설명(A열=유입 이벤트, B열=유입 연월일, 등) 주석 포함
   - 샘플 데이터 3줄 실제 예시 포함

2. 필수 컬럼 안내
   - 필수 컬럼 목록(유입 이벤트, 유입 연월일, 유입 채널 코드 등)을 팝업으로 노출
   - 필수 컬럼 누락 시 업로드 차단 및 '필수 컬럼 N개 누락' 오류 메시지 표시

3. 테스트 업로드 유도
   - 첫 업로드는 샘플 5행 이내 데이터로 테스트하도록 안내
   - 업로드 성공 시 '테스트 데이터 N건 성공 확인' + '이제 실제 데이터를 업로드하세요' 안내 버튼 노출
   - 테스트 업로드 데이터는 온보딩 완료 후 자동 삭제 또는 별도 테스트 데이터셋으로 분리하여 보고서에 포함되지 않도록 처리


### 6.3 온보딩 완료 체크리스트 및 대시보드 준비 상태 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 관리자

초기 설정 3단계 완료 여부를 체크리스트로 보여주고, 미완료 여부에 따라 대시보드의 활성화 상태를 제어합니다.

## 체크리스트 항목 구성
| 항목 | 상태 | 설명 |
|---|---|---|
| 광고 계정 연결 | ✔️ / ⏳ | 1개 이상 채널 연결 시 완료 |
| CRM 데이터 첫 업로드 | ✔️ / ⏳ | 첫 업로드 성공 시 완료 |
| 필수 컬럼 설정 | ✔️ / ⚠️ | 필수 컬럼 매핑 정상 시 완료 |

## 대시보드 활성화 조건
- **전체 활성화**: 광고 계정 1개 이상 연결 + CRM 첫 업로드 완료
- **부분 활성화**: 위 조건 중 1개만 충족 → 완료된 데이터 범위만 수치 노출
- **비활성화**: 0개 충족 → '데이터 없음' 플레이스홀더 표시

## 미완료 안내 UI
- 미완료 항목이 있는 체크리스트 완료 화면: '지금 설정 완료하기' 주요 버튼 + '나중에 하기 (제한된 보기로 진입)' 보조 버튼
- 대시보드 진입 후에도 설정 메뉴에서 '온보딩 설정 재시작' 가능
- 온보딩 완료 후 7일이 지나도 '테스트 데이터 삭제' 작동이 없으면 관리자에게 알림 발송

## 7. 매출 관리
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

계약 매출 데이터를 직접 조회·편집하고, 월별 목표 대비 실적을 비교하며, 매출 추이를 시각화하여 팀장과 대표가 매출 현황을 한눈에 파악할 수 있도록 지원합니다.

**수용 기준:**
```
1. ○ 계약 완료 건의 매출 데이터를 목록으로 조회하고 개별 편집할 수 있어야 한다.
2. ○ 월별 매출 목표를 설정하고, 실적과 비교한 달성률을 대시보드에 표시할 수 있어야 한다.
3. ○ 매출 추이를 기간별(주간·월간·연간) 그래프로 시각화하여 확인할 수 있어야 한다.
4. ○ 채널별·캠페인별 매출 기여도를 분석하여 확인할 수 있어야 한다.
```


### 7.1 계약 매출 목록 조회 및 편집
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

계약 완료 고객의 매출 데이터를 목록 형태로 조회하고, 개별 계약에 대한 매출 정보를 수정할 수 있는 화면입니다.

## 목록 조회
- 검색/필터: 기간, 채널, 담당자, 계약 상태(진행 중/완료/취소)
- 정렬: 계약일, 계약 금액 순 정렬 지원
- 열 구성: 고객명, 채널, 고객 담당자, 계약일, 계약 금액, 유입 경로, 상태

## 개별 편집
- 계약 금액, 계약일, 유입 경로, 담당자, 메모 필드 수정 가능
- 수정 이력 자동 기록(수정자, 수정 시각)
- 계약 금액 편집 시 ROAS 자동 재계산


#### 7.1.1 계약 매출 목록 필터 및 정렬

계약 매출 목록에서 다양한 조건으로 필터링하고 정렬할 수 있는 기능입니다.

## 필터 항목
- **기간**: 계약일 범위 선택 (시작일 ~ 종료일)
- **채널**: 네이버 / 메타 / 구글 / 전체
- **담당자**: 등록된 실무자 목록에서 다중 선택
- **상태**: 진행 중 / 완료 / 취소
- **계약 금액 범위**: 최소 ~ 최대 구간 입력

## 정렬
- 계약일 최신순 / 쳤제순
- 계약 금액 높은순 / 낙은순
- 고객명 가나다순

## 키워드 검색
- 고객명, 연락처, 메모 텍스트 키워드 검색
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 7.1.2 계약 매출 개별 상세 조회 및 편집 화면

목록에서 개별 계약을 클릭하면 상세 화면으로 이동하고, 부분 편집이 가능한 화면입니다.

## 상세 화면 구성
- 고객 기본 정보: 이름, 연락처, 담당자
- 채널 및 유입 경로, 계약일, 계약 금액, 상태
- 메모 필드(자유입력)
- 수정 이력: 수정자 / 수정 시각 / 필드명 / 이전값 / 변경값

## 편집 조건
- 인라인 직접 편집 (페이지 이동 없이)
- 편집 완료 시 저장 후 '수정되었습니다' 토스트 알림
- 계약 금액 변경 시: 'ROAS가 재계산됩니다' 팝업 + 확인 후 저장
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 7.1.3 계약 매출 벌크 작업 및 내보내기

목록에서 여러 건을 선택해 일괄 작업하거나 데이터를 내보내는 기능입니다.

## 및 크 선택 및 작업
- 목록 상단 '전체 선택' 체크박스 + 개별 쪽체크박스
- 벌크 상태 변경: '진행 중 / 완료 / 취소' 일괄 변경
- 담당자 일괄 변경: N건 선택 후 담당자 검색·선택

## 데이터 내보내기
- CSV 다운로드: 조회 중인 필터 조건 기준 데이터 내보내기
- 내보내기 권한: 마케팅 팀장 / 관리자만 가능
- 최대 5,000건 제한, 취과 시 '로가운로드 상담 권장' 안내
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


### 7.2 월별 매출 목표 설정 및 달성률 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

월별로 매출 목표를 설정하고, 실적 대비 달성률을 대시보드에 시각화하는 기능입니다.

## 목표 설정
- 월별 매출 목표 금액 입력(월 단위, 원)
- 채널별 매출 목표 세분 설정 가능(선택)
- 목표 설정 권한: 마케팅 팀장 / 관리자

## 달성률 표시
- 대시보드 수치 카드: 이번 달 매출 / 목표 / 달성률(%)
- 목표 이상 달성 시 만은 출력, 미달성 시 붉은 시각 표시
- 월 중 달성 예측치(현재 속도 기준 월말 예상 매출) 표시


#### 7.2.1 월별 매출 목표 입력 및 관리

마케팅 팀장 또는 관리자가 월별 매출 목표를 입력하고 수정하는 화면입니다.

## 입력 UI
- 월별 일런더 형태: 클릭하면 해당 월 목표 입력 팝업 노출
- 필드: 매출 목표 금액(원), 선택적 체널별 세분 목표
- 동일 월 목표가 이미 있으면: '기존 목표 N원 — 덧써우시겠어요?' 확인 팝업
- 미래 월 목표 입력 가능 (2개월 선입력 허용)

## 데이터 검증
- 음수 또는 0월 입력 시 '유효하지 않은 목표 금액입니다' 오류
- 달성률 표시에는 최소 1원 이상이어야 함

## 수정 이력
- 목표 수정 시 수정일시, 수정자, 이전값/변경값 자동 기록
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 7.2.2 달성률 대시보드 카드 및 예측치 표시

이번 달 매출 실적 / 목표 / 달성률을 대시보드에 시각화하는 수치 카드입니다.

## 카드 구성
- **이번 달 매출**: N원 (ROAS 작성 기준 계약 금액 합산)
- **목표**: N원
- **달성률**: N% (실적/목표 × 100)
- **예측치**: 현재 속도 기준 월말 예상 매출 (N원)

## 시각 규칙
- 달성 100% 이상: 만낙 출력 + 초록 배경
- 늬성 70% 미만: 불안 경고 표시 + 빨간 배경
- 달성률 진행률 바: 실적/목표 범위 시각화

## 업데이트 주기
- 실적 수치: 계약 금액 입력 시 실시간 갱신
- 예측치: 일별 매출 펌균 × 남은 일수 기준
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 7.2.3 목표 미달성 알림 및 돈보 기능

월말 마감 전후 달성 상태에 따른 알림과 관리화면 투어 진입 수단입니다.

## 알림 기준
- 월 마지막 영업일 3일 전: 달성률 70% 미만 시 마케팅 팀장 알림
- 월 마지막 영업일: 달성률 90% 미만 시 팀장 + 관리자 알림
- 알림 임계치(기본 70%): 관리자 설정에서 조정 가능

## 돈보 기능
- 알림 클릭 시 '목표 대비 실적 현황' 데이터를 데시보드에서 바로 확인
- '채널별 매출 대비표' 링크 제공 (기여도 분석 화면으로 이동)
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


### 7.3 매출 추이 및 채널별 기여도 분석
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

기간별 매출 추이를 그래프로 확인하고, 채널별·캠페인별 매출 기여도를 분석하는 화면입니다.

## 매출 추이 그래프
- 기간: 주간 / 월간 / 연간 토글
- 그래프 유형: 꼽은선(매출 추이) + 목표 선 병행 표시
- 전년 동기 대비 증감률 표시

## 채널별 기여도
- 채널별 매출 합산 및 비중(%) 파이 차트
- 캠페인별 매출 기여도 기준 정렬 목록
- ROAS와 매출 기여도 연계 조회(출마 성과가 높은 채널 하이라이트)
