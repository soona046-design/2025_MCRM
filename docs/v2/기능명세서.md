## 1. 프로젝트 개요
---
### 한 줄 정의
통합 마케팅 성과 및 고객 관리 CRM 툴

### 제품 목표
마케팅 ROI를 개선하고, 분산된 광고 및 고객 데이터를 통합하여 관리함으로써, 부서 팀장들이 월간 단위를 기본 판단 기준으로, 주간 단위를 경과 확인용으로 활용해 광고 성과를 신속하고 정확하게 파악하고 효과적인 의사결정을 내릴 수 있도록 지원합니다. 이를 통해 효율적인 사내 보고 자료를 제공하고 전반적인 마케팅 효율성 증대를 목표로 합니다.

### 배경
다양한 디지털 광고 채널의 확산으로 인해 광고 성과 추적이 복잡해지고 있으며, 광고 데이터와 고객 유입 단계 데이터가 여러 시스템에 분산되어 관리되는 비효율성이 발생하고 있습니다. 이로 인해 마케팅 팀장들은 통합된 시각에서 성과를 분석하고 인사이트를 도출하는 데 어려움을 겪고 있으며, 이는 보고 및 전략 수립 과정의 지연과 마케팅 ROI 저하로 이어지고 있습니다.

[출시 단계 구분]
- 1차 출시 — 광고 성과 자동 연동 및 통합 대시보드 / 고객 유입 단계 데이터 관리(수기 입력 및 파일 업로드) / 맞춤형 보고서 및 분석: 팀장의 주간·월간·연간 보고 및 의사결정 지원에 필수적인 핵심 기능
- 2차 출시 검토 — 자동 잠재 고객 점수화 및 우선순위 지정 / 개인화된 고객 커뮤니케이션 도구: 1차 출시 후 실사용 데이터를 바탕으로 범위 및 우선순위를 재검토

### 사용자 문제
마케팅 부서 팀장들은 여러 광고 채널(네이버, 메타, 구글 등)에 걸친 광고 성과를 통합적으로 추적하고 분석하는 데 어려움을 겪고 있으며, 고객 문의부터 계약까지의 유입 후 데이터가 광고 성과 데이터와 분리되어 있어 전반적인 마케팅 퍼널의 효율성을 판단하기 어렵습니다. 이는 분산된 데이터와 수동 보고 작업으로 인한 비효율성으로 이어집니다.

### 해결 방안
주요 광고 플랫폼(네이버, 메타, 구글) API 연동을 통해 광고 노출-클릭 데이터를 자동으로 수집하고, 문의-상담-예약-계약 데이터를 수기 입력 및 스프레드시트 파일 업로드를 통해 유연하게 관리할 수 있도록 합니다. 이 모든 데이터를 하나의 대시보드에서 주간/월간/연간 단위로 통합 분석하여 제공함으로써, 마케팅 퍼널 전체에 대한 명확한 성과 시각화를 제공하고 맞춤형 보고서 작성을 지원합니다.

### 차별점
경쟁사와 달리, 본 CRM 툴은 여러 광고 채널의 데이터를 API로 자동 연동하여 통합하고, 고객 유입 후 단계 데이터를 수기 및 스프레드시트 파일 업로드라는 유연한 방식으로 결합합니다. 특히, 사내 보고 자료를 주로 사용하는 팀장급 사용자의 니즈에 맞춰 채널별 성과 비교, 캠페인 ROI/ROAS, 키워드 효율 분석 등 맞춤형 보고 대시보드를 제공합니다.

※ 자동 잠재 고객 점수화와 개인화된 커뮤니케이션 도구는 2차 출시 검토 항목으로, 1차 출시 후 실사용 데이터 축적을 바탕으로 범위와 일정을 확정합니다. 1차 출시의 핵심 차별점은 광고 성과 자동 연동·통합 대시보드·맞춤형 보고서입니다.

### 타겟 사용자
주요 타겟 사용자는 마케팅 또는 영업 부서의 팀장입니다. 이들은 여러 광고 채널의 성과를 통합적으로 분석하고, 고객 유입 단계별 전환율을 파악하여 사내 보고 자료를 작성하며, 마케팅 전략 수립 및 개선을 담당합니다. 데이터 기반의 빠른 의사결정과 효율적인 보고서 작성을 필요로 합니다.

### 사용자 시나리오
**[첫 접속 온보딩 시나리오]**
마케팅 팀장인 김 팀장이 처음 CRM 툴에 접속했을 때, 시스템은 3단계 초기 설정 마법사를 안내합니다.

1단계 — 광고 계정 연결: 네이버·메타·구글 광고 계정을 단계별 가이드에 따라 순서대로 연결합니다. 각 채널 연결 완료 시 즉시 연결 성공 여부와 마지막 데이터 수집 시각이 표시됩니다. 계정 연결을 건너뛸 수 있으나, 미연결 채널은 대시보드에서 '미설정' 상태로 표시됩니다.

2단계 — CRM 데이터 업로드 설정: 업로드 템플릿(스프레드시트)을 다운로드하고, 유입 경로·상담일·계약 여부 등 필수 컬럼과 매핑 규칙을 안내받습니다. 첫 업로드 시 샘플 데이터를 활용한 테스트 업로드를 유도하여 필드 매핑이 올바른지 사전 확인합니다.

3단계 — 온보딩 완료 확인: 연결된 채널 수, 업로드 완료 여부, 필수 필드 설정 상태를 체크리스트 형태로 보여줍니다. 모든 항목 완료 시 대시보드가 활성화되며, 미완료 항목이 있을 경우 대시보드 상단에 '초기 설정 미완료 — 일부 수치가 표시되지 않을 수 있습니다' 배너가 노출됩니다.

**[일상 사용 시나리오]**
마케팅 팀장인 김 팀장은 매주 월요일 오전, 지난 한 주간의 광고 성과를 검토하고 보고서를 작성해야 합니다. 김 팀장은 이 CRM 툴에 접속하여 대시보드에서 네이버, 메타, 구글 광고의 노출, 클릭 수치를 주간 단위로 확인하고, 각 채널별 성과를 비교합니다. 또한, 영업팀에서 수기로 입력하거나 스프레드시트로 업로드한 문의, 상담, 예약, 계약 데이터를 함께 확인하여 전체 마케팅 퍼널의 전환율을 분석합니다. 이를 통해 어떤 광고 채널이나 캠페인이 가장 효율적이었는지 파악하고, 다음 주 전략을 수립하며, 이를 기반으로 사내 주간 보고 자료를 신속하게 작성합니다.

### 핵심 지표
- **ROAS(배) 개선**
  - 기준선: 도입 전 최근 4주(또는 직전 1개월) 채널·캠페인별 ROAS 평균
  - 목표: 도입 후 8주 내, 기준선 대비 **ROAS 10% 이상 개선**(동일 채널/캠페인 조건에서 비교)
  - 측정: 주간 집계(대시보드) + 월간 요약(보고서)
  - 유의: 계약 금액 입력률 70% 미만 구간은 '신뢰도 낮음'으로 분리 집계

- **주간 보고서 작성 시간 단축**
  - 기준선: 도구 도입 전 팀장 기준 주간 보고서 작성 소요 시간(사전 설문 1회)
  - 목표: 도입 후 4주 내 **작성 시간 50% 단축**
  - 측정: 팀장 대상 주 1회 1문항 체크(이번 주 보고서 작성에 걸린 시간)

- **대시보드 데이터 최신성(SLA)**
  - 목표: 광고 성과/비용 데이터가 **최근 24시간 이내 갱신**
  - 측정: 화면에 '최종 업데이트 시각' 표시, 채널/캠페인 단위로 최신성 확인
  - 운영 지표: 24시간 초과 지연 건수 및 비중(데이터 신뢰도 지표)

- **고객 유입 단계 입력률(운영 KPI)**
  - 목표: **90%**
  - 정의: '상담/문의(유입 이벤트) 발생 후 24시간 이내에 CRM 전환 퍼널 단계(리드/상담/견적/계약) 중 해당되는 단계가 1회 이상 기록된 건'의 비율
  - 분모: 유입 이벤트 전체(전화/카톡/폼/DM/예약 등)
  - 분자: 24시간 내 단계 기록 완료 유입 이벤트 수
  - 책임: 담당자(실무자)(입력), 마케팅 팀장(모니터링/개선), 관리자(필드·정책 설정 및 권한)
  - 측정: '미입력/지연' 목록과 지연 시간(예: 24h 초과)을 함께 대시보드/리포트로 확인

- **온보딩 완료율(초기 설정 완료)**
  - 기준선: 첫 접속 사용자 기준
  - 목표: 가입/초대 후 7일 내 **온보딩 완료율 80% 이상**
  - 정의: 광고 계정 1개 이상 연결 + 업로드 템플릿 매핑/첫 업로드 완료 + 온보딩 체크리스트 완료
  - 측정: 온보딩 체크리스트 완료 상태 집계(주간)

## 2. 주요 기능 목록

## 1. 광고 성과 자동 연동 및 통합 대시보드
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

다중 광고 채널의 성과 데이터를 자동으로 연동하여 통합 대시보드에서 한눈에 확인할 수 있도록 지원한다.

## 대시보드 메인 화면 기본 레이아웃 (정상 상태 기준)

> 메인 대시보드는 **그래프·차트·카드 위주**로 구성한다. 수치 표(테이블)는 세부 페이지(채널별 분석, 보고서)에서 제공한다.

| 행 | 구성 요소 | 시각화 유형 |
|---|---|---|
| 1행 (요약) | ROAS / 광고비 / 전체 매출 / 전환수 | 수치 카드 4개 (전월 대비 증감 표시) |
| 2행 (비교) | ROAS · ROI 기간별 추이 | 이중축 라인 차트 (ROAS 좌축 / ROI 우축) |
| 3행 (비교) | 채널별 성과 비교 | 가로 막대그래프 (CTR·CPC·광고비 기준) |
| 3행 (비교) | 전환 퍼널 | 깔때기(Funnel) 차트 (유입 이벤트 → 리드 → 상담 → 견적·제안 → 계약) |
| 4행 (추이) | 채널 비중 변화 | 누적 막대 또는 파이 차트 |
| 하단 | 데이터 신뢰도 요약 | 상태 배지 + 수집 완결도 수치 |

### 시각화 원칙
- 메인 대시보드: 그래프·차트·카드만 배치. 표는 노출하지 않는다.
- 세부 페이지(채널별 분석, 캠페인 분석, 보고서): 표(테이블)와 그래프를 나란히 배치한다.
- 기간 기본값: 월간. 주간 토글은 보조 선택지로 배치 (S-SVKWCD 기준).
- 빈 상태: 수치 카드에 0 표시 금지. 데이터 없음 플레이스홀더 또는 '데이터 대기 중' 안내로 대체.
- 데이터 신뢰도 경고(누락 채널, 낮은 입력률)는 해당 카드/차트 위에 배너로 노출.

**수용 기준:**
```
1. ○ 네이버, 메타, 구글 등 주요 채널 API가 연동되어 매일 자동으로 데이터가 갱신된다.
2. ○ 채널별, 캠페인별 성과 지표(CTR, CPC 등)를 대시보드에서 세분화하여 확인할 수 있다.
3. ○ ROAS가 자동 계산되어 대시보드에 시각화된다.
4. ○ API 연동 오류 발생 시 관리자에게 알림이 제공된다.
5. ○ 1개 이상 채널 수집 실패 시, 대시보드에 실패 채널명과 누락 구간을 명시한 경고 배너가 자동 노출된다.
6. ○ 채널 실패 비율이 20% 이상일 때 합계 수치는 숨김 또는 '데이터 불완전' 상태로 표시되며, 0으로 채우지 않는다.
7. ○ 부분 집계 허용 시 합산 수치 툴팁에 '수집 성공 채널 N/전체 N 기준' 주석이 표시된다.
8. ○ 메인 대시보드에는 표(테이블)를 노출하지 않고, 카드·그래프·차트만 배치한다.
```


### 1.1 다중 광고 채널 API 연동 자동화
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

네이버, 메타, 구글 등 다양한 광고 플랫폼의 API와 안정적으로 연동하여 광고 성과 데이터를 자동으로 수집하고 갱신하는 기능입니다. 연동 상태를 모니터링하고 API 변경에 따른 오류를 자동 감지하여 관리자가 즉시 조치할 수 있도록 알림을 제공합니다.

## API 연동 실패 시 처리 원칙

**1) 실패 유형 분류**
- 전체 실패: 모든 채널 수집 불가 → 대시보드 전체에 '수집 중단' 배너 표시, 이전 정상 수집 데이터 유지
- 부분 실패: 일부 채널만 실패 → 성공한 채널 데이터만 집계, 실패 채널은 '데이터 없음' 별도 표기
- 지연: 수집은 됐으나 기준 시각(매일 오전 6시) 초과 → '업데이트 지연' 표기 및 마지막 수집 시각 노출

**2) 데이터 표시 규칙**
- 누락 구간의 수치는 0으로 채우지 않고 회색 공백으로 표시하여 팀장이 누락 여부를 즉시 인지할 수 있도록 한다.
- 보고서 생성 시 실패 채널이 있으면 상단에 경고 배너('○○ 채널 ○월 ○일 데이터 누락')를 표시한다.
- 전체 합산 수치(광고비 합계, ROAS 등)는 수집 성공 채널 기준으로만 산출하며, 집계 기준 주석을 함께 표기한다.

**3) 소급 반영**
- 재수집 성공 시 누락 구간 데이터를 자동 소급 반영하고, 반영 시각을 이력에 기록한다.
- 소급 반영 이후 관련 보고서의 수치가 변경된 경우, 해당 보고서에 '데이터 업데이트됨' 알림을 표시한다.

## 수집 스케줄 정의 (단일 기준)

> ⚠️ **'매일 자동 갱신', '실시간 수집' 등의 표현은 이 기능에서 사용하지 않는다.** 수집 방식은 아래 두 가지 모드로만 정의한다.

| 수집 모드 | 트리거 | 수집 범위 | 대시보드 반영 시점 | 제한 |
|---|---|---|---|---|
| **정기 수집** | 매일 오전 6시 자동 실행 | 전일 데이터 전체 (3개 채널 동시) | 수집 완료 후 10분 이내 | 하루 1회, 채널별 차이 없음 |
| **수동 재수집** | 관리자/팀장이 '지금 수집' 버튼 클릭 | 최근 7일 데이터 재수집 | 수집 완료 후 5분 이내 | 하루 최대 3회, 토큰 유효 시에만 가능 |

- 채널별 수집 시각 차이는 없다. 3개 채널을 동시에 수집하며, 각 채널 완료 시각은 다를 수 있으나 정기 수집의 기준 시각은 **오전 6시 단일**이다.
- '수시 수집', '실시간 갱신' 모드는 제공하지 않는다. 당일 최신 데이터는 익일 오전 6시 정기 수집 이후 반영된다.

- 대표 / 이사는 기본적으로 접근 불가(운영 화면 성격)

## 실패 지속 시간별 알림 단계
| 실패 지속 시간 | 알림 유형 | 수신자 |
|---|---|---|
| 1시간 이내 | 시스템 내 알림(배지 + 광고 관리 화면 배너) | 관리자 |

| 24시간 이상(일일 데이터 없음) | 이메일 + 대시보드 전체 수집 중단 배너 | 관리자 + 모든 사용자 |

## 채널 부분 실패 시 수치 취급 기준

| 실패 채널 비율 | 합산 수치(광고비·ROAS 등) 처리 | 경고 표시 |
|---|---|---|
| 0% (전체 성공) | 정상 표시 | 없음 |
| 1~19% (일부 실패) | 성공 채널 기준 부분 집계 허용 | ⚠️ '일부 채널 데이터 누락 — 수집 성공 채널 기준' 배너 필수 |
| 20% 이상 (대규모 실패) | 합산 수치 숨김, '데이터 불완전' 상태로 표시 | 🔴 '데이터 불완전 — 수치 신뢰 불가' 경고 배너 필수 |
| 100% (전체 실패) | 대시보드 전체 '수집 중단' 배너, 이전 정상 데이터 유지 | 🔴 전체 수집 중단 배너 |

- 임계치(20%)는 관리자가 시스템 설정에서 조정 가능하다(기본값 20%).
- 부분 집계 허용 구간에서도 합산 수치 툴팁에 '수집 성공 채널 N/전체 N 기준' 주석을 반드시 표기한다.

## 대시보드 및 보고서 내 누락 구간 표시 규칙
- 누락 구간의 데이터는 0으로 채우지 않고 회색 공백으로 표시하며, 툴팁으로 '마지막 수집: ○월 ○일 ○시 ○분' 안내
- 보고서 생성 시 실패 채널이 1개 이상이면 보고서 상단에 '⚠️ ○○ 채널 ○월 ○일 데이터 누락 — 포함된 수치는 수집 성공 채널 기준입니다' 경고 표시
- 전체 합산 수치(광고비 합계 등)는 '수집 성공 채널 기준' 주석을 함께 표기하여 누락 포함 여부를 팀장이 스스로 판단할 수 있도록 한다.

## 재수집 및 소급 반영
- 실패 후 재수집 성공 시 누락 구간 데이터 자동 소급 반영, 반영 시각을 이력에 기록
- 소급 반영 이후 관련 보고서의 수치가 변경되면 해당 보고서에 '데이터 업데이트됨 (○월 ○일 ○시)' 표시

## 연동 실패 이력 조회
- 관리자는 채널별 실패 이력(시작·종료 시각, 실패 사유, 재수집 성공 여부)을 최소 90일 치 조회 가능
- 이력 데이터를 광고 데이터 신뢰도 평가 및 팀내 도구 선정 평가 시 사용 가능

## 0) 수집 모드 개요 (단일 기준 정의)

> 이 스펙의 상태 전환 규칙은 F-JOCPFK에 정의된 **두 가지 수집 모드**를 기준으로 한다.

| 모드 | 트리거 | 동작 특성 |
|---|---|---|
| **정기 수집** | 매일 오전 6시 자동 | 3개 채널 동시 수집. 하루 1회. 수집 완료 후 대시보드 수치 자동 갱신. |
| **수동 재수집** | 관리자/팀장 '지금 수집' 버튼 | 최근 7일 재수집. 하루 최대 3회. 토큰 유효 시에만 가능. |

- **수시/실시간 수집 모드는 제공하지 않는다.** 당일 최신 데이터는 익일 오전 6시 수집 후 반영된다.
- 다음 업데이트 예정 시각: '다음 업데이트: 내일 오전 6시' 항상 노출 (QA 검증 기준: 대시보드에서 반드시 표시되어야 함)

## 1) 수집 상태 종류 및 정의

| 상태 | 조건 (단일 기준) | 아이콘 |
|---|---|---|
| **수집 중** | 정기 또는 수동 수집 API 호출 진행 중 | 파란 스피너 또는 애니메이션 아이콘 |
| **수집 완료** | 마지막 수집 성공 후 24시간 이내 | 녹색 ✔️ + '마지막 수집: ○월 ○일 ○시 ○분' |
| **수집 지연** | 수집은 됐으나 **매일 오전 6시 정기 수집 기준 시각** 초과 | 노란 ⚠️ + '업데이트 지연 — 마지막 수집: ○시간 전' |
| **수집 실패** | API 오류 또는 토큰 만료 | 빨간 경고 표시 + '수집 실패' |

> QA 검증 포인트: 위 4가지 상태 전환 조건은 **주시/수동 구분 없이 동일하게 적용**된다. 각 모드별 특이 상태는 쟠 3번 이후 '지연' 여부만 다르다(오전 6시 기준).

## 2) 노출 위치 및 형태
- **대시보드 상단 바**: 전체 화면 상단에 '현재 데이터 상태 요약' 바 표시
  - 예: "네이버 ✔️ · 메타 ⚠️ 지연(3시간 전) · 구글 ✔️ · 최종 업데이트: 2시간 전"
  - 실패 또는 지연 채널이 있으면 바 배경색 주황으로 변경
- **대시보드 각 채널 카드**: 채널명 옆에 상태 아이콘 + 최종 수집 시각 한 줄 표시
- **수집 중 상태**: 수치 카드 위에 '업데이트 중...' 스피너 오버레이 (5초 이상 지속 시에만 노출)

## 3) 상태별 화면 규칙

### 수집 중
- 수치: 직전 회차 데이터 유지 표시(입력이 있는 경우), 없으면 '업데이트 준비 중' 플레이스홀더
- 툴팁: '○ 채널 데이터를 처음 가져오는 중입니다. 일반적으로 10분 이내 완료됩니다.'
- 수집이 완료되면 카드 자동 갱신(페이지 전체 리로드 없이)

### 수집 완료
- '마지막 수집: ○월 ○일 ○시 ○분' 툴팁 제공
- **'다음 업데이트: 내일 오전 6시'** 안내 표시 (이 문구에서 QA는 다음 수집 시각을 검증함)

### 수집 지연
- '업데이트 지연 — 마지막 수집: ○시간 전' 표시
- '마지막 수집 데이터 보기' 링크 노출 (클릭 시 해당 기간 필터 자동 적용)
- 6시간 이상 지연 시 관리자에게 이메일 알림 발송 (S-IBWSRN 규칙 연동)
- **지연 판정 기준**: '오전 6시 시작 후 현재까지도 완료 되지 않은 상태' = 지연. 수동 재수집은 비교 대상이 아님(오전 6시 기준 엄수함)

### 수집 실패
- '특정 채널 데이터를 받아오지 못했습니다. (실패 시각: ○월 ○일 ○시)' 메시지
- '재시도' 버튼: 클릭 시 즉시 해당 채널 API 재호출, 버튼은 '업데이트 중...' 스피너로 변경
- '관리자 알림 보내기' 보조 링크: 클릭 시 관리자에게 실패 내용 알림 전송
- 실패 지속 시간별 단계적 알림은 S-IBWSRN 규칙 준수

## 4) 대시보드 상단 '데이터 신뢰도' 요약 설계

**보고 타이밍(월요일 오전) 사용 최적화**
- 대시보드 접속 시 상단에 '현재 수치 신뢰도' 요약 표시:
  - 모든 채널 수집 완료: "현재 수치 신뢰할 수 있어요 — 마지막 업데이트: ○시간 전 / 다음 업데이트: 내일 오전 6시"

#### 1.1.3 광고 채널 연동 토큰 보안 정책

### 1.2 채널별 및 캠페인별 상세 성과 분석
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)

#### 1.2.2 성과 저하 요인 및 개선 인사이트 제공

광고 채널·캠페인의 성과가 기준치 이하로 떨어졌을 때 자동으로 감지하고, 저하 원인을 분류하여 개선 방향을 제시하는 기능입니다.

## 성과 저하 감지 기준

| 지표 | 저하 기준(기본) | 기준 단위 |
|---|---|---|
| CTR(클릭률) | 전주 대비 20% 이상 하락 | 채널·캠페인별 |
| CPC(클릭당 비용) | 전주 대비 20% 이상 상승 | 채널·캠페인별 |
| ROAS | 전월 대비 15% 이상 하락 | 캠페인별 |
| 광고비 소진율 | 일 예산 대비 30% 미달 3일 연속 | 캠페인별 |

- 임계치는 관리자가 시스템 설정에서 조정 가능(기본값 위 표 기준)
- 저하 감지 주기: 매일 오전 6시 수집 완료 직후 자동 실행

## 저하 원인 자동 분류 규칙

| 원인 유형 | 판단 기준 | 표시 라벨 |
|---|---|---|
| 광고비 부족 | 일 예산 조기 소진(오후 3시 이전 소진) 3일 이상 | '예산 조기 소진' |
| 데이터 수집 누락 | 해당 채널 완결도 70% 미만 | '데이터 부족 — 판단 보류' |
| 경쟁 심화(CPC 급등) | CPC 20% 이상 상승 + 노출 수 감소 동시 | 'CPC 급등 — 경쟁 심화 가능성' |
| 광고 소재 노출 빈도 포화 | 동일 소재 3주 이상 운영 + CTR 지속 하락 | '소재 교체 필요' |
| 계절·시즌 요인 | 전년 동기 대비 동일 패턴 | '시즌 요인 — 전년 비교 확인' |
| 원인 미특정 | 위 조건 해당 없음 | '추가 확인 필요' |

- 원인 분류는 1개 캠페인에 복수 라벨 동시 부여 가능
- 데이터 수집 누락이 원인인 경우, 다른 원인 라벨과 함께 표시하고 '수집 완료 후 재확인 권장' 문구 노출

#### 1.2.3 성과 데이터 추세 분석 및 예측 시각화

과거 광고 성과 데이터를 기반으로 향후 추세를 계산하고, 그래프에 예측 구간을 시각화하는 기능입니다.

## 예측 방식
- **방법**: 최근 N주(기본 8주) 이동평균(Moving Average)으로 다음 2주 예측값 산출
- **예측 대상 지표**: CTR, CPC, ROAS, 광고비
- **최소 데이터 요건**: 최소 4주치 완결도 70% 이상 데이터 필요. 미충족 시 예측 미표시 + '데이터 부족으로 예측을 표시할 수 없습니다(최소 4주 필요)' 안내
- 이동평균 기간(기본 8주)은 관리자가 4~12주 범위에서 조정 가능

## 예측 구간 표시 기준

| 데이터 완결도 | 신뢰 구간 표시 | 라벨 |
|---|---|---|
| 90% 이상 | ±10% 범위 밴드 | '예측 (신뢰도 높음)' |
| 70~89% | ±20% 범위 밴드 | '예측 (참고용)' |
| 70% 미만 | 예측 미표시 | '데이터 부족' |

- 예측 구간은 실제 데이터 선과 시각적으로 구분되도록 점선 + 음영 밴드로 표시
- 과거 예측값과 실제값을 겹쳐 표시하여 예측 정확도를 팀장이 확인 가능

## 예측 그래프 UI
- 과거 실적 선: 실선
- 예측 구간: 점선 + 색상 반투명 밴드 (채널별 색상 동일 유지)
- X축: 날짜(주 단위 기본). 예측 기간 시작점에 '← 실적 / 예측 →' 구분선
- '예측값은 이동평균 기반 참고 수치입니다. 광고 운영 조건 변경 시 실제와 달라질 수 있습니다' 주석 고정 노출

- 예측 그래프 우측 상단 토글로 '예측 표시 켜기/끄기' 선택 가능 (기본: 켜기)
- 끄면 과거 실적 선만 표시

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


#### 1.2.4 UTM 기반 캠페인 매칭 및 온라인/오프라인 전환 분리 집계

광고 API로 수집된 캠페인 데이터와 CRM에 입력된 전환 데이터를 UTM/클릭 ID 기준으로 연결하여 캠페인·키워드 단위 ROAS 산출이 가능하도록 합니다. S-EDXCFJ의 귀속 정책을 채널별 성과 분석 화면에 적용하는 실행 로직입니다.

## 1) 매칭 로직 실행 순서
1. CRM 전환 이벤트에서 강한 증거(캠페인 ID / UTM / 클릭 ID) 확인
2. 강한 증거가 있는 경우: 광고 플랫폼 캠페인 데이터와 1:1 매칭 → '검색광고(확정)' 귀속
3. 강한 증거가 없는 경우: '소스 미상(온라인 추정)' 처리, 광고비와 미연결

## 2) 채널별 성과 분석 화면에서의 표시
- 각 채널/캠페인 행에 '확정 귀속 전환 수 N건 / 소스 미상 N건' 분리 표시
- 캠페인 귀속률(%) = 확정 귀속 전환 / 전체 전환 × 100
- 귀속률 낮은 캠페인에 '⚠️ 귀속 데이터 부족 — ROAS 신뢰도 낮음' 경고 배지 표시

## 3) 드롭다운 입력과 매칭 연동
- 수기 입력에서 드롭다운으로 선택된 캠페인(신규 추가 스펙 연동)은 캠페인 ID 기반 자동 매칭 처리
- 파일 업로드에서 UTM 자동 감지(신규 추가 스펙 연동)된 행도 동일 매칭 로직 적용

## 4) 매칭 결과 데이터 품질 지표
- 채널별 성과 분석 상단에 '이 기간 캠페인 귀속률 N% / 소스 미상 N%' 요약 바 노출

#### 1.2.5 채널×기간 수치 검증용 상세 표

대시보드에서 드릴다운으로 진입한 팀장이 원천 수치를 빠르게 검증할 수 있도록, 채널별×기간별 에스컬레이터형 상세 표를 제공합니다. 메인 대시보드는 표를 노출하지 않으며, 이 화면(채널별 상세 분석)에서만 표를 제공합니다.

### 1.3 광고 ROI/ROAS 계산 및 시각화
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

통합된 광고비와 전환 데이터를 기반으로 캠페인별 **ROAS(신규 유입 고객 매출÷광고비)** 와 **ROI(전체 매출 기준 수익률)** 를 자동 계산하고, 대시보드에 직관적인 그래프와 수치로 표시하는 기능입니다.

## 지표 정의(제품 내 표준)

| 지표 | 분자(매출) | 분모(비용) | 표시 위치 | 라벨 |
|---|---|---|---|---|
| **ROAS(배)** | 신규 유입 고객 계약 금액 | 광고비 합산 | 광고 성과 대시보드 / 보고서 | '신규 유입 기준' 필수 표기 |
| **ROI(%)** | 전체 계약 금액(신규+기존) — 광고비 | 광고비 합산 | **전체 매출 대시보드(매출 관리)** | '전체 매출 기준' 필수 표기 |

> ✅ **ROI 용어 사용 원칙**
> - ROI는 **전체 매출을 표기하는 대시보드(매출 관리 영역)에서만** 사용한다.
> - 광고 성과 대시보드(유입 기반 채널 분석 화면)에서는 ROAS만 사용하고, ROI는 사용하지 않는다.
> - 한 화면에 ROAS와 ROI가 함께 표시될 때는 두 지표의 분자 차이를 반드시 라벨로 구분한다.

## 데이터 기준
- **ROAS 분자**: 유입 관리와 연결된 신규 유입 고객 중 '계약 완료' 건의 계약 금액(매출 관리 메뉴에서 입력·연결)
- **ROI 분자**: 전체 계약 금액(신규 유입 + 기존 고객) — 광고비
- **분모(비용)**: 광고 플랫폼 API 연동 광고비 합산
- 계약 금액 입력은 **매출 관리 메뉴에서만** 가능

## 결측/부분 데이터 처리
- 광고비 또는 계약 금액 둘 중 하나라도 빠진 경우, 해당 지표를 '금액 정보 없음' 또는 '광고 비용 업데이트 대기 중'으로 안내합니다(0으로 채운 계산 금지).

## 수기 입력 의존 리스크 및 운영 방침
> ⚠️ 매출(계약 금액)은 매출 관리 메뉴에서 수기 입력에 의존할 수 있습니다. 입력 지연/누락/편차가 기간·캠페인별 ROAS/ROI 신뢰도를 떨어뜨릴 수 있으므로, 수치와 함께 '계약 금액 입력률'을 반드시 같이 운영합니다.

1. 팀장 주간 판단 시 '계약 금액 입력률' 확인을 기본으로 한다.
2. 입력률 저하 시 시스템 알림(24/48시간 미입력 알림, 입력률 임계치 미만 팀장 알림)을 사용한다.
3. 안정성이 중요한 팀은 '외부 매출 데이터 연동 옵션'을 통해 수기 의존도를 낮춘다.

#### 1.3.4 ROAS/전환율 계산 시 온라인 광고 매칭 규칙 및 미분류 처리

ROAS 및 퍼널 전환율 계산에서 '광고 성과(비용/클릭)'와 'CRM 전환(문의~계약)'을 연결하는 기준을 명확히 정의합니다. 핵심 원칙은 **증거 기반으로만 광고 성과(ROAS/CPA 등)에 귀속**하고, 근거가 부족한 전환은 '미상'으로 분리해 신뢰도를 지키는 것입니다.

0) 용어 정리(핵심)

- 귀속(Attribution): 전환을 특정 채널/캠페인에 연결하는 규칙

1) 유입 채널(소스) 분류 원칙
- 검색광고(확정): 광고 클릭/캠페인 근거가 있어 **특정 매체/캠페인까지 귀속 가능한 경우**
  - 예: gclid/gbraid/wbraid, 네이버 클릭 ID(nclk 등), UTM(utm_source, utm_medium, utm_campaign), 광고 플랫폼 전환 이벤트ID

2) 온라인 광고 '증거(매칭 근거)' 정의
- 강한 증거(권장): 캠페인ID/광고플랫폼 전환 이벤트ID/UTM/클릭 ID(gclid 등)
- 약한 증거(보조): 상담 시 '검색광고 보고 왔다' 진술 + 상담 채널/기간/지역 등 추가 맥락

3) 귀속 및 집계 우선순위(ROAS/CPA 계산 기준)
- (1) 강한 증거로 매체/캠페인 특정 가능 → '검색광고(확정)'로 귀속
  - 광고비(비용/클릭)와 연결해 ROAS/CPA 산출 가능
- (2) 온라인 광고로는 추정되나 캠페인 특정 불가(강한 증거 없음) → '소스 미상(온라인 추정)'으로 분리
  - **광고비에 매칭하지 않음**(ROAS/CPA 분모·분자 왜곡 방지)

  - UTM/클릭 ID/전환 이벤트ID 등 근거가 있으면: 해당 매체/캠페인으로 귀속(검색광고 확정)
  - 근거가 없으면: '소스 미상'으로 남김(다이렉트로 자동 간주 금지)

5) 리포트(대시보드/보고서) 노출 원칙
- 광고 ROAS/CPA 리포트에는 '검색광고(확정)' 전환만 포함(증거 기반)
- 퍼널/전체 성과 리포트에는 전체 전환을 포함하되, '확정'과 '미상'을 반드시 분리 표기
- 데이터 품질 지표 함께 표기:
  - 캠페인 귀속률(%): 전체 전환 중 '검색광고(확정)'로 귀속된 비율
  - 소스 미상 비중(%): 전체 전환 중 '소스 미상' 비율

6) 확정 전환 0건 기간의 ROAS 카드 처리 (빈 상태 규칙)
- **ROAS 카드에 0배 또는 0을 표시하지 않는다.**
- 표시 기준:
  - 광고비 있음 + 확정 전환 0건 → '검색광고(확정) 전환 없음 — ROAS 계산 불가' 플레이스홀더
  - 소스 미상 전환이 있는 경우 → '소스 미상 전환 N건 (ROAS 미포함)' 별도 행으로 표시
  - 광고비도 없고 전환도 없음 → ROAS 카드 숨김 + 온보딩 안내 CTA
- 툴팁: 'UTM/클릭 ID가 포함된 전환 데이터를 입력하면 ROAS가 산출됩니다.'

7) 예시
- 구글 검색광고 클릭 후 랜딩→폼 제출(gclid/UTM 존재) → '검색광고(확정)'로 귀속, 캠페인별 ROAS/CPA 계산에 포함
- 네이버 검색광고 클릭 후 폼 제출(nclk 존재) → '검색광고(확정)'로 귀속
- 폼 제출은 있었지만 UTM/클릭 ID 없음 → '소스 미상'(전환/매칭은 집계, 광고비 미연결)
- 전화로 들어왔고 '검색광고 보고 왔다'라고 말했지만 캠페인 근거 없음 → '소스 미상(온라인 추정)'(전환만 집계, 광고비 미연결)
- 간판 보고 전화 옴 → 오프라인(광고 ROAS 제외, 유입 채널 성과로만 비교)

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 1.3.5 ROAS 분자 및 집계 정책 표준

ROAS 분자(매출)의 포함 범위, 집계 기간, 매출 확정 기준을 제품 내 멘 하나의 정책으로 정의합니다.

## 1) 핵심 정의(이 문서에서 명확히 하는 것)

| 지표 | 분자 | 설명 |
|---|---|---|
| **ROAS(배)** | **신규 유입 고객 계약 금액** | 광고 채널을 통해 유입된 고객의 계약 금액만 포함. 광고 효율을 순수하게 측정 |
| **계약 금액 합계** | **신규+기존 고객 전체** | 매출 관리 메뉴에서만 표시. ROAS 계산에는 미포함 |

> ⚠️ **ROAS와 계약 금액 합계가 다른 수치를 나타내는 것은 의도된 차이입니다.** 한 화면에 함께 표시될 때는 반드시 '신규 유입 기준' / '전체 포함' 라벨을 비교 안내 툴팁과 함께 노출합니다.

## 2) 매출 입력 위치 원칙
- 전체 계약 금액(신규+기존) 입력은 **매출 관리 메뉴에서만** 가능
- 광고 성과 화면(대시보드, 채널별 분석 등), 유입 고객 관리 화면에는 매출 직접 입력 UI 노출 안 함

## 3) 집계 기간 단위 정의 및 우선순위

> ⚠️ 신규 유입 고객 매출 집계 및 팀 보고·전략 판단의 기준 단위는 **월간**입니다. 주간은 경과 확인용 참고 지표, 연간은 누적 성과 확인용입니다.

| 기간 단위 | 판단 기준 | 주요 용도 |
|---|---|---|
| **월간(기본)** | ✅ 팀 보고·전략 판단의 기준 단위 | ROAS·광고비·계약 금액 집계, 월간 보고서 |
| 주간(참고) | 프 경과 확인용 참고 지표 | 중간 점검·이슈 탐지 |
| 연간(요약) | 프 누적 성과 확인 | 연말 보고·예산 기준 |

- 주간 수치 옵에는 '경과 확인용 — 전략 판단은 월간 기준을 사용하세요' 안내 문구 필수 노출

## 4) 매출(계약 금액) 확정 시점
- CRM 상태가 '계약 완료'로 변경된 **날짜(KST 기준)** 를 매출 인식 시점으로 사용한다.

## 5) 매출 값 출처 및 우선순위
- **1순위**: 외부 연동 옵션(ERP/결제 시스템/타 CRM)이 설정된 경우 연동 데이터 자동 수집
- **2순위(fallback)**: 담당자가 CRM에 수기 입력한 계약 금액 필드

## 6) 결측 데이터 처리
- 계약 완료이지만 계약 금액 빈 건 → ROAS 계산에서 제외, '매출 미입력 건수: N건' 별도 표기
- 광고비 누락 기간/캠페인 → '광고비 데이터 대기 중' 표시, ROAS 미표시

## 7) 수기 입력 지연/누락 경고 및 알림
- 계약 완료 후 24시간 내 미입력: 담당자 알림
- 48시간 초과 미입력: 담당자 + 팀장 이메일
- 월간 입력률 70% 미만: 팀장 대시보드 배너 + 이메일(월간 마감 전 마지막 영업일에 추가 알림)

## 8) 소급 변경 규칙
- 계약 금액 수정 시 해당 기간 ROAS 자동 재계산
- 재계산 시점을 이력에 기록(수정 전/후 값 동시 보존)
- 전월 마감 후 소급 수정 시 '전월 ROAS 재계산됨 (○월 ○일 기준)' 알림 노출

> 관련 스펙(통합 지원/대시보드)은 본 정책을 참조하여 계산식 불일치가 발생하지 않도록 한다.

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 관리자


#### 1.3.6 채널별 데이터 신뢰도 지표 표시

ROAS와 합계 수치 옆에 각 채널의 데이터 완결도를 숫자로 표시하여, 팀장이 현재 수치를 얼마나 신뢰할 수 있는지 직관적으로 판단할 수 있도록 지원합니다.

## 접근 권한(역할)
- 마케팅 팀장 / 대표 / 이사 / 관리자만 조회 가능
- 담당자(실무자)는 기본적으로 접근 불가(보고/의사결정 화면 성격)

## 1) 채널별 데이터 완결도 표시 규칙
- 대시보드 각 채널 컨테이너에 **데이터 완결도(%)** 표시
  - 산산 방식: 조회 기간 중 정상 수집된 일수 ÷ 전체 일수 × 100
  - 예시: 7일 조회 기간 중 5일 수집 성공 → 완결도 71%
- 완결도 시각화 기준
  - 100~90%: 정상 (초록 아이콘)
  - 89~70%: 주의 (노란 ⚠️ 아이콘)
  - 69% 이하: 경고 (빨간 🔴 아이콘)
- 전체 ROAS/합계 수치 툴팁에 '이 수치는 X개 채널 중 Y개 기준입니다 (데이터 완결도 Z%)' 주석 자동 표시

## 2) 수치 그룹별 표기 기준
- 완결도 90% 이상: 정상 계산된 수치 표시
- 완결도 70~89%: 수치 + ⚠️ 아이콘 노출, 툴팁으로 누락 날짜·채널 표시
- 완결도 70% 미만: 합계 수치에 '포함된 데이터만 기준, 신뢰도 제한' 표시

## 3) 예외 처리
- 특정 채널이 전체 기간 동안 수집 불가 상태일 경우: '○○ 채널 수집 구간 없음 — ROAS 제외됨' 표시
- 완결도 임계치(70%)(~기본값)는 관리자가 시스템 설정에서 조정 가능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 대표, 이사, 관리자


#### 1.3.7 외부 매서 데이터 연동 옵션 — ERP/결제 시스템/타 CRM

수기 입력에만 의존하는 ROAS 분자(계약 금액) 데이터를 외부 시스템에서 자동 수집할 수 있는 연동 옵션입니다. 연동을 설정하면 수기 입력 동얰어 관계없이 안정적으로 ROAS 분자를 확보하여 ROAS 신뢰도를 높일 수 있습니다.

> 이 기능은 **옵션**입니다. 미설정 시 수기 입력(fallback)으로 동작합니다.

## 1) 지원 연동 유형

| 연동 방식 | 설명 | 대상 시스템 예시 |
|---|---|---|
| 웹훅(Webhook) | 외부 시스템에서 계약 확정 시 실시간 신호 수신 | 코주카운슬, 독자 ERP |
| REST API 주기 폴링 | 설정한 주기(1일 1회 등)로 자동 집계 | 영업 CRM, 결제 PG |
| 파일 자동 업로드 | 제정 시각에 특정 위치에 놓인 파일을 자동 처리 | Excel/CSV 배치 수신 |

## 2) 연동 설정 항목
- 연동 시스템 유형 선택 (Webhook / API 폴링 / 파일 자동 업로드)
- 수집할 필드 매핑: 외부 시스템의 '계약 금액' 필드 ↔ 본 CRM '계약 금액' 필드
- 확정 상태 필드 매핑: '계약 완료' 상태를 나타내는 필드의 값 정의
- 연동 실패 시 fallback: '수기 입력' / '마지막 성공 수집 값 유지' 중 선택 (기본: 마지막 성공 수집 값 유지)

## 3) 연동 데이터 처리 규칙

### 연동 데이터 vs 수기 입력 충돌 시
- 연동 데이터를 **우선** 사용하고, 수기 입력 값은 '참고값'으로 병기

### 연동 실패/지연 시 화면 처리
- 실패 감지 즉시: '외부 데이터 수집 실패 — 마지막 성공 데이터 유지 중' 표시
- 24시간 이상 실패 지속 시: '외부 연동 실패 — 수기 입력 데이터로 대체 중' 상태로 전환
- 관리자에게 실패 시작 시각 + 원인 + 발송 전환 여부 알림

## 4) 연동 상태 표시

| 연동 상태 | 대시보드 표시 | 카드 레이블 |
|---|---|---|
| 정상 수집 중 | 마지막 수집 시각 | 철 녹선 아이콘 |
| 지연(예상 시각 초과) | '연동 수집 지연 중' | 철 노란 아이콘 |
| 실패 | '연동 실패 — 수기 입력 대체 중' | 철 빨간 아이콘 |
| 미설정 | 수기 입력(fallback) 중 | 회색 상태 |

## 5) 데이터 품질 비교 지표
- 연동 수집 건수 vs 수기 입력 건수 비교 표를 대시보드에 제공
- '연동 데이터 N건 / 수기 입력 N건 / ROAS 산출 포함 N건' 3가지 수치를 요약 표시

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 관리자


#### 1.3.8 대시보드 메인 화면 핵심 카드 및 차트 기본 레이아웃

팀장이 접속했을 때 '무엇을 먼저 봐야 하는지' 헤매지 않도록, 메인 대시보드에 기본으로 배치되는 카드·차트·그래프의 구성과 순서를 정의합니다.

> ✨ **메인 대시보드 원칙**: 표(테이블)는 노출하지 않는다. 모든 정보는 카드·그래프·차트로 시각화한다. 세부 수치 표는 채널별 분석·보고서 등 세부 페이지에서만 제공한다.

## 0) 상단 기간 선택기 (페이지 전체에 적용)

- **위치**: 대시보드 최상단 오른쪽 고정 영역에 항상 노출
- **기본값**: 월간
- **선택지**: 주간 / 월간 / 연간 / 직접 입력(날짜 범위)
- **동작**: 기간 변경 시 대시보드 전체(요약 카드, ROAS 차트, 퍼널 등) 실시간 갱신
- **주간 선택 시**: '중간 점검용 참고 지표 — 전략 판단은 월간 기준을 사용하세요' 안내 문구 자동 노출

## 1) 요약 수치 카드 행 (1행)

| 카드 | 표시 지표 | 부가 정보 |
|---|---|---|
| **ROAS(신규 유입 기준)** | X.X배 | 전월 대비 증감률 배지 + 계약금액 입력률 표시. **라벨: '신규 유입 고객 기준'** 필수 표기 |
| **전체 매출(ROI 기준)** | N원 | 신규+기존 고객 전체 매출. **라벨: '전체 매출 기준'** 필수 표기. 미입력 건 제외 주석 표시 |
| 광고비 | N원 | 수집 성공 채널 기준 주석 자동 표기 |
| 전환수 | N건 | 확정 귀속(검색광고 확정) / 소스 미상 분리 표시 |

> ⚠️ **ROAS 카드와 전체 매출 카드는 서로 다른 분자를 사용합니다.** ROAS = 신규 유입 고객 기준, 전체 매출 = 전체 포함. 두 카드의 수치가 다를 수 있으며 이는 의도된 차이입니다.

- 데이터가 없는 카드: 0 표시 금지. '데이터 대기 중' 플레이스홀더 노출
- 기간 기본값: 월간. 상단 기간 선택기에 연동되어 실시간 갱신

## 2) ROAS 기간별 추이 차트 (2행 전체)

> 대시보드에서 광고 투자 효율(ROAS)의 흐름을 한눈에 파악할 수 있는 핵심 영역입니다.

**차트 유형**: 단일 라인 차트
- **ROAS 선(왼쪽 Y축)**: ROAS(배) 추이 — 신규 유입 기준, '신규 유입 고객 기준' 라벨 차트 제목에 표기
- **광고비 막대(선택 계열, 배경)**: 실제 지출 수준 참고용 막대 바 병행 표시

> ℹ️ ROI(전체 매출 기준)는 1행 요약 카드에서 확인하세요. 보고서의 ROAS·ROI 비교 차트에서 두 지표를 함께 볼 수 있습니다.

**X축**: 상단 기간 선택기 고정 연동
- 월간 선택 시: 월별 표시
- 주간 선택 시: 주별 표시
- 연간 선택 시: 월별 표시

**데이터 누락 구간**: 선 차트에서 회색 점선 구역으로 표시

**차트 제목**: 'ROAS 기간별 추이 (신규 유입 고객 기준)'

**툴팁**: X축 한 점 클릭 시 ROAS N배 / 광고비 N원 동시 노출

## 3) 채널별 성과 비교 (3행 좌)
- **차트 유형**: 가로 막대그래프
- **X축**: 채널명 (네이버 / 메타 / 구글)
- **Y축**: 기본값 광고비. 우측 차트 상단에 CTR / CPC / ROAS 탭 전환
- 레이블: 광고비 N원 + ROAS N배 동시 표시
- 누락 채널: 해당 채널 막대는 회색 포인트로 표시 + '데이터 누락' 툴팁

## 4) 전환 퍼널 (3행 우)

> 퍼널 단계 기준은 S-VTSOND(CRM 전환 퍼널 단계 정의)를 단일 소스로 따릅니다.

- **차트 유형**: 깔때기(Funnel) 차트
- **단계**: 유입 이벤트 → 리드 → 상담 → 견적·제안 → 계약 (5단계, S-VTSOND 기준)
- 각 단계별 전환율 백분율을 데이터 레이블로 표시
- 데이터 없는 단계: '입력 없음' 플레이스홀더, 0% 표시 금지
- 견적·제안 단계 비활성화 시 4단계(유입 이벤트→리드→상담→계약)로 자동 축소

## 5) 채널 비중 변화 (4행 좌)
- **차트 유형**: 누적 막대그래프
- **X축**: 기간, **Y축**: 채널별 광고비 비율(%)
- 팀장이 특정 채널의 비중 변화를 확인할 수 있음

## 6) 데이터 신뢰도 요약 (하단)
- 채널별 수집 완결도 상태 배지 (하단 1행 가로 나열)
- 실패/지연 채널이 있으면 클릭 시 세부 상태 패널 토글
- 계약금액 입력률 바(패널 진도막대스타일): 입력률 70% 미만 시 붉은색 전환

## 7) 카드·차트 공통 규칙
- 툴팁: 해당 수치 + 수집 기준 주석
- 0, 빈 데이터 표시 금지. 플레이스홀더 또는 '데이터 대기 중' 안내로 대체
- 실패/누락 데이터: 0 또는 빈 값으로 채우지 않고 회색 공백으로 표시

### 2.1 수기 입력 UI 및 유효성 검사

## 2) 역할 정의

- **대표 / 이사**: 모든 데이터 읽기 가능, 개인정보(연락처)는 마스킹 처리. 입력·수정·내보내기 불가.
- **팀장**: 데이터 수정, 보고서 생성·내보내기, 업로드, 전체 이력 조회, 광고 계정 연동/해제 등 최대 권한.
- **담당자(실무자)**: 파일 업로드, 수기 입력(본인 입력분 수정), 데이터 읽기 가능. 광고 계정 연동 불가.
- **관리자**: 전체 권한 + 시스템 설정 변경 권한 + 토큰 감사 로그 조회 권한 보유.

> ⚠️ 위 매트릭스는 기본값이며, 권한 관리 메뉴에서 아이디별·기능별로 개별 조정 가능합니다.

## 3) 내보내기 이력(감사 추적)

- PDF/Excel 내보내기 실행 시 다음 항목을 자동 기록:
  - 실행자(이름 + 역할), 실행 시각, 파일명, 보고서 종류, 조회 기간 및 필터 조건
- 관리자는 내보내기 이력을 최소 90일 치 조회 가능
- 이력 데이터는 수정·삭제 불가 (불변 로그)

## 4) 반출 파일 후 통제 규칙 (파일 유출 예방)

> 파일이 사용자 단말로 반출된 이후는 시스템 접근권한의 직접 통제 범위를 벗어납니다. 아래 규칙은 **운영 정책**으로 기환하여 내보내기 화면 시 사용자에게 고지합니다.

### 파일 내 데이터 기준 (역할별)
- 보고서 PDF: 역할별 포함 데이터는 S-GDGSPV 기준을 따른다
- Excel/CSV 내보내기: 파일에 포함되는 데이터는 S-BPTWNR 기준을 따른다
- **대표/이사 기준 PDF**: 연락처 원문 포함 금지 (마스킹 포함)
- **팀장/관리자 기준 PDF**: 연락처 원본 포함 가능

### 재배포 금지 고지
- 보고서(PDF) 스펙: 마지막 페이지는 '사내 보고 목적으로 생성. 외부 배포 및 재공유 금지.' 고지 문구 고정 삽입 (S-GDGSPV)
- 계약 매출 Excel: '개인정보 포함 파일 — 사내 보고 외 외부 배포 금지.' 고지 행 자동 삽입 (S-BPTWNR)

### 내보내기 실행 시 사용자 안내
- '이 파일에는 개인정보가 포함되어 있습니다. 보관 및 재배포는 개인정보 처리방침에 따라 진행해 주세요.' 다운로드 직전 팝업 또는 하단 안내 문구로 노출

## 5) 권한 위반 처리

- 권한 없는 기능 접근 시: '해당 기능은 [역할명]만 사용할 수 있습니다' 안내 표시
- URL 직접 접근 등 비정상 접근 시도: 시스템 자동 차단 + 관리자 이력 기록

## 6) 권한 변경 절차

- 관리자만 역할별 권한을 수정할 수 있다.
- 권한 변경 즉시 해당 역할 사용자에게 적용(실시간 반영).
- 권한 변경 이력(바꾼 역할, 변경자, 변경 날짜, 변경 전후 권한 목록)은 불변 로그로 보존.

## 7) 개인정보 보관·삭제 정책 연동

- 이 매트릭스 안에서 어떤 역할이 어떤 필드를 볼 수 있는지는 S-HAFECH(개인정보 보관·삭제·익명화 정책)에서 정의한 개인정보 범위를 준수한다.
- 결론: 화면 접근 기준 = 권한 매트릭스 / 파일 내 데이터 기준 = S-GDGSPV(PDF) + S-BPTWNR(Excel) / 자동 마스킹 대상 필드 = S-HAFECH / 토큰 보안 구현 기준 = S-TDMIEH

## 8) 개인정보 보관 기간 정책 운영 R&R (추가)

- **관리자**: 개인정보 보관 기간/삭제·익명화 정책 설정 및 변경 권한 보유(정책 변경 이력 포함)
- **마케팅 팀장**: 주 단위로 삭제/익명화 처리 리포트(처리 예정/완료/실패)를 확인하고, 실패 건에 대한 재시도/조치 요청 책임
- **담당자(실무자)**: 고객의 삭제 요청이 접수되면 표준 사유를 기록하고 관리자에게 전달(정책에 따라 처리)
- **대표/이사**: 정책 준수 여부를 요약 지표로 확인(개인정보 원문 접근 없이 마스킹 상태로만)

> 상세 규칙(보관 기간, 대상 필드, 트리거, 처리 방식)은 상세 기능 '개인정보 보관·삭제·익명화 정책'을 따른다.

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 관리자


#### 2.1.5 아이디별·기능별 권한 관리 메뉴

관리자가 각 사용자 아이디별로 기본 역할에 더해 기능별 권한을 개바로 설정할 수 있는 별도 관리 메뉴입니다.

## 1) 메뉴 위치
- 시스템 설정 > 사용자 권한 관리
- 관리자 역할만 접근 가능

## 2) 주요 기능

### 사용자 목록 보기
- 전체 사용자 아이디 목록(이름, 역할, 마지막 로그인 시각, 계정 상태)
- 역할로 필터링 가능

### 역할 할당
- 아이디마다 기본 역할 지정(마케팅 팀장 / 대표 / 이사 / 담당자(실무자) / 관리자)
- 역할 변경 즉시 실시간 적용

### 기능별 권한 개바 설정
- 기본 역할 권한 외에, 특정 아이디에 한해 다음 항목을 개별 처리 가능:

| 설정 항목 | 옵션 |
|---|---|
| 보고서 내보내기 (PDF/Excel) | 허용 / 차단 |
| 스프레드시트 업로드 | 허용 / 차단 |
| 비용/ROAS 데이터 조회 | 허용 / 차단 |
| 고객 연락처 원본 보기 (마스킹 해제) | 허용 / 차단 |
| 수정 이력 전체 조회 | 허용 / 차단 |

- 개바 설정은 담당자도 팀장/대표 등 다른 역할의 기본 권한을 초과하여 부여할 수 없음(=하향 상한).

## 3) 변경 이력
- 어떤 관리자가, 언제, 어떤 아이디의, 어떤 설정을 바꿏는지 전체 이력 보존
- 이력은 수정·삭제 불가 (불변 로그)

## 4) 예외 처리
- 권한 없는 메뉴 접근 시: 접근 차단 + '해당 메뉴는 관리자만 사용 가능' 안내
- 사용자 자신이 본인 권한을 스스로 상향 조정할 수 없다(=자기 권한 스스로 상승 불가)

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 관리자


#### 2.1.6 개인정보 보관·삭제·익명화 정책

CRM 데이터(리드/상담/계약)에서 개인정보를 어떤 기준으로 보관하고, 언제/어떻게 삭제 또는 익명화(개인 식별이 불가능하게 바꽄기)할지 정의합니다.

> 원칙: 분석/보고 목적의 지표는 가능한 유지하되, 개인을 식별할 수 있는 정보는 최소 기간만 보관하고 만료 시 자동으로 삭제 또는 익명화합니다.

## 1) 개인정보 범위(예시)

| 분류 | 필드 예시 | 비고 |
|---|---|---|
| 직접 식별자 | 이름, 전화번호, 이메일, 카카오/메신저 ID | 삭제/익명화 1순위 |
| 준식별/민감 가능 | 주소, 회사명/부서, 상담 내용 메모 | 정책에 따라 일부 마스킹/익명화 |
| 비식별 분석 지표 | 유입 경로, 쳪페인ID, 전환 단계/시각, 광고비, 계약 금액, ROI | 개인 식별 불가 형태로 유지 권장 |

## 2) 기본 보관 기간(완초안)

| 데이터 종류 | 기본 보관 기간 | 만료 시 처리 |
|---|---:|---|
| 리드(잠재고객) 개인정보(직접 식별자) | 12개월 | 익명화(권장) 또는 삭제 |
| 상담/문의 메모(내용) | 12개월 | 부분 마스킹 또는 삭제 |
| 계약 데이터(금액/상태 등 비식별 지표) | 36개월 | 비식별 지표는 유지 |
| 내보내기/수정 이력(감사 로그) | 90일 | 개인정보는 마스킹된 형태로만 보관 |

> 위 기간은 기본값 완초안이며, 관리자가 업종/법적 요구에 맞춰 설정 화면에서 조정할 수 있어야 합니다.

## 3) 삭제/익명화 트리거
- 보관 기간 만료(자동)
- 고객의 삭제 요청(수동 처리 + 이력 기록)
- 계정(고객사) 서비스 해지 후 N개월(예: 3개월) 경과

## 4) 처리 방식 정의

### A. 익명화(권장 기본값)
- 이름/전화/이메일/메신저ID 등 직접 식별자는 빈 값 또는 해시 처리(역추적 불가)
- 상담 메모는 내용 삭제 후 '메모 삭제됨(보관 만료)' 표기만 남김
- 리드ID는 내부 분석용 임의 식별자로 교체(개인 식별 불가)

### B. 삭제
- 직접 식별자 및 리드 레코드 자체를 삭제
- 단, 분석용 집계(예: 유입 경로별 전환 수, 쳪페인별 ROI)는 개인을 식별할 수 없는 집계 데이터만 남김

### 갱신 트리거
- **비동기 배치 처리**: 파일 업로드 완료 → 서버 큐 등록 → 백그라운드 처리 → 완료 시 알림. 업로드 중 화면을 떠나도 처리는 계속된다.
- '즉시 반영'은 파일 업로드에 적용하지 않는다. 사용자에게는 '처리 중' 상태를 명시한다.

### 허용 지연 및 처리 시간
| 파일 규모 | 허용 처리 시간 | UI 상태 표시 |
|---|---|---|
| 100행 이하 | **30초 이내** | 스피너 + '업로드 처리 중...' |
| 100~1,000행 | **3분 이내** | 진행 바(%) + '처리 중 N/전체 N건' |

### 실패 처리
- **유효성 오류**: 업로드 전 사전 검증(필수 컬럼 누락, 형식 오류) → 업로드 차단 + 오류 목록 표시. 처리 진입 없음.
- **처리 중 오류**: 일부 행 실패 시 성공 건만 반영, 실패 행 목록을 '업로드 결과 리포트'로 제공. 전체 롤백 없음.
- **전체 실패**: 서버 오류로 큐 처리 불가 시 자동 재시도 **2회** 후 실패 시 '파일 업로드에 실패했습니다. 파일을 다시 업로드해 주세요' 오류 알림 발송.

### 동시 업로드 처리
- 동일 계정에서 파일 2개 이상 동시 업로드 시: 순차 큐 처리. 두 번째 파일은 '대기 중' 상태로 표시.
- 업로드 데이터와 수기 입력 데이터가 동일 고객 ID로 충돌 시: 최신 업로드 시각 기준 적용(Last Write Wins), 충돌 행을 결과 리포트에 표시.

- 조건부 필수: 유입 경로=온라인 광고일 때 광고 채널, 캠페인명(또는 캠페인ID/UTM 중 가능한 것)
- 누락/오류 처리: 필수 컬럼 누락 또는 값이 허용 목록에 없으면 업로드를 실패로 처리하고, 행 단위 오류 메시지로 어떤 값이 문제인지 안내합니다.
- ‘캠페인 미상’ 값 허용: 온라인 광고지만 캠페인을 특정 못하는 경우에도 업로드 가능하되, 보고서에서 ‘온라인 광고-미상’으로 분리 집계합니다.

## 확정 귀속(검색광고 포함)을 만들기 위한 업로드 규칙(추가)
- ‘검색광고(확정)’으로 귀속하려면 아래 강한 증거 컬럼 중 1개 이상이 매핑되어야 함(가능하면 2개 이상 권장)
  - utm_source, utm_medium, utm_campaign(UTM)
  - click_id(예: gclid/gbraid/wbraid/nclk 등)
  - conversion_event_id(광고 플랫폼 전환 이벤트ID)
  - campaign_id(캠페인ID)
- 강한 증거 컬럼이 전부 비어 있는데 유입 경로가 ‘온라인 광고/검색광고(확정)’로 들어오면:
  - 업로드 실패(엄격 모드) 또는
  - ‘소스 미상(온라인 추정)’으로 자동 강등(관대 모드)
  - 기본값은 ‘자동 강등(관대 모드)’로 하되, 대시보드에 데이터 품질 경고를 표시
- 강한 증거가 없으면 다이렉트/자연검색/리퍼럴·소셜로 자동 분류하지 않음(근거 없으면 미상)

## 에러 메시지(행 단위) 예시
- "유입 경로가 검색광고(확정)인데 UTM/클릭ID/전환 이벤트ID/캠페인ID가 없습니다. 소스 미상(온라인 추정)으로 저장됩니다."

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 2.2.5 업로드 템플릿 광고 매칭 키 필수 컬럼 지정

스프레드시트 업로드 템플릿에 광고비↔전환 매칭을 위한 키 컬럼을 필수 또는 조건부 필수로 지정합니다.

## 1) 필수 컬럼 구성

| 컬럼명 | 필수 여부 | 설명 |
|---|---|---|
| 유입 경로(1차) | 필수 | 온라인 광고 / 온라인 비광고 / 오프라인 광고 / 지인소개 / 기타 |
| 발생일 | 필수 | 이벤트 발생일 (KST 기준 날짜) |
| 상담 채널 | 필수 | 전화 / 카카오톡 / 홈페이지 폼 / 기타 |

## 2) 조건부 필수 컬럼 (유입 경로 = 온라인 광고일 때)

| 컬럼명 | 설명 | 비고 |
|---|---|---|
| utm_source | 유입 매체 (예: google, naver) | 없으면 경고 표시 |
| utm_medium | 유입 방식 (예: cpc, organic) | 없으면 경고 표시 |
| utm_campaign | 캠페인 이름 또는 ID | 없으면 경고 표시 |
| click_id | 광고 클릭 ID (gclid / nclk / gbraid 등) | 없으면 소스 미상 처리 |
| campaign_id | 광고 플랫폼 캠페인 ID | click_id 없을 때 보조 매칭 키 |

## 3) 업로드 시 처리 규칙
- 유입 경로=온라인 광고이고 위 조건부 필수 컬럼이 **모두 비어 있을 경우**:
  - 기본 동작(관대 모드): 업로드는 허용하되 해당 행에 '소스 미상(온라인 추정)' 자동 표기 + 경고 메시지 노출
  - '캠페인 미상' 명시 입력 시: 업로드 허용 + 보고서에서 '온라인 광고-미상' 항목으로 분리 집계
- 필수 컬럼(유입 경로, 발생일, 상담 채널) 자체가 누락된 경우: 업로드 차단 + 행 단위 오류 메시지

## 4) 템플릿 파일 구성
- 다운로드 제공 Excel 템플릿에 위 컬럼 전체 포함 (헤더 행 색상 구분: 필수=빨간, 조건부=주황, 선택=회색)
- 각 컬럼 헤더 셀에 주석으로 입력 예시 및 허용값 범위 안내 포함
- 샘플 데이터 3줄: UTM/클릭ID 있는 예시 1줄, 없는 예시 1줄, 오프라인 예시 1줄

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 2.2.6 업로드 엑셀 UTM 컬럼 자동 감지 및 캠페인 매핑 제안

스프레드시트 업로드 시, 파일 내 UTM 관련 컬럼명을 자동으로 인식하고 시스템 필드에 매핑 제안을 제공합니다. 기존 S-LLJTQA(자동 매핑 제안)를 UTM 특화 규칙으로 보완합니다.

## 1) 자동 감지 대상 컬럼명 패턴
| 인식 패턴 | 매핑 시스템 필드 |
|---|---|
| utm_source, 유입매체, source | utm_source |
| utm_medium, 유입방식, medium | utm_medium |
| utm_campaign, 캠페인명, campaign | utm_campaign |
| gclid, 구글클릭ID, click_id | click_id |
| nclk, 네이버클릭ID | click_id |
| campaign_id, 캠페인ID | campaign_id |

- 대소문자·공백·밑줄 변형 모두 허용 (예: 'UTM Source', 'utm source', 'UTM_SOURCE' 모두 인식)

## 2) 매핑 제안 UI
- 업로드 후 '필드 매핑 확인' 화면에서 UTM 컬럼을 별도 그룹('광고 매칭 키 컬럼')으로 분리 표시
- 자동 감지된 컬럼: '자동 매핑됨' 초록 배지 표시
- 미감지 컬럼: '직접 지정' 드롭다운 제공

## 3) 저장된 매핑 규칙 재사용
- 이전 업로드에서 사용한 컬럼 매핑 패턴을 저장하고 다음 업로드 시 자동 적용 제안
- '이전 매핑 규칙 적용' 클릭 시 동일 패턴 매핑 자동 완성

## 4) 데이터 품질 미리보기
- 매핑 확인 화면에서 UTM 컬럼이 비어있는 행 비율을 '광고 매칭 가능 비율 N%'로 미리 표시
- 매칭 가능 비율이 낮을 경우: '업로드 후 소스 미상 비율이 높을 수 있습니다' 사전 경고 노출

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

### 3.1 주간·월간·연간 보고서 기본 템플릿 정의
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)

팀장이 사내 보고에 바로 활용할 수 있도록 주간·월간·연간 보고서의 기본 섹세 구성과 포함 지표를 정의합니다.

## 보고서 종류별 기본 템플릿 구성

| 보고서 | 섹세 순서 | 핵심 지표 |
|---|---|---|
| 주간 | 채널별 성과 비교 → 전환 퍼널 요약 → ROAS 카드 → 데이터 신뢰도 주석 | CTR, CPC, ROAS, 계약 금액 입력률 |
| 월간 | 광고비·ROAS·계약 금액 집계 → 채널별 성과 → 캠페인 비교 → 전환 퍼널 | 광고비, ROAS, 계약 금액, 채널별 전환율 |
| 연간 | 월별 트렌드 그래프 → 연간 ROAS 요약 → 채널 비중 변화 → 주요 성과 하이라이트 | 연간 ROAS, 채널별 비중 변화, 월별 광고비 추이 |

## 시각화 원칙
- 수치 표(테이블)를 단독으로 쓰지 않고, 각 섹션에 대응하는 그래프를 반드시 병행 배치합니다.
  - 채널별 성과 비교 → 막대 그래프(채널별 CTR/CPC/광고비)
  - 전환 퍼널 → 깔때기(Funnel) 차트
  - ROAS 추이 → 꺾은선 그래프(기간별)
  - 채널 비중 변화 → 누적 막대 또는 파이 차트
- 데이터 신뢰도 경고(누락 채널, 낮은 입력률)는 해당 섹세 상단에 배너로 노출

## 기간 단위 기본값
- 월간 집계를 기본값으로 선택, 주간은 보조 선택지로 배치 (S-SVKWCD 기준)

## 템플릿 커스터마이징
- 팀장은 기본 템플릿에서 섹세 표시/숨김을 선택할 수 있으나, 기본 구성은 변경되지 않습니다.


#### 3.1.1 주간 보고서 템플릿 — 섹션 구성 및 차트 배치

주간 보고서 선택 시 자동으로 생성되는 기본 섹션 구성과 시각화 요소를 정의합니다.

> 퍼널 단계 기준은 S-VTSOND(CRM 전환 퍼널 단계 정의)를 단일 소스로 따릅니다.

## 섹션 배치 순서

**ⓢ 상단: 데이터 신뢰도 상태바**
- 표시 내용: 채널별 수집 상태(완료 ✔️ / 지연 ⚠️ / 실패 포함 시 주황 배경)
- 누락 채널이 있으면 '⚠️ ○○ 채널 ○월 ○일 데이터 누락 — 포함된 수치는 수집 성공 채널 기준입니다' 경고 문구 자동 삽입

**ⓣ 1영역: 채널별 성과 비교**
- 차트 유형: 막대 그래프(채널별 막대 병렬)
- X축: 채널명(네이버/메타/구글)
- Y축: 해당 주간 광고비(기본) — 지표 토글 시 CTR·CPC로 전환 가능
- 데이터 레이블: 카드별 수치 직접 표시

**ⓤ 2영역: ROAS · 전체 매출 주간 현황 (카드 나열)**
- 구성: ROAS 카드(채널별)와 전체 매출(ROI 기준) 카드를 **별도 카드**로 나란히 배치
- **ROAS 카드**: 채널명 + ROAS(배) + 전주 대비 증감 화살표(↑ 증가 / ↓ 감소) + **'신규 유입 고객 기준'** 라벨 + 계약 금액 입력률 N% 표시
- **전체 매출(ROI 기준) 카드**: 전체 매출 N원 + ROI(%) + **'전체 매출 기준'** 라벨 + 전주 대비 증감률
- 두 카드가 나란히 있을 때 분자가 다름을 라벨로 명확히 구분. 라벨 없이 나란히 놓는 것 금지
- '경과 확인용 — 전략 판단은 월간 기준 사용하세요' 안내 문구 자동 노출

**ⓥ 3영역: 전환 퍼널 요약**
- 차트 유형: 깔때기형(퍼널) 그래프
- 단계: 유입 이벤트 → 리드 → 상담 → 견적·제안 → 계약 (5단계, S-VTSOND 기준)
- 각 단계 카드에 건수와 단계 간 전환율(%) 표시
- '이 주 유입 이벤트 N건 → 계약 N건(전환율 N%)' 형태로 요약 표시
- 견적·제안 단계 비활성화 시 4단계(유입 이벤트→리드→상담→계약)로 자동 축소

**ⓦ 하단: 미주 참고 안내**
- '전주 보고서 보기' 링크 + '월간 보고서로 이동' 버튼 노출

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


#### 3.1.2 월간 보고서 기본 구조 정의

월간 보고서 선택 시 자동으로 생성되는 기본 섹션 구성과 시각화 요소를 정의합니다. 월간 보고서는 팀 보고 및 전략 판단의 기본 단위입니다.

> 퍼널 단계 기준은 S-VTSOND(CRM 전환 퍼널 단계 정의)를 단일 소스로 따릅니다.

## 섹션 배치 순서

**ⓢ 상단: 데이터 신뢰도 상태바**
- 주간 템플릿과 동일 규칙 적용

**ⓣ 1영역: KPI 요약 카드**
- KPI 카드 4종 가로 나열: 광고비 합계 / ROAS / 전체 매출 / ROI(%)
- 각 카드에 당월 수치 + 전월 대비 증감률(%) + 방향 화살표 직접 표시
- **ROAS 카드**: '신규 유입 고객 기준' 라벨 + 계약 금액 입력률 N% 표시
- **전체 매출 카드**: '전체 매출 기준' 라벨
- **ROI(%) 카드**: '전체 매출 기준' 라벨 별도 카드로 배치

> ⚠️ ROAS와 ROI(전체 매출 기준)는 분자가 다릅니다. 카드 라벨로 반드시 구분 표시합니다.

**ⓤ 2영역: ROAS 기간별 추이 차트 + ROI 별도 카드**
- **ROAS 추이 차트**: 단일 라인 차트
  - Y축: ROAS(배) — 신규 유입 기준, '신규 유입 고객 기준' 라벨 표기
  - X축: 해당 월의 주차(1~5주)
  - 광고비 막대를 배경 계열로 병행 표시
  - 툴팁: 주차 클릭 시 ROAS N배 / 광고비 N원 동시 노출
  - 데이터 누락 구간: 회색 점선 구역으로 표시
- **ROI 요약 카드 (차트 우측 배치)**: ROI(%) 수치 + 전월 대비 증감률 + '전체 매출 기준' 라벨
  - 보고서 내 ROI와 ROAS가 나란히 있을 때 반드시 라벨로 분자 차이 구분
  - 상세 ROAS·ROI 비교가 필요하면 '채널별 상세 분석' 링크 제공

**ⓥ 3영역: 채널별 성과 상세**
- 차트 ①: 꺾은선 그래프(채널별 주차 단위 광고비·ROAS 추이 중첩)
- 차트 ②: 도넛 차트(지출 비중별 채널 비율)
- 누락 구간은 차트 내 회색 점선 구역으로 표시

**ⓦ 4영역: 캠페인별 ROAS 비교**
- 차트: 수평 막대(캠페인별 ROAS 내림차순 정렬), 상위 3개 강조 색상

**ⓧ 5영역: 전환 퍼널 월간 요약**
- 차트: 단계별 전환율 선(주차 변화), 이 달 평균 기준선 점선
- 단계: 유입 이벤트 → 리드 → 상담 → 견적·제안 → 계약 (5단계, S-VTSOND 기준)
- 견적·제안 단계 비활성화 시 4단계로 자동 축소

**ⓩ 하단: 데이터 신뢰도 주석**
- 수집 성공 채널 N/전체 N 기준 주석 자동 삽입

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


#### 3.1.3 Year-End 보고서 구조 정의

연간(연말) 보고서 선택 시 자동으로 생성되는 기본 섹션 구성과 시각화 요소를 정의합니다.

## 섹션 배치 순서

**ⓢ 상단: 데이터 신뢰도 상태바**
- 당년 월별 수집 성공률 요약

**ⓣ 1영역: 월별 트렌드**
- 차트: 복합(막대 + 겹친 선) — 막대=월별 광고비, 선=ROAS·전체 매출 중첩
- X축: 1월~12월, 누락월 회색 점선 구역

**ⓤ 2영역: 연간 ROAS · ROI 요약**
- 차트 ①: 반원형 게이지 — 누적 ROAS(배) 바늘 + 목표 표시 ('신규 유입 기준' 라벨)
- 차트 ②: 연간 ROI(%) 수치 카드 — 전체 매출 기준, '전체 매출 기준' 라벨 필수 표기
- 계약금액 연간 입력률 N%를 게이지 아래 표시

> ⚠️ ROAS 게이지와 ROI 카드는 분자가 다릅니다. 두 수치가 다를 수 있으며 이는 의도된 차이입니다.

**ⓥ 3영역: 채널 비중 변화**
- 차트: 스택 막대(100% 스택, 채널별 지출 비율)
- X축: 1월~12월, 각 막대에 채널명 레이블

**ⓦ 하단: 데이터 품질 요약**
- 연평균 ROAS + 연간 ROI + 연간 계약금액 입력률 + '수집 성공 채널 기준' 주석 자동 삽입

> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)


### 3.2 맞춤형 보고서 필터 및 조건 설정
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자)

팀장이 광고 채널, 캠페인, 기간, KPI 지표를 자유롭게 선택해 원하는 조합의 보고서를 만들 수 있는 필터·조건 설정 기능입니다.

## 설정 가능한 필터 항목
- **기간**: 직접 입력(날짜 범위) 또는 주간·월간·연간 빠른 선택
- **채널**: 네이버 / 메타 / 구글 / 전체
- **캠페인**: 채널 선택 후 해당 캠페인 목록에서 다중 선택
- **지표**: ROAS, 광고비, CTR, CPC, 전환율, 계약 금액 중 포함할 항목 선택

## 시각화 원칙
- 필터 결과를 표(테이블)와 그래프를 나란히 배치합니다.
  - 다중 채널 선택 시 → 채널별 꺾은선 그래프 자동 생성
  - 캠페인 비교 시 → 가로 막대 그래프(ROAS/광고비 기준 정렬)
- 필터 조건이 변경되면 그래프와 수치가 실시간으로 갱신됩니다.

## 필터 저장 및 재사용
- 자주 쓰는 필터 조합을 이름 붙여 저장하고, 다음 접속 시 바로 불러올 수 있습니다.
- 저장된 필터는 보고서 생성 시 불러오기 가능.

## 데이터 신뢰도 안내
- 선택한 기간·채널에 누락 구간이 있으면 '○○ 채널 ○월 ○일 데이터 누락' 경고를 필터 패널 상단에 표시합니다.


#### 3.2.1 필터 저장 및 즐겨찾기 관리

자주 쓰는 필터 조합을 이름 붙여 저장하고, 다음 접속 시 바로 불러올 수 있는 기능입니다.

- 이름: 최대 30자, 중복 불가
- 저장 항목: 기간, 채널, 캠페인, 지표 조합 전체
- 사용자별 최대 10개 저장 가능 (초과 시 오래된 것 삭제 안내)

## 불러오기 및 재사용
- 보고서 필터 패널 상단 '저장된 필터 불러오기' 드롭다운에서 선택
- 선택 즉시 필터 조건 자동 적용, 차트/수치 실시간 갱신
- 보고서 생성 시에도 저장된 필터 불러오기 가능

## 삭제 및 수정

## 4. 자동 잠재 고객 점수화 및 우선순위 지정

> - 선행 조건 2: 점수화 알고리즘에 필요한 행동 데이터(문의 횟수, 단계 이동 패턴 등) 수집 설계 완료

```

2. ○ 점수가 높은 잠재 고객을 우선순위에 따라 정렬하여 표시해야 한다.
3. ○ 사용자는 잠재 고객의 점수와 우선순위 지정 기준을 확인할 수 있어야 한다.
4. ○ 잠재 고객 점수화가 전환율 향상에 기여했는지 측정할 수 있는 지표를 제공해야 한다.
```


### 4.1 잠재 고객 점수 자동 계산 엔진
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

고객의 유입 경로, 행동 데이터, 문의 내용 등을 기반으로 잠재 고객의 전환 가능성을 자동으로 점수화하는 알고리즘 엔진입니다. 이 엔진은 실시간 데이터 분석을 통해 점수를 갱신하며, 점수 부여 기준과 로직을 관리자 화면에서 설정 및 수정할 수 있도록 지원합니다.

| 변수 | 기본 가중치 | 산출 기준 |

| 단계 변경 횟수 | 15% | 적을수록 높은 점수 (반등 없을수록 유리) |
| 담당자 활동 기록 | 10% | 메모 입력, 단계 변경 등 최근 7일 내 활동 여부 |

## 6. 초기 설정 온보딩 흐름
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

처음 접속한 팀장 및 관리자가 광고 계정 연결 → CRM 데이터 업로드 설정 → 온보딩 완료 확인의 3단계를 거쳐 대시보드가 의미 있는 수치로 채워지도록 안내합니다. 초기 설정이 완료되지 않은 상태에서는 대시보드 주요 수치가 비활성(미설정) 상태로 표시되어, 불완전한 데이터로 의사결정을 내리는 상황을 방지합니다.

**수용 기준:**
```
1. ○ 첫 로그인 시 3단계 초기 설정 마법사가 자동으로 실행된다.
2. ○ 광고 계정 미연결 채널은 대시보드에서 '미설정' 상태로 표시되며, 수치가 0으로 채워지지 않는다.
3. ○ 온보딩 완료 체크리스트에서 미완료 항목이 있을 경우 대시보드 상단에 경고 배너가 노출된다.
4. ○ 온보딩 완료(광고 계정 1개 이상 연결 + 첫 업로드 완료) 시 대시보드가 전체 활성화된다.
```

## 7. 매출 관리
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

계약 매출 데이터를 직접 조회·편집하고, 월별 목표 대비 실적을 비교하며, 매출 추이를 시각화하여 팀장과 대표가 매출 현황을 한눈에 파악할 수 있도록 지원합니다.

**수용 기준:**
```
1. ○ 계약 완료 건의 매출 데이터를 목록으로 조회하고 개별 편집할 수 있어야 한다.
2. ○ 월별 매출 목표를 설정하고, 실적과 비교한 달성률을 대시보드에 표시할 수 있어야 한다.
3. ○ 매출 추이를 기간별(주간·월간·연간) 그래프로 시각화하여 확인할 수 있어야 한다.
4. ○ 채널별·캠페인별 매출 기여도를 분석하여 확인할 수 있어야 한다.
```


### 7.1 계약 매출 목록 조회 및 편집
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

신규 유입 고객과 기존 고객 모두의 매출을 **이 화면(매출 관리 메뉴)에서만** 직접 입력하고 관리합니다. 매출 입력 UI는 매출 관리 메뉴 외 어떤 화면(광고 성과 대시보드, 유입 고객 목록 등)에도 노출하지 않습니다.

## 지표 기준 구분
- **ROAS 계산 기준(신규 유입 고객 매출)**: 유입 관리 화면에서 등록된 신규 유입 고객의 계약 금액만 ROAS 분자에 포함됩니다.
- **전체 계약 금액 합계**: 신규 유입 고객 + 기존 고객 매출을 합산하여 이 화면의 집계 카드에 표시합니다. 이 수치는 ROAS 분자와 다르며, 카드 라벨에 '전체 포함'으로 명시합니다.

## 고객 유형 구분
- **신규 유입 고객**: 유입 관리 화면에서 등록된 고객. 고객 ID로 유입 데이터와 연결되며 ROAS 계산에 포함됩니다.
- **기존 고객**: 유입 관리에 등록되지 않은 고객(재계약, 직접 거래 등). 매출 집계에는 포함되나 ROAS 계산에서는 제외됩니다.

## 매출 직접 입력
- 신규 고객 입력: **유입 고객 목록에서 검색·선택** 시 고객 ID 자동 연결
- 기존 고객 입력: **기존 고객 목록에서 검색·선택** 하면 저장된 이름·연락처·담당자가 자동 입력 (다이렉트 입력도 가능)
- **기존 고객 자동 완성**: 이름 또는 연락처 일부 입력 시 기존 고객 목록에서 일치하는 고객 자동완성 제안

### 7.3 매출 추이 및 채널별 기여도 분석
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

기간별 매출 추이를 그래프로 확인하고, 채널별·캠페인별 매출 기여도를 분석하는 화면입니다. 신규 유입 고객과 기존 고객을 분리하여 조회할 수 있어 채널 성과와 재구매/기존 고객 성과를 구분해서 파악할 수 있습니다.

## 고객 유형 필터
- **전체**: 신규 유입 고객 + 기존 고객 매출 합산
- **신규 유입 고객**: 유입 관리에서 유입된 고객의 매출만 집계 (ROAS 연계 대상)
- **기존 고객**: 재계약·직접 거래 등 유입 데이터가 없는 고객 매출

- 기간: 주간 / 월간 / 연간 토글

- 고객 유형 필터 적용 시 그래프 제목에 '(신규 유입 고객 기준)' 또는 '(기존 고객 기준)' 자동 표기

## 채널별 기여도
- 채널별 매출 합산 및 비중(%) 파이 차트

- ROAS와 매출 기여도 연계 조회 (신규 유입 고객 기준, 광고 성과가 높은 채널 하이라이트)
- 기존 고객 기여도는 '기존 고객 매출' 항목으로 별도 표시


#### 7.3.1 매출 추이 그래프 및 ROAS · ROI 연동 시각화

기간별 매출 추이를 그래프로 표시하고, ROAS와 ROI를 동일 차트에서 비교할 수 있는 이중축 시각화 화면입니다.

## 매출 추이 그래프
- 기간: 주간 / 월간 / 연간 토글
- 그래프 유형: 꼽은선(매출 추이) + 목표선 병행 표시
- 전년 동기 대비 증감률 표시
- 고객 유형 필터 적용 시 그래프 제목에 '(신규 유입 고객 기준)' 또는 '(기존 고객 기준)' 자동 표기

## ROAS · ROI 연동 이중축 차트
- 차트 유형: 동일 X축을 공유하는 이중 라인 차트
- **ROAS 선(왜쪽 Y축)**: ROAS(배) 추이 — 신규 유입 고객 기준, '신규 유입 기준' 라벨 표기
- **ROI 선(오른쪽 Y축)**: ROI(%) 추이 — 전체 매출 기준, '전체 매출 기준' 라벨 표기
- **X축**: 기간 토글에 연동 (주간/월간/연간)
- **광고비 막대(배경 계열)**: 실제 지출 수준 참고용
- 툴팁: X포인트 클릭 시 ROAS N배 / ROI N% / 매출 N원 / 광고비 N원 동시 노출
- 데이터 누락 구간: 회색 점선 구역 표시

> ⚠️ ROAS와 ROI는 분자가 다릅니다. 상승 추세이면서 ROI가 ROAS보다 높이 나타나는 경우, 기존 고객 재계약 성과가 높은 신호일 수 있으므로 두 지표를 함께 해석하도록 툴팁 안내 문구 삽입.

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


### 7.4 기존 고객 목록 관리 및 정보 편집
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자

매출 관리 페이지에서 신규 등록된 고객과 기존에 유입된 신규 고객의 재계약 등으로 기존 고객으로 분류된 고객 목록을 통합 관리합니다.

## 기존 고객 분류 기준
- **자동 분류**: 매출 관리에서 신규 입력된 고객(= 유입 관리에 연결되지 않은 고객)은 자동으로 기존 고객으로 등록됨
- **수동 분류**: 유입 고객이었던 신규 고객이 재계약하는 경우, 담당자가 '기존 고객으로 전환' 처리 가능
- **표기**: 목록에서 신규 유입 고객은 '신규' 배지, 기존 고객은 '기존' 배지로 시각적 구분

## 목록 화면
- **필터**: 고객 유형(전체 / 기존) / 담당자 / 마지막 계약일 범위
- **정렬**: 이름 가나다순 / 마지막 계약일 최신순 / 누적 매출액 높은순
- **열 구성**: 고객명, 고객 유형(신규/기존), 연락처, 담당자, 마지막 계약일, 누적 매출액
- **검색**: 고객명 / 연락처 키워드 검색

## 개별 고객 상세 화면
- 기본 정보: 이름, 연락처, 게시맩, 담당자, 첫 계약일, 유입 가능 여부(신규 유입 고객 연결 상태)
- 계약 이력: 해당 고객의 등록된 전체 계약 목록 (연동, 담당자, 계약일, 계약금액)
- 수정: 고객 기본 정보(이름, 연락처, 담당자, 메모) 편집 가능
- **연결 상태**: 신규 유입 고객에서 전환된 경우 '유입 데이터 보기' 링크 제공


#### 7.4.1 기존 고객 자동 등록 및 분류 규칙

매출 관리에서 입력된 고객이 기존 고객으로 자동 등록되는 규칙을 정의합니다.

## 자동 등록 트리거
- 유입 관리 페이지에 연결되지 않은 신규 고객으로 계약 매출을 입력하면 자동으로 '기존 고객'으로 등록
- 신규 유입 고객이 재계약 등으로 계약이 추가 입력되는 경우, 담당자가 '기존 고객으로 전환' 수동 처리 가능

## 중복 마케징 방지 (동일 연락처 검색)
- 이름 또는 연락처가 이미 등록된 기존 고객과 일치할 경우 '동일 고객으로 병합하시겠습니까?' 확인 팬업 노출
- 병합 선택 시 기존 고객에 계약 이력이 연결됨
- 병합 거절 시 별도 기존 고객으로 등록

## 자동 등록 시 포함 정보
- 이름, 연락처, 담당자, 첫 맳망 계약일
- 고객 유형: '기존' 자동 표기
- 유입 데이터 연결 여부: '연결 없음' 설정

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 7.4.2 기존 고객 다중 계약 이력 조회

기존 고객이 여러 차례 계약한 이력을 시간순으로 확인할 수 있는 화면입니다.

## 계약 이력 목록
- 열 구성: 계약일, 유입 경로(유입 데이터 연결 여부), 채널, 담당자, 계약 금액, 상태
- '신규 유입' 태그: 유입 관리에서 신규로 유입된 건에만 표기
- **누적 매출 요약**: 상단에 '이 고객의 지금까지 누적 엣 계약 · 눈계 N월 · 합산 계약금액 N원' 요약 카드

## 계약 기간 비교
- 첫 계약일 ~ 최신 계약일 기마로 '곘래 기간 N요일' 자동 계산
- 재계약 주기 파악에 활용 가능
- 재계약 주기가 일정 이상 지났을 경우 '재계약 여락 시점일 수 있음' 안내 표시

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자


#### 7.4.3 매출 입력 시 기존 고객 자동완성 및 정보 연동

매출 관리에서 새 매출을 입력할 때 기존 고객 정보를 자동으로 불러와 입력을 도와주는 기능입니다.

## 자동완성 동작
- 고객명 또는 연락처 일부를 입력하면 기존 고객 목록에서 일치하는 것을 드롭다운으로 제안
- 제안 드롭다운에는 '이름 · 연락처 · 마지막 계약일 · 담당자' 요약 노출
- 선택 시 한사, 연락처, 담당자 필드 자동 채움

## 연동되는 정보
- 이름, 연락처, 담당자
- 지난 계약 요약: '지난 계약 N건 · 마지막 계약 N원 · N일 전' 인라인 표시
- 사용자가 필요 시 직접 입력으로 우선 적용 가능 (자동완성은 덮어쓰기 가능)

## 예외 시나리오
- 의도적으로 기존 고객을 선택하지 않는 경우: '신규 고객으로 입력' 선택 가능 → 매출 완료 후 기존 고객으로 자동 등록
- 기존 고객 연결 전환: 매출 입력 후에도 연결할 기존 고객를 검색해서 연결 론전 가능

> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 마케팅 팀장, 담당자(실무자), 관리자
